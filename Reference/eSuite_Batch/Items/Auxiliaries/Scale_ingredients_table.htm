eSuite - Scale Ingredients (Scale Text) Table

eSuite > Items > Item auxialaries > Scale Tables

Information

The Scale ingredients (scale text) table contains the lists of ingredients for items that are made in the store. The Scale text table works with the Scale information table.

Displaying the Scale text table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Scale Tables and then on the Scale Ingredients: the Scale Text tab opens in the same window, displaying the Scale Text List.
  3. To modify an existing scale text record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Scale text table opens, overlapping the work area.
  4. To create a new scale text record, click on Add in the bottom menu of the grid: the Scale text table opens, overlapping the work area.

Content

Here is a description of the Scale text table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier (the store group).
The default value of the Target field is PAL.
Number The Number field contains the identifier number of the scale ingredient information. It is used to link the scale information to the scale ingredient information.
S-Dept The Sub-department field links the scale ingredients information to a sub-department.
For details, see the Sub-department field section below.
OperLevel The Operator level field contains the number of the minimum operator level needed to manage the scale ingredients records.
Font size The Font size field specifies the size of the font to be used for the ingredient text.
Descriptor The Descriptor field contains the main description of the ingredients.
Ingredient text The Ingredient text field contains the list of ingredients.
For details, see the Ingredient text field section below.

The Sub-department field

The Sub-department (S-Dept) field links the scale ingredients information to a sub-department.

In most POS systems, item sales are linked to sale categories for reporting purposes. Because different POS systems use different terminology for sale links, it is important to understand the eSuite terminology and how it relates to the POS system.

The eSuite software uses a three-tier sale link level organization:

  1. Item sales are accumulated in sub-departments;.
  2. Sub-department sales are accumulated in departments (a department sales total contains sales from one or more sub-departments).
  3. Department sales are accumulated in group sales (a group sales total contains sales from one or more departments).

The eSuite software sale links levels (sub-departments, departments and groups) are defined in the Sub-department and Department tables.

Whether or not the POS system uses the same terms, eSuite requires that items be linked to a sub-department or to a first sale link level.

The Ingredient text field 

The Ingredient text field contains the list of ingredients.

The Max width and Max height fields define the available space for the ingredient information on the label. Once a sub-department is selected, these two fields are automatically adjusted according to the definition of the smallest scale located in the specified sub-department. The values come from the XchScale driver information.

The Ingredient text field has a line auto-feed feature. The information is deployed as a single string and no carriage return. It is required to calculate the number of spaces to arrive at the next line.
