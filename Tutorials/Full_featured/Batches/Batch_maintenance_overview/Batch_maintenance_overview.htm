Batch Maintenance

eSuite > Items > Batches

Information

Batches are used for managing changes on multiple items.

They may be related to the main item, POS, price (price, general price, sale, TPR, instore, redeem, main item, POS and coupon link batches) and cost (cost, general cost, allowance, rebate, discounts, cost breaks and bill backs batches).

The eSuite software also allows creating open batches and delete batches. Open batches mix batches related to item maintenance, price and cost. Delete batches help deleting items from the database.

  • From a host office: batches must be deployed to the stores using Deploy batch.
  • From a backoffice: batches must be executed (Apply single batch or Apply all due batches), then deployed to the front-end workstations using Deploy change.

NOTE: When creating a batch for new items, these items must be added to the Future Price batch. This prevents orphan main item records to be deployed to the store if no price is meant for that store.

Batches in eSuite

Batches are managed from the common eSuite modules to make the workflow more efficient.

The batch options are accessible from the Batches entry under Items. They are grouped in the Start window.

  • Batches opens the List batch wizard for batch filtering. It provides access to the list of instore batches.
  • All opens the list of all instore batches.
  • New displays the list of all batch types. It provides access to the Batch header table.
  • Items opens the wizard for filtering item batches.
  • Prices opens the wizard for filtering price batches.
  • Costs opens the wizard for filtering cost batches.
  • POS information opens the wizard for filtering point of sale batches.
  • Main item opens the wizard for filtering main item batches.
  • Deploy batch allows sending batches from the head office to the stores.
  • General changes allows displaying the changes that have been made in batches.
 

Batches

The Batches entry in the Start window provides access to the list of instore batches.

It opens the List batch wizard for filtering according to batch status and type, description, start date range, end date range.

The following fields are available:

  • Batch status allows the selection between future and old batches.
  • Batch type provides the list of future batch types.
    It queries the Type field in the Batch header table.
  • Description allows entering keywords for filtering batches by their description.
    It queries the Description field in the Batch header table.
  • Start date range allows setting the length of time in which the search batches became effective.
    It queries the Start date field in the Batch header table.
  • Stop date range allows setting the length of time in which the search batches expired.
    It queries the Stop date field in the Batch header table.
  • Execute launches the search.

Once the query executed, results are displayed in the Instore batches window:

In the top menu:

  • Delete allows deleting the selected batches.
    It prompts a confirmation message.
  • Add batch displays the list of batch types.
    It allows creating a new batch.

In the bottom menu:

  • Edit opens the Batch header table.
  • Select places the selected batches in the Context tools window.
  • Items opens the list of the items in the selected batch.
  • Execute allows executing the selected batches.
    It opens a wizard that allows the selection of a date for batch execution.

To view items for old batches:

  • From the Wizard, select Old batch for the Batch Status and Execute

 

  • From the Grid, select the batch to view the items and click on the items.

 

  • It will retrieve all items that belong in the selected old batch.

Items

The Item list entry in the Start window provides access to the items available in the database.

It opens the Items - General list wizard for filtering by item code, description, target, store, dub-department, category, vendor.

The following fields are available:

  • Any code allows filtering by any of the codes in the main, POS, price and cost tables.
  • Any description allows filtering by any description in the main, POS, price and cost tables.
  • Target allows filtering by the target.
  • Vendor opens the list of items belonging to the selected vendor.
  • Sub-department opens the list of items according to the sub-department.
  • Category opens the list of items according to the category.
  • Show:
    • POS opens the list of results with POS-related information.
    • Price opens the list of results with price-related information.
    • Cost opens the list of results with cost-related information.

Once the query executed, results are displayed in the Items - General list window:

In the bottom menu of the grid:

  • Edit provides access to the following tables:
    • Special items: Scale information, Alternate codes, Contract price, Electronic coupon link, Inventory trigger, Recipe, Kit, Modifier, Item classification, Images.
    • Item batch: Future main item and Future POS.
    • Price batch: Future price, Future price general, Future sale, Future TPR, Future instore, Future redeem.
    • Cost batch: Future cost, Future cost general, Future allowance, Future rebate, Future discount, Future break quantity, Future billback.
    • Coupon: Edit coupon.
  • Analyze provides access to item queries and subqueries.
  • Action allows assigning and unassigning promotions, generating labels and deploying items.

Prices

The Prices entry in the Start window provides access to the price-related batches.

It opens the Price batch wizard for filtering by batch number, description and type, item code and description, target, vendor ID and code.

The following fields are available:

  • Batch opens the list of price-related batches.
    It is populated with the number of the batch selected in the list.
    Mandatory field. When populated, it enables the Execute button.
  • Batch description contains the description of the batch.
  • Batch type displays the type of price-related batch.
  • Item code allows entering or selecting an item code.
  • Description allows entering keywords for filtering items by description.
  • Target gives access to the list of store numbers.
  • Vendor ID indicates the vendor to which the searched item is linked.
  • Vendor code indicates the code used by the vendor for the searched item.
  • Show cost allows displaying the cost values for the items.
    The Cost column is shown when selected in the header menu.
  • When Show existing items is checked, the list of results will only display the items in the selected batch.
  • By store allows dislaying the results as grouped by store.
  • Execute is disabled by default. Enabled when the Batch type field is populated.

Once the query executed, results are displayed in the price batch window of the selected batch.

In the bottom menu:

  • Edit provides access to the following tables:
    • Special items: Scale information, Alternate codes, Contract price. Electronic coupon link, Inventory trigger, Recipe, Kit. Modifier, Item classification, Images.
    • Item batch: Future main item and Future POS.
    • Price batch: Future price, Future general price, Future sale, Future TPR, Future instore, Future redeem.
    • Cost batch: Future cost, Future general cost, Future allowance, Future rebate, Future discount, Future break quantity, Future billback.
  • Analyze allows performing item subqueries and linked queries.
  • Action allows generating labels.

Deploy batch

The Deploy batch job sends batches from the head office to the stores.

To deploy a batch:

  1. From the Items entry in the launcher, click Batches.
  2. From the Start window, select Deploy batch: the wizard opens on the desktop, allowing the selection of the batch to deploy.
  3. Select the batch number from the drop-down menu of the Select a batch field.
  4. Click on Confirm: the wizard opens on the desktop, allowing the selection of a target.
  5. Select the target from the drop-down menu.
  6. Click on Confirm to launch the deploy job.

Editing batches

Batches in eSuite may be edited from the Batch header table or from the future tables.

Editing the main batch information

The main batch information is edited in the Batch header table. For details, refer to the Batch header table document.

Editing batch-type information

Specific information, such as methods or rebate amounts, may be edited through the future tables:

  • Future price
  • Future TPR price
  • Future sales price
  • Future instore price
  • Future redeem price
  • Future general price
  • Future cost
  • Future allowance
  • Future rebate
  • Future discount
  • Future break quantity
  • Future cost break
  • Future bill back
  • Future general cost
  • Future POS
  • Future main item

The future tables are available from the following options: Items (all future tables), Prices (price future tables), Costs (cost future tables), POS information (POS future tables) and Main item.

The steps below describe how to access the cost-related future tables:

  1. In the launcher, click the arrow in the Items entry to display its menu.
  2. Click on Batches: the Start window opens in the Batches work area.
  3. Select Costs.
  4. In the wizard, click on the Batch field to disaply the list of all cost-related batches.

  5. If necessary, use the Search and Macro fields to filter the batches.
  6. Select a batch and click OK: the wizard displays the batch number and type:
  7. Click Execute.
  8. Select an item in the list and click Edit in the bottom menu.
    The following options are available:
    • Special items provides access to some of the item tables.

    • Item batch provides access to the main item batches and the POS batches.

    • Price batch displays the list of price-related batch types.

    • Cost batch displays the list of cost-related batch types.

  9. Select a type of cost batch to open the list of corresponding batches.
  10. Select a batch from the list.
  11. Click OK: the corresponding future table opens on the desktop.
  12. Make the necessary changes and click Save in the toolbar.
    The modified item will automatically be added to the current batch.

Editing items in the grid

Batch grids allow editing certain item information, such as price and cost. This will automatically add the modified items to the selected batch.

  1. In the Start window, select either Prices, Costs, POS information or Main item:
  2. Enter the search criteria in the wizard.

    Check Show existing items to display only the items that are already part of the selected batch.
    Uncheck Show existing items or the list to display all the items in the database.
  3. Click Execute: the list of items openss within the selected batch.
  4. Choose the item(s) to add to the batch and edit the price, cost, POS information.
  5. Press Enter to apply each change.
  6. Click Save in the toolbar.
  7. To display the list of items that are part of the current batch, take a step back to the wizard and select Show existing items:
  8. Click Execute to list the items that are part of the selected batch and the new values:

Deleting batches

Batches in eSuite may be deleted from the list of instore batches:

  1. From the Items entry in the launcher, click Batches.
  2. In the Start window, select All to display the Instore batches list.
  3. Select the batches to delete.
  4. Click Delete in the toolbar: a confirmation message is prompted.
  5. Click Yes to delete the selected batches.
