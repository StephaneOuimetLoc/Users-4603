eSuite - Recipe Table

eSuite > Items > Master > Item list

Information

The Recipe table allows retailers to create a relationship between the products used as ingredients in a recipe and the resulted items.

The Recipe table may be used in different retail scenarios, such as when creating recipes in the Deli department or producing meat cuts. Meat cuts are obtained when a bulk piece of meat is separated into final retail cuts like steaks, rumps or ground beef.

This feature can also be used outside of the grocery world. For example, if building and selling sheds, recipes may help creating a shed item and all its constituent parts.

In eSuite, recipes are considered items and have their own unique UPCs. The major difference is that they include other entries from the item list as part of their ingredients. A recipe may contain an unlimited number of items. These items must already exist in the item list.

After the first recipe is created in the system, eSuite must be reloaded. The software checks if there are any recipes in the system, and if there are not, it disables the recipe update engine to speed up the transaction process.

Displaying the Recipe table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens in the Items work area.
  2. Click on Item list: the Wizard tab opens in the same window, displaying the Items - General List.
  3. Enter the search criteria and click Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select the recipe item (the final product) in the list and click on Edit on the bottom of the item grid: the option menu is displayed.
  5. Select Special items and then Recipe: the Recipe edit table opens overlapping the work area.

Items that will make up the recipe, including the recipe item, need to be added one by one. See the Creating recipe entries in eSuite topic below.

Layout

The Recipe table is divided in two parts: the main section and the Recipe table grid.

Main section

* Click on a field to get information on it.

Recipe table grid

* Click on a field to get information on it.

The Recipe table grid offers quick information on the items that are used in the recipe (final product and ingredients). The following columns are displayed: Code and Quantity.

Columns are movable, resizable and may be selected/deselected for a customized experience. They allow sorting the records in an ascending or a descending order.
By default, records are displayed in ascending order by code. Selecting a record in the grid allows displaying and editing its information in the main section of the Recipe table.

Content

Here is a description of the Recipe table fields and their functionalities:

FIELD DESCRIPTION
Main section
Recipe code The Recipe code field contains the codes and the description of the items used in the recipe (final product and ingredients).
It is used to link all the items in the recipe.
Items used in the recipe table must already exist in the database.
Quantity The Quantity field contains the quantity of the item included in the recipe.
For details, see the Quantity field section below.
Ratio cost The Ratio cost field contains the percentage the produced item holds in the recipe.
For details, see the Ratio cost field section below.

The Quantity field

The Quantity field contains the quantity (positive or negative) of the item included in the recipe.

Positive quantities are used for the final products, i.e.  the items that have been obtained using the recipe.
Negative quantities are used for the ingredients, i.e. the items that have been used in the recipe.

When the production of the recipe is executed, items are subtracted from the ingredient inventory and items are added to the final product inventory.

NOTE: The unit of the Quantity field follows the unit of the item, i.e. if the item is flagged Scalable, the quantity is in pounds or kilograms and if the item is not flagged Scalable, the quantity is in unit.

Back to layout

The Ratio cost field

The Ratio cost field contains the percentage that the final item holds in the recipe.

The cost of all ingredients (the items that have a negative quantity) is added up and the total is assigned to the final item. When there are more final products obtained from the recipe, the ratio cost must be spread to all of them.

Creating recipe entries in eSuite

Creating a recipe for a single main item made up of multiple ingredients or parts

Before proceeding, make sure that all recipe items (final product and ingredients) exist in the database.

To enter the main recipe item (final product) in the Recipe table, do the following:

  1. In Items - General list, select the main recipe item (the final product).
  2. Open the Recipe table.
  3. Press Add in the toolbar to enable editing. The Item field displays the recipe code (read-only).
  4. Add the recipe UPC number in the Recipe code field and press Enter.
    The Recipe code field displays the recipe UPC number and description.
  5. In the Quantity field, enter the number of final products that will be obtained by using this recipe.
    In most cases this is 1, but it could be different depending on the recipe and the ingredients that are used.
  6. As this is the final product, enter 100 in the Ratio cost field.
  7. Press Enter to save the changes or click Save in the toolbar.
To enter ingredients for this recipe, do the following:
  1. Press Add in the Recipe table toolbar to enable editing.
  2. Click the arrow in the Recipe code field to open the Recipe code window.
    Enter the ingredient item description and press Go. In the list of results, select the item and press OK.
    In the Recipe table, the Recipe code field displays the item UPC number and description.
  3. In the Quantity field, place the amount of this ingredient used in the recipe; this can be a decimal quantity if using less than a full item of the ingredient, e.g.: 0.5 g or 1/2 banana.
  4. When making a recipe made up of several ingredients, leave the Ratio cost field empty in the ingredient entries.
  5. Press Enter to save the changes or click Save in the toolbar.

Creating a recipe containing multiple items or parts created from one main ingredient/source

Before proceeding, make sure that all recipe items (final product and ingredient/source) exist in the database.

To enter the main recipe item ingredient/source in the Recipe table, do the following:

  1. In Items - General list, select the main recipe ingredient. This is the main source from which the individual items are made.
  2. Open the Recipe table.
  3. Press Add in the toolbar to enable editing. The Item field displays the recipe item source (read-only).
  4. Add the source ingredient UPC number in the Recipe code field and press Enter.
    The Recipe code field displays the source ingredient UPC number and description.
  5. In the Quantity field, enter the amount of the ingredient item used as a source for creating the other items in the recipe.
    For the source ingredient, the quantity field is negative as this will be relieved from inventory when creating the produced items. In many cases this may be 1, but it could be different depending on the recipe and the ingredients that are used.
  6. Leave the Ratio cost field empty, as this is the source ingredient from which other items will be created.
  7. Press Enter to save the changes or click Save in the toolbar.
To enter the items created for this recipe, do the following:
  1. Press Add in the Recipe table toolbar to enable editing.
  2. Click the arrow in the Recipe code field to open the Recipe code window.
    Enter the item description and press Go. In the list of results, select the item and press OK.
    In the Recipe table, the Recipe code field displays the item UPC number and description.
  3. In the Quantity field, place the amount created of this item for the recipe.
  4. Fill in the Ratio cost field.
  5. Press Enter to save the changes or click Save in the toolbar.
