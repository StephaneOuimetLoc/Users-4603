eSuite - Coupon Value Table

eSuite > Items > Item auxiliaries > UCC Coupon Value Table

Information

The UCC Coupon value table contains the UCC 5 standard definitions, used with the vendor coupons. It is already filled in according to the UCC 5 standards and does not need to be changed, unless the UCC 5 standards change.

NOTES:
  • The Coupon value table is used when UCC prefix 5 coupons are scanned according to the default standards (entries 1 to 99).
  • prefix 99 coupons also use the default standards (entries 1 to 99) of the Coupon value list. See the UCC/EAN prefix 99 coupons information for more details.
    • UCC/EAN prefix 98 coupons use the entries 101 to 199 of the Coupon value list. These entries must be added according to the store's needs. See the UCC/EAN prefix 98 coupon information for more details.

Displaying the UCC Coupon value table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on UCC Coupon value table: the tab opens in the same window, displaying the Coupon Value List.
  3. To modify an existing coupon value record (entries 101 to 199), select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Coupon value table opens, overlapping the work area.
  4. To create a new coupon value record (entries 101 to 199), click on Add in the bottom menu of the grid: the Coupon value table opens, overlapping the work area.

Content

Here is a description of the Coupon value table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier (the store group).
It is a required field used to uniquely identify records in the Coupon Value list.
The default value of the Target field is PAL.
Code The Code field contains the code that identifies the UCC 5 standard.
It corresponds to the last two digits of the UPC on the vendor coupon (excluding the check digit).
Descriptor The Descriptor field contains the description of the UCC 5 standard.
Min The Minimum field contains the minimum quantity of the UCC 5 standard, required to obtain the coupon.
Value The Value field contains the value of the UCC 5 standard that corresponds to the coupon discount value.
Method The Method field contains the code that defines the behavior of the UCC 5 standard.
For details, see the Method field section below.

The Method field

The Method field contains the code that defines the behavior of the UCC 5 standard.

* Click on a field to get information on it.

To enter a method, type its code in the Method field or click to open the field menu and make a selection.

The available methods are:

METHOD BEHAVIOUR
1.
Manual value entry		 The Manual value entry specifies that the value of the coupon is entered manually by the cashier.

The items are matched using the manufacturer code. The family code may be added to the matching. See the Printed coupon matching topic for more details.
EXAMPLE: For a special promotion, the customer gets a new item for free.
2.
Buy x of same item, get y free		 The Buy x of same item, get y free method specifies that y items are given for free when x of the same item are bought by the customer in a transaction. The item given for free is the same as the items bought.

The items are matched using the manufacturer code and the family code. See the Printed coupon matching topic for more details.

NOTE: The Buy x of same item, get y free method is currently not effective, i.e. the system does not verify whether the same items are bought. It will be effective in a future version of the software.

EXAMPLE: The customer gets 1 bag of Oreos for free when he buys 2 bags of Oreos. A coupon can be used for every 2 bags of Oreos bought (see the note).
3.
Buy x or more of same item,
get y free		 The Buy x or more of same item, get y free method specifies that y items are given for free when x or more of the same item are bought by the customer in a transaction. The items given for free are the same as the items bought.

The items are matched using the manufacturer code and the family code. See the Printed coupon matching topic for more details.

NOTE: The Buy x or more of same item, get y free method is currently not effective, i.e. the system does not verify whether the same items are bought, and a coupon is accepted each time x items are bought. It will be effective in a future version of the software.

EXAMPLE: The customer gets 1 bag of Oreos for free when he buys 2 or more bags of Oreos. Even if he buys 4 bags of Oreos, only 1 free bag of Oreos will be given (see the note). 
4.
Buy x, get y free		 The Buy x, get y free method specifies that y items are given for free when x items of the same manufacturer are bought by the customer in a transaction. The items given for free are the cheapest ones.

The items are matched using the manufacturer code and the family code. See the Printed coupon matching topic for more details.

EXAMPLE: When the customer buys 2 bags of Oreos (same item), he gets one for free.
 or
 When the customer buys 1 bag of Oreos and 1 box of Christie crackers (different items), he gets the cheaper one for free.
5.
Buy x or more, get y free		 The Buy x or more, get y free method specifies that y items are given for free when x or more items of the same manufacturer are bought by the customer in a transaction. The items given for free are the cheapest ones.

The items are matched using the manufacture code and the family code. See the Printed coupon matching topic for more details.

NOTE:The Buy x or more, get y free method is currently not effective, i.e. a coupon is accepted each time x items are bought. It will be effective in a future version of the software.

EXAMPLE: When the customer buys 2 items among Oreo bags and Christie crackers boxes, he gets the cheaper one for free. Even if he buys 4 items among Oreo bags and Christie crackers boxes, only 1 item for free will be given (see the note).
6.
Get y amount off		 The Get y amount off method specifies that the discount value (y) entered in the Value field is given when 1 item is bought by the customer in a transaction.

The item is matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

EXAMPLE: The customer gets $0.50 off when he buys an Oreo bag. A coupon can be used for each Oreo bag bought.
7.
Buy x, get y amount off		 The Buy x, get y amount off method specifies that the discount value (y) entered in the Value field is given when x items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

EXAMPLE: When the customer buys 2 items among Oreo bags and Christie crackers boxes, he gets $0.50 off on the cheaper one. A coupon can be used each time 2 items among Oreo bags and Christie crackers boxes are bought.
8.
Buy x or more, get y amount off		 The Buy x or more, get y amount off method specifies that the discount value (y) entered in the Value field is given when x or more items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

NOTE: The Buy x or more, get y amount off method is currently not effective, i.e. a coupon is accepted each time x items are bought. It will be effective in a future version of the software.

EXAMPLE: When the customer buys 2 items or more among Oreo bags and Christie crackers boxes, he gets $0.50 off on the cheapest one. Even if he buys 4 items among Oreo bags and Christie crackers boxes, only $0.50 off will be given (see the note).
9.
Buy x or more of same item,
get y amount off		 The Buy x or more of same item, get y amount off method specifies that the discount value (y) entered in the Value field is given when x or more of the same item are bought by the customer in a transaction.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

NOTE: The Buy x or more of same item, get y amount off method is currently not effective, i.e. the system does not verify that same items are bought, and a coupon is accepted each time x items are bought. It will be effective in a future version of the software.

EXAMPLE: The customer gets $0.50 off when he buys 2 or more Oreo bags. Even if he buys 4 Oreo bags, only $0.50 off will be given (see the note).
10.
Buy x of same item,
get y amount off		 The Buy x of same item, get y amount off method specifies that the discount value (y) entered in the Value field is given when x of the same item are bought by the customer in a transaction.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

NOTE: The Buy x of same item, get y amount off method is currently not effective, i.e. the system does not verify that same items are bought. It will be effective in a future version of the software.

EXAMPLE: The customer gets $0.50 off when he buys 2 Oreo bags. A coupon can be used for each 2 Oreo bags bought (see the note).
11.
Buy x, get y percent off		 The Buy x, get y percent off method specifies that the discount value (y) entered in the Value field is considered as a percentage of discount given when x items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

Example
When the customer buys 2 items among Oreo bags and Christie crackers boxes, he gets 5% off on the cheaper one. A coupon can be used each time 2 items among Oreo bags and Christie crackers boxes are bought.

 Buy x or more, get y percent off		 The Buy x or more, get y percent off method specifies that the discount value (y) entered in the Value field is considered as a percentage of discount given when x or more items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

EXAMPLE: When the customer buys 2 or more items among Oreo bags and Christie crackers boxes, he gets 5% off on the cheaper one. Even if he buys 4 among Oreo bags and Christie crackers boxes, only $0.50 off will be given (see the note).
13.
Buy x, get y amount per pound off		 The Buy x, get y amount per pound off method specifies that the discount value (y) entered in the Value field is given per pound when x pounds of items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

EXAMPLE: The customer buys 10 pounds of meat, he gets $0.20 off per pound on these 10 pounds.
14.
Buy x or more, get y amount per pound off		 The Buy x or more, get y amount per pound off method specifies that the discount value (y) entered in the Value field is given per pound when x or more pounds of items of the same manufacturer are bought by the customer in a transaction. The coupon discount is given on the cheapest item.

The items are matched using the manufacture code. The family code may be added to the matching. See the Printed coupon matching topic for more details.

NOTE: The Buy x or more, get y amount per pound off method is currently not effective, i.e. a coupon is accepted each time x pounds are bought. It will be effective in a future version of the software.

EXAMPLE: The customer buys at least 5 pounds of meat, he gets $0.10 off per pound.
