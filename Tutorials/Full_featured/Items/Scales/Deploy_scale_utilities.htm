eSuite - Scale Tools

eSuite > Items > Item auxiliaries > Scale Tables

Information

The document explains how Scale tools (XchScale - Tools) can be used when working with production scales.

 

The Scale Tools page can be opened from the 'Items' drop-down in the left hand menu of eSuite. This will allow utilization of additional features related to production scales.

NOTE: If using ePlum, the only utilities available are "Send item to Scale" and "Send future item change to scale". There is no read back between SMS and ePlum.

Send Item to Scale

'Send item to scale' is used for deploying item changes to the production scales. The image to the above demonstrates the default values set (date is blurred) when deploying changes to the production scales.

  • Data to send: - There is the option to send the item only, item and ingredient, ingredient only or nutrifact only.
  • Sub-Department Range: - There is the option to send a specific sub-department range.
  • Target: - The target scale to send the data. DSC is set by default, which means all scales.
  • Date: - This window allows a date to be chosen.
  • Send data by: - There are two (2) options that can be selected:
    • 'Add or Replace' allows the item to be added or replaced (if re-using an existing PLU)
    • 'Create and Load' allows the items in the SMS database to overwrite all exisiting data in the productions scale.
      NOTE: If this option is selected, be aware that the data that was in the scale prior to deployment will be removed/erased.
 
  • After selecting either of the two (2) options for 'Send data by', taction confirmation pop-ups will display to confirm the action. Select 'Yes' to proceed to deploy changes or 'No' to cancel.

Get Item from Scale

'Get item from scale' is used for importing items from production scales to SMS. When the processor(s) is finished processing all the data retrieved from the scale, an external batch will be created.

  • Data to retrieve: - There are three (3) options that can be selected:
    • Item Only
    • Ingredient Only
    • Nutrition Only
  • Target: The target scale to send the data. DSC is set by default, which means all scales.

External Batch Action

By default with production scales, the "ADD" action type will be for data application. If another action is required, it will need to be modified manually in the batch header. Possible actions are: ADD, ADDRPL, CHANGE or REMOVE. Refer to the image below.

WARNING: This should only be done by a technician upon a new install/upgrade. ALL items in the external batch should be reviewed to ensure data accuracy/integrity before being applied/executed.

Poll Report

'Poll Report' is used to obtain a report.

  • Target: - The target scale to send the data. DSC is set by default, which means all scales.
  • Date: - This window allows a date to be chosen.
  • Reset the scales: - There are two (2) options that can be selected:
    • No (X Reading)
    • Yes (Z Reading)

NOTES
• 'X Reading' refers to simply reading the data from the scale, while 'Z Reading' refers to reading and then resetting the data from the scale.
• Totalizer 8630 (Packaged) is used when polling production during the End of Day (Scheduler) process.

Status Check

'Status Check' is used to check the 'Online' or 'Offline' status of the scale.

  • Target: - The target scale to send the data. DSC is set by default, which means all scales.

NOTE: An urgent message will only be received if there is an issue with the scale connectivity. Urgent messages should always be reviewed in order to rectify the issue. Issues left unresolved may have consequences.

Send Future Item Changes to Scale

'Send future item changes to scale' is used for deploying future item changes to the production scales.

  • Date: - This window allows a date to be chosen.
  • Target: - The target scale to send the data. DSC is set by default, which means all scales.

NOTE: This feature will not work with random weight codes entered as alternate codes (F154).
