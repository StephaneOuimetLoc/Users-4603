eSuite - Price Generation for Items and Batches

eSuite > Items > Price Generation

Information

The eSuite software offers two price generation tools by splitting up the functions for Items and Batches. These tools will offer different options, depending on whether they refer to items or to batches in eSuite.

The price generation functions may be found in multiple areas within eSuite. See below for the locations.

Price generation for items

Price generation functions for items are selected either with the "$" sign icon or by confirming "Yes" when using the multi-selection in the "Items - Price list" action. Once selected, they open the Price generate window:

Batches

From the eSuite Batch module, click on Batches in the launcher to open the Batches start window. The Price generate batch button is on the right of the Prices button.

NOTE: In order to use this function, the batch of choice (retail, TPR, sale, instore, etc.) needs to be created beforehand, as it needs to be selected in the Price generate batch wizard in order to execute. The Execute action button will remain greyed out until an existing batch is selected from the Batch field pull-down.

Selecting an existing batch will automatically populate the following three fields: Batch, Batch description and Batch type. This allows using multiple options as the same time.

Description of price generation functions

The following is a description of the price generation functions available in eSuite for items and for batches.

Send to section

The Send to section is available in the Price generation window for items. It indicates the selling price for which the price value is generated.

Four selling prices are available:

NAME DESCRIPTION
Retail Indicates that the generated price value will be sent to the Retail price (Price table).
TPR Indicates that the generated price value will be sent to the TPR price (Price table).
Sale Indicates that the generated price value will be sent to the Sale price (Price table).
Instore Indicates that the generated price value will be sent to the Instore price (Price table).

Base cost section

The Base cost section is common to the price generation tool for both items and batches. It specifies the base from which the selling price (profit) is generated. Its default value is Base.

Three bases are available in eSuite:

  • Base specifies that the selling price is generated (or the profit is made) from the Base cost field of the Cost table.
  • Landed specifies that the selling price is generated (or the profit is made) from the sum of the base cost amount and the applicable fees.
  • Net specifies that the selling price is generated (or the profit is made) from the Cost field of the Cost table.
    Usually, the Net cost value is the same as the Base cost value unless discounts, rebates, etc. are given by the vendor. In that case, the Net cost value changes.

The three tabs of the Base cost section allow changing the default base from which the selling price is generated (from which the profit is made).

NAME DESCRIPTION
+ (plus) Cost fees Adds the value of the cost fees to the seeling price that will be generated.
Checked by default in the Landed and Net tabs.
Enabled when the - Cost fees checkbox is disabled and vice-versa.
- (minus) Cost fees Subtracts the value of the cost fees from the selling price that will be generated.
Enabled when the + Cost fees checkbox is disabled and vice-versa.
- (minus) Off-invoice deals Subtracts the value of the off-invoice deals from the selling price that will be generated.
Checked by defautlt in the Net tab.
- (minus) Bill-back deals Subtracts the value of the bill-back deals from the selling price that will be generated.
Apply future batches on cost (for batches only) Applies the changes made in the selected batch.
Checked by default in all tabs (Base, Landed and Net).

Margins section

The Margins section is common to the price generation tool for both items and batches.

The Generate price utility is used to calculate the selling prices from the cost amounts entered in the Cost table. The calculation is based on a margin.

NAME DESCRIPTION
Retail Contains the retail profit margin on the item in percentage. It is used to generate the item retail price.
Package Contains the package price profit margin on the item in percentage.
Limited Contains the limited quantity price profit margin on the item in percentage.
VAT Contains the VAT taxes profit margin in percentage.

Buttons section

The price generation tools have several buttons to generate the selling prices. Each one of them has its particular function.

NAME DESCRIPTION
Yes The Yes button generates the selling price for the current item/batch.
Always The Always button automatically generates the selling price for each item whose information has been changed in the Cost table during the current maintenance session, without having to open the Generate price window each time.
No The No button does not generate the selling price for the current item.
Never The Never button does not generate any selling price for any item during the current maintenance session.
Cancel The Cancel button closes the Price generate window without generating any selling price.
