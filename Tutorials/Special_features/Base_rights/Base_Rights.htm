Base Rights - User Level Permissions

The eSuite software has basic user level restrictions.

Information

These basic rights can be modified to increase or decrease the current access levels provided by the existing base rights. The Base Rights function gives the ability to create or modify access for user levels. Base rights are local to each module of eSuite.

The Base Rights function has been created with 4 main fields that will allow customizing the access levels.

  1. Win is the window that will be impacted by the change.
    It is identified by its Sencha class xtype. For example, the price window is called "item.itm.edit.price.master".
  2. Query is used to search for a component within the window selection above.
    We call a component any single visual element on a form window. Here are the supported values:
    Value Description
    self Means the window component itself. The whole window can be disabled so the user cannot open it.
    Note    : When the self value is entered, the only value that can be used in the Action field is disable.
    [empty] Empty means all the components within the window.
    All components may be set inside a form by making them all read-only.
    #itemId The most common query is based on the itemId value. There is an itemId on all components of the application.
    It may be used to control access to each component individually.
    xtype Changes may be applied to all the components of a certain class xtype.
    For example, if set to button, it will affect all the buttons in the window.
    #addBtn It will affect the button in the window.
    #deleteBtn It will affect the button in the window.
    #saveBtn It will affect the button in the window.
  3. Level is used to provide the access to the components and action functions.
    If  set it to 6, it means the "owner" and all levels above it (programmer) have access to it.
  4. Action is used to select the kind of modifier to apply to the component(s).
    Here are the actions supported:
    Value Description
    read This will set the component to read-only
    hide This will hide the component completely
    disable This will disable the component. The component will be grayed out.

Creating base rights

Start creating Base Rights for the System module and then in the other modules. From the most restrictive to the least restrictive. When viewing the Base Rights for the system module almost every element is restricted to level 7 except for a few things are level 5.

Note: Log in with a programmer level access, to have the permission to create base rights for all levels of eSuite.

In the example below, all levels other than Programmer (level 7) will be blocked in the Configurator component in the Edit section of the System module.

  1. To access to the Base Rights, click on the Session icon in the right corner of the top menu bar: a drop list option will display.

  2. Once clicked, this will bring up the Base Rights start window:

  3. Click on the Base Rights text inside the Start window to open the Base Rights list window where create, delete or undo will be available.

  4. To start adding the Base Rights, click on the Grab icon. This will automatically prompt the Grab a view field icon on the menu bar.



    Note: At this point, open any section to create access levels for,  any eSuite module .
    In this example, the window that will have restricted access is the Configurator window.Use the tab key until the section is highlighted then use the Grab a View field to return to the Base Rights section.
  5. Click on the Grab view and field icon from the top menu bar. This will bring you back to the Base Rights list window, with the Win field populated with the details of the selected window.


  6. In order to block access to the Configurator window, enter the text word self into the Query field and press enter. This will set the field to the text entered with a red outline that has not yet been saved.
  7. Press Tab or click into the Level field to select the level access name.
  8. Enter the level number (1-9) or double click in the Level field to bring up the pull-down menu and select the name of the level from the list provided.


  9. Press Enter or Tab to move the cursor over.
  10. Type in text or double click in the Action field. The double click will bring up a pull-down menu with the 3 actions allowed (read, disable and hide). In this example the operator levels 6 and below will have the element win.cfg.start disabled.
  11. Press the Enter key to put the cursor in the Comment field. As it is not first evident which each element is, confusion may be avoided by entering a comment to clarify what this setting does.
  12. Enter text to describe the base right that was created.
    Note: The Comment field is not required in order to save the base right(s) created.
  13. Click on Save to save the new base right(s).
    That will set all details entered for the created base right(s). When they are saved there are no red outlines for the fields.


Setting restrictions

To set restrictions on listed options or buttons in wizards and windows (Add, Save, Edit, pull down menus) either from the grids or Context Tools action buttons, use the cursor, the tab key and the left and right arrow keys to have the focus highlighted on the desired button/grid results.
