Mailbox - Monitoring Exchange Messages

eSuite System > Exchanges > Mailboxes

Information

  • Mailboxes are used to manage messages being sent across the different PCs on the LOC SMS environment. Each office terminal has four (4) mail boxes: an input box, an output box, normal message box and an urgent message box.
  • For more details regarding the overall Mailbox topic; visit the Mailbox Concept overview.

Monitoring Exchange Messages

One of the main daily tasks consists in verifying that all Messages in the Mailboxes are being processed. You can do this on the Exchange/Mailboxes module of the eSuite System. If a message fails to be processed the Bit Archive will be added to the file and nothing in that mailbox will be processed until the Bit Archive has been removed from the given file. Normally mailboxes would be either empty or processing incoming messages.

Displaying the Mailbox Folders

In the Launcher, click the Exchanges entry:

  1. The Wizard window opens from where you can filter your messages by Target or receiving mailbox

  3. Once your search criteria entered click Execute to display the desired contents on the folder list

  5. Double-click on a folder record from the list to access its contents (Date, file name and Size)

  7. Double-click on a file record in order toview the contents of the message

  9. You can also remove the archive bit in case a file is stuck on a mailbox and is not being processed. This will allow processing to resume.

Content

Here is a description of the Exchange provider fields and their functionalities:

FIELD DESCRIPTION
Main section  
Node The Node field contains the six-digit node number (store and terminal number).
Status The Status field contains the last status of the node (online or offline).
Fails The Fails field contains the number of failures (if the system has not been reset since).
Last fail The Last fail field contains the date and time of the last failure (if the system has not been reset since).
Description The Description field contains the description of the node.
Send driver The Send driver field contains the sending information of the exchange driver needed for the communication and the transfer of information between the node and the backoffice server.
Receive driver The Receive driver field contains the receiving information of the exchange driver needed for the communication and the transfer of information between the node and the backoffice server.)
Bottom section  
Stop The Stop option stops the communication for the selected node.
Stop all The Stop all option stops all nodes listed in the Exchange provider table.
Restart The Restart option restarts the communication with the selected node.
Restart all The Restart all option restarts all offline nodes listed in the Exchange provider table.
