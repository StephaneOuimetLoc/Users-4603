eSuite - Price Level Table

eSuite > Items > Item auxialaries > Price levels

Information

The eSuite software provides the Price level table to support different price levels.

The Price table can contain different price records for the same item based on the price level.
Customer records in the Customer table are also linked to price levels in the Shopper level table.

NOTE: Because it contains vital information to the system functionality, the Price level table requires a high security level access (owner and programmer).

Displaying the Price level table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Price levels: the Levels tab opens, displaying the Price level List.
  3. To modify an existing price level, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Price level edit table opens, overlapping the work area.
  4. To create a new price level, click on Add in the grid toolbar: the Price level table opens, overlapping the work area.

Layout

* Click on a field to have information on it.

Content

Here is a description of the Price level table fields and their functionalities:

FIELD DESCRIPTION
Level The Level field contains the price level number.
See the Level field section below.
Descriptor The Descriptor field contains the description of the price level.
It is not a required field, however a descriptor should be entered to identify the price level in the Price level list.
Warehouse to host target The Warehouse to host target field contains the target identifier, i.e. the store group.
The default value of the Warehouse to host target field is PAL.
It is a required field.

The Level field

The Level field contains the price level number. It is a required field.

NOTE:
Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.
