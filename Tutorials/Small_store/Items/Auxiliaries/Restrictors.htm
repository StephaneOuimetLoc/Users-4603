eSuite - Restriction Table

The Restriction table contains information that allows controlling the item availability; in other words, one can set time limits for when the items can be sold.

By default, selling restrictions are bypassed by the assistant, manager, owner and programmer operator levels. Ask the reseller for more details.

Displaying the Restriction table

From the launcher in eSuite Care, Manage, Buy or Sell, choose Items drop down and select Item auxiliaries to display the Start wizard

Select Restrictors to display the Restriction List then select   in the grid tool bar to display the edit table

To create a Restriction

1. Display the Restriction Edit table as above or click  in a populated edit table.

2. Enter a Level, the Descriptior and set the daily Start and Stop times an item is not allowed to be sold

Note: If it is simpler, lets say an item is only allowed to be sold on Saturday, instead of restricting every other day of the week, you can set the start and stop times for Saturday and check the box "This is a permission and not restriction".  In this case it would be allowed to sell during the specified times on Saturday.

3. To save, click the  on the edit tool bar or press enter on your keyboard.

4. To review your entry, select refresh  at the bottom of the grid.

To modify a Restriction

1. From the Restriction list grid, select/highlight the entry you wish to edit

2. Double click on the entry, press F3 on your keyboard or click on Edit from the bottom grid menu.

3. The edit table will open

4. Modify as required

5. Save

6. Refresh

Applying a Restriction

Once created, the restriction level needs to assigned to the item(s). There are 3 ways to accomplish this, item by item in POS table, by entire Sub-department or entire Department. 

All methods only require you to choose the drop down arrow of the table's Restrictor field and choose the appropriate Restriction level.
