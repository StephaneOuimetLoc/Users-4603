eSuite Batches - Future POS Table

eSuite > Items > Batches > Item list

Information

The Future POS table is used to enter the future POS sales information.

Displaying the Future POS table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Item list: the Items - general list wizard opens in a new tab.
  3. Enter the search criteria for the item to add to the future POS batch.
  4. Click on Execute: results will display in the Items - General list window.
  5. Right-click on a record and select POS: the Future POS information table opens on the work area.

Content

Here is the description of the Future POS table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
Not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future POS record is added, linking the future POS record to the current batch.
Target The Target field contains the target identifier, i.e. the store group.
When a future POS record is added, the target value of the current POS record is automatically displayed.
Not editable unless the Multi target option in the System.ini file is selected.
POS description The POS description field contains the future description of the item displayed on the POS screen and/or printed on the receipt.
When a future POS record is added, the POS description of the current POS record is automatically displayed.
Sub-department The Sub-department field links the item to a future sub-department.
When a future POS record is added, the sub-department number of the current POS record is automatically displayed.
Bottle link  The Bottle link field links the item to a future bottle deposit value.
When a future POS record is added, the bottle deposit number of the current POS record is automatically displayed.
Other flags The Other flags field contains future additional information on the item.
When a future POS record is added, the additional information of the current POS record is automatically displayed.
WIC alias  The WIC alias field contains a future alternate code specific to the WIC program.
When a future POS record is added, the WIC alias of the current POS record is automatically displayed.  
Divisor The Divisor field contains the future weight divisor of the item.
When a future POS record is added, the divisor value of the current POS record is automatically displayed.
The following options are available: Ounces, 100 g, Half pound and Quarter pound.
Tare link The Tare link field links the item to a future tare value. The tare value is used to subtract the item's packaging weight from the total weight of the item sold by weight, selling only the true item weight.
When a future POS record is added, the tare link number of the current POS record is automatically displayed in the Tare link field.
Proportional tare The Proportional tare (Prop tare) field contains the future constant factor (in percentage) used to calculate the item tare value at the time of sale.
When a future POS record is added, the proportional tare value of the current POS record is automatically displayed.
Follow sub-department The Follow sub-department (Follow sub) field specifies whether the item will follow the linked sub-department sales information programming (checked), or not (unchecked).
When a future POS record is added, the follow sub-department value of the current POS record is automatically displayed.
FSA eligible The FSA eligible field specifies whether the item could be purchased using a FSA card as tender (checked), or not (unchecked).
When a future POS record is added, the FSA  eligible value of the current POS record is automatically displayed.
FSA products include medical expenses such as clinical services, dental services, prescription / RX only, vision/optical, OTC (over the counter items).
WIC item The WIC item field specifies whether the item will be WIC able (checked), or not (unchecked).
When a future POS record is added, the WIC item value of the current POS record is automatically displayed.
Food stamp The Food stamp field specifies whether the item could be purchased using food stamps as tender (checked), or not (unchecked).
When a future POS record is added, the food stamp value of the current POS record is automatically displayed.
Scalable The Scalable field specifies whether the item will be sold by weight, scaled at the POS (checked), or not (unchecked).
When a future POS record is added, the scalable value of the current POS record is automatically displayed.
The Scalable field does not have to be checked if the item is pre-scaled and coded with the price (code-2 items, also called random weight items), except if the weight sold must be calculated (e.g. on a discount per pound).
Manual The Manual field specifies whether the weight of the scalable item could be manually entered using the POS keyboard (checked), or not (unchecked).
When a future POS record is added, the manual value of the current POS record is automatically displayed.
Taxable 1 The Taxable 1 field specifies whether the item will have the first tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 1 value of the current POS record is automatically displayed.
Taxable 2 The Taxable 2 field specifies whether the item will have the second tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 2 value of the current POS record is automatically displayed.
Taxable 3 The Taxable 3 field specifies whether the item will have the third tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 3 value of the current POS record is automatically displayed.
Taxable 4 The Taxable 4 field specifies whether the item will have the fourth tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 4 value of the current POS record is automatically displayed.
Taxable 5 The Taxable 5 field specifies whether the item will have the fifth tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 5 value of the current POS record is automatically displayed.
Taxable 6 The Taxable 6 field specifies whether the item will have the sixth tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 6 value of the current POS record is automatically displayed.
Taxable 7 The Taxable 7 field specifies whether the item will have the seventh tax for the retail region (checked), or not (unchecked).
When a future POS record is added, the tax 7 value of the current POS record is automatically displayed.
Restriction tab
Restrictor The Restrictor field links the item to a future restriction. For example, in areas where stores are allowed to sell beer, wine or alcohol, their sales might be legally restricted to certain hours of the day.
When a future POS record is added, the restriction number of the current POS record is automatically displayed.
Status The Status field contains the future identification code of the item status.
When a future POS record is added, the status value of the current POS record is automatically displayed.
Customer age The Customer age field contains the future minimum age the purchaser must be to buy the item. For example, in areas where stores are allowed to sell beer, wine or alcohol, purchasers must be of legal adult age (for the store's region) to buy these items.
When a future POS record is added, the customer age of the current POS record is automatically displayed.
Cashier age The Cashier age field contains the future minimum age the cashiers must be to sell the item. For example, in areas where stores are allowed to sell beer, wine or alcohol, cashiers must be of legal adult age (for the store's region) to sell these items.
When a future POS record is added, the cashier age of the current POS record is automatically displayed.
Stop discount The Stop discount field specifies whether discounts could be applied on the item when it will be sold at the POS (unchecked), or not (checked).
When a future POS record is added, the stop discount status of the current POS record is automatically displayed.
Not for sale The Not for sale field specifies whether the item could be sold at the POS (unchecked), or not (checked).
When a future POS record is added, the not for sale value of the current POS record is automatically displayed.
Not in net The Not in net field is a flag field that specifies whether the item sales will be added to the store's net sales (unchecked), or not (checked).
When a future POS record is added, the not in net value of the current POS record is automatically displayed.
The Not in net field is often referred to as "hash sales".
Force price The Force price field specifies whether the item price should be manually entered when the item will be sold at the POS (checked), or not (unchecked).
When a future POS record is added, the force price status of the current POS record is automatically displayed.
Force quantity The Force quantity field specifies whether the item quantity should be entered before the item could be sold at the POS (checked), or not (unchecked).
When a future POS record is added, the force quantity status of the current POS record is automatically displayed in the Force quantity field.
Stop quantity The Stop quantity field specifies whether an item quantity could be entered for the item when it will be sold at the POS (unchecked), or not (checked).
When a future POS record is added, the stop quantity status of the current POS record is automatically displayed.
Visual verify The Visual verify field specifies whether the item price should be verified at the POS before the item could be sold (checked), or not (unchecked).
When a future POS record is added, the visual verification status of the current POS record is automatically displayed.
Price override The Price override field specifies whether the cashiers will be allowed to temporarily change the item price at the POS using the Price modify function (checked), or not (unchecked).
When a future POS record is added, the price override status of the current POS record is automatically displayed.
Stop repeat The Stop repeat field specifies whether cashiers could see the same item more than once using the Repeat last function (unchecked), or not (checked).
When a future POS record is added, the stop repeat status of the current POS record is automatically displayed.
Stop refund The Stop refund field specifies whether the item could be refunded (unchecked), or not (checked).
When a future POS record is added, the stop refund status of the current POS record is automatically displayed.
Coupon tab
Family The Family field contains the future coupon (vendor and store) family code, which specifies on which items the coupons apply. The family code is needed when the scanned coupons give items for free.
When a future POS record is added, the family code value of the current POS record is automatically displayed.
Coupon type The Coupon type field contains the future type of coupon that can be applied to the item.
The following options are available: Manual, Electronic, Targeted, Automatic and Public.
When a future POS record is added, the coupon type of the current POS record is automatically displayed.
Promotion ID The Promotion ID field contains the future promotion identification number. The promotion identification number will associate the item to a promotion.
When a future POS record is added, the promotion identification number of the current POS record is automatically displayed.
Ticket The Ticket field contains the future name of the coupon template to be used instead of the default one.
When a future POS record is added, the ticket value of the current POS record is automatically displayed.
Store coupon The Store coupon field specifies whether the item will be a store coupon (checked), or not (unchecked).
When a future POS record is added, the store coupon status of the current POS record is automatically displayed.
Vendor The Vendor field specifies whether the item will be a vendor coupon (checked), or not (unchecked).
When a future POS record is added, the vendor coupon status of the current POS record is automatically displayed.
Multipliable The Multipliable field specifies whether double coupons will be accepted at the POS for the item (checked), or not (unchecked).
When a future POS record is added, the multipliable status of the current POS record is automatically displayed.
Reject The Reject field specifies whether vendor coupons will be refused at the POS for the item (checked), or not (unchecked).
When a future POS record is added, the reject status of the current POS record is automatically displayed.
Print coupon The Print coupon field specifies whether coupons will be generated for the item (checked), or not (unchecked).
When a future POS record is added, the print status of the current POS record is automatically displayed.
Bottle return The Bottle return field specifies whether the item will be a bottle return item (checked), or not (unchecked).
When a future POS record is added, the bottle return value of the current POS record is automatically displayed.
Rain check The Rain check field specifies whether rain checks could be generated for the item (checked), or not (unchecked).
When a future POS record is added, the rain check status of the current POS record is automatically displayed.
Capture data The Capture data field specifies whether the item sales will be tracked by customer (checked), or not (unchecked).
When a future POS record is added, the capture data value of the current POS record is automatically displayed.
Miscellaneous tab
Alternate description The Alternate description field contains a future alternate description of the item displayed on the POS screen and/or printed on the receipt.
Mainly used for a second language POS description.
When a future POS record is added, the alternate description of the current POS record is automatically displayed.
Creation date The Creation date field contains the creation date of the item.
When a future POS record is added, the creation date of the current POS record is automatically displayed.
Not editable. Used for information purposes only.
Normalized UOM The Normalized unit of measure (UOM) field contains the identification characters of the future buying format available for the item.
When a future POS record is added, the normalized UOM of the current POS record is automatically displayed. 
PLU # The PLU # field contains the future memory number which links the PLU to the scales or the cash registers.
When a future POS record is added, the PLU # of the current POS record is automatically displayed.
Quantity for rent The Quantity for rent (Qty rent) field contains the future quantity of the item available for rent.
When a future POS record is added, the number of the item available for rent of the current POS record is automatically displayed.
Commission The Commission field contains the future salesman's commission rate for the item.
When a future POS record is added, the commission rate of the current POS record is automatically displayed.
Advanced tab
Behavior script type The Behavior script type field contains the future renting behavior for the item.
When a future POS record is added, the renting behavior of the current POS record is automatically displayed.
Add using function The Add using function field contains the future number of the function that must be used to identify the totalizers when the item will be sold instead of the regular one.
When a future POS record is added, the add using function number of the current POS record is automatically displayed.
URL The URL field contains future special parameters to be used when the item is sold.
When a future POS record is added, the URL of the current POS record is automatically displayed.
POS table grid
Target column The Target column displays the list of target identifiers (store groups) for the item.
