eCommerce - Items Management

The Featured products

The Featured products section is used to promote specific items. On the Host server or Store server depending on the selected scenario, the keyword Featured must be defined for each item in order to display them in eStore/mStore applications.

To display items in the Featured products section in the eStore and mStore applications, follow these steps:

  1. Under the Margin tab of the Price table, enter the keyword Featured in the Discount description field.


  2. Deploy changes.

NOTE: Featured products will be available on the eStore/mStore applications after the jobs 270-Generate local mStore promotion and 275-Generate local eStore promotion have been executed on the eCommerce server at the scheduled date and time.

Search filters

Several search filters are available for customers to refine their search and make it easier to find items in the eStore/mStore applications.

Search bar

The first section is the Search bar, when entering the search keyword, the query is executed on the following fields: UPC code, Brand description, Expanded description, Long description, and POS description to displayed the items.

 

Other search filtes

The second section contains multiple types of search filters. Each selection is used to display the items, several selections allows to refine the search.

eStore application mStore application Description

From the eStore application:

The Categories section is separated into 2 levels:

  1. The first level displays all departments and filters items found in the selected department.
  2. The second level displays all categories associated to each items found in the selected department.

From the mStore application:

The Departments section displays all departments and filters items found in the selected department.
 

From the mStore application:

The Categories section displays all categories associated with each items according to the selected Department.
 

From the eStore application:

The Product types section displays all the Classifications (from the Item Classification table) associated with each items according to the selected Categories.

From the mStore application:

The Classifications section displays all the Classifications (from the Item Classification table) associated with each items according to the selected Departments.
   

From the eStore application:

The Brands section displays the brands name, from the Brand field of the Main item table, associated to each items based on the Categories filter selected.

From the mStore application:

The Brands section is not available.


 		  

From the eStore application:

The Filters section contains more generic filters:

  • Featured items filters featured items only.
  • Purchased before filters items already purchased (close and processed transactions only).
  • Purchase with points filters items available for purchase with points.
  • All promotions section filters items based on the active price of the different types of promotions:
    • Weekly sale: filters items with the SALE active price.
    • Long term rebate: filters items with the TPR active price.
    • Manager special: filters items with the STORE active price.

From the mStore application:

The Filters section offers the same filters as eStore. The Featured items filter is available from the menu.
   

From the eStore and mStore applications:

The Preferences section displays all the preferences and filters items according to the keyword(s) predefined in the Classification field of the Main item table.

It is possible for customers to set default preferences in their customer account, then when they access the Shopping tab of the eStore/mStore applications, the items are automatically displayed according to these preferences.

Preferences

The Preferences section on the eStore/mStore applications is used to filter items according to the predefined keyword(s). On the Host server or Store server depending on the selected scenario, the keyword(s) must be defined in the Classification field of the Main item table. Several preferences can be defined using ';' to separate them.

Item modifiers

These item modifiers are especially for the eCommerce solution. When an item, defined with one of these keywords, is added to the shopping cart, the item details automatically open to allow customers to make a choice in additional fields.

A keyword must be defined in specific items, in the Type filed of the POS table of the Host server or the Store server depending on the selected scenario. Only one keyword can be defined.

Keyword Description
CAKE Used for preparation of personalized cakes. Additional fields are related to flavor, size, color, icing and sprinkles as well as adding a message and an image/picture to be placed on the cake.
FRUIT Used for fresh produce. Additional fields are related to specify the ripeness and size of the item.
GIFT Used for item to be send as a gift. Additional field is for entering an email address.
MEATBYQTY Used for selling meat by quantity. Additional fields are related to the quantity of item, marbling, trimming and quantity per package.
MEATBYWEIGHT Used for selling meat by weight. Additional fields are related to the quantity of item, thickness or weight by steaks, marbling, trimming and quantity per package.
MEATDELI Used for selling deli item. Additional fields are related to the total weight, thickness and lbs per package.
MEATGROUND Used for selling meat grounded item. Additional fields are related to the total weight and lbs per package.
MEATOTHER Used for selling other meat item. Additional fields are related to the quantity of item and quantity per package.
MEATROAST Used for selling meat roast item. Additional fields are related to the quantity of item, quantity per package and weight of each roast.
NOTE By default, all items have this modifier, even if the Type field is empty. It is used to add a basic note to an item.
TRAYPARTY Used for selling tray item. Additional fields are related to the serving size and tray suggestion.

Special order

The Special order is used to notify the customer that the selected item may take longer to arrive. The customer is not charge immediately for this item, the quantity will be entered at the store level at the preparation step, and then the customer will be charged.

The Special order (SP) must be defined in certain items, in the Status field of the POS table of the Host server or the Store server depending on the selected scenario.

NOTE: The value defined for the overshoot parameter, in the Profile table, must contain an additional amount related to special orders in order to complete the order at the store level.

Extra Preparation Time

The Extra preparation time is used to notify the customer, when selecting Delivery or Pickup handling method, that the selected item, such as custom cake, needs an extra preparation time.

 The time must be defined in certain items, in the Time included field of the PRICE table of the Host server or the Store server depending on the selected scenario. In the Note section of the shopping cart, the comment 'Your order requires an extra preparation time of x day(s)' (in red) is displayed at the handling method selection to prevent customers from selecting a Delivery/Pickup time prior to the extra preparation time.

NOTE: The the number of days to copy must be increased in the route_copy script according to the Extra preparation time choices, allowing customers to complete their order.
