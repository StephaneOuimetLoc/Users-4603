mFloor - Item Pictures

mFloor Item Management

Information

This document explains how to take pictures with mFloor to maintain the image database corresponding to every items.

Preparing the picture's information

Before taking the picture directly, it is important to create its information in the system. To achieve this, please follow these steps:

  1. Choose the Types of the picture (Main, Nutri or Plano)
  2. Chose the Angle of the picture (Back, Bottom, Front, Left, Main, Right or Top)
  3. Choose the Size of the picture (Thumb or Detail)
  4. Select the Top-Right menu and then Add Image.
    A new line will appear in the list with these new information.
  5. Select this new line and move to the next part: Adding the picture.

When choosing MAIN, MAIN, THUMB, this picture will be the default picture used everywhere (POS, mFloor, etc.). All the others are additional views of the product.

Adding the picture

When ready to add the picture, there are two methods: Choosing a picture already stored on the device or use the Camera to take it on the spot.

The first method will open a file browser and will ask the user to choose a picture from the device. It might have been taken using the device's camera application before using mFloor.

The second method will trigger the Camera directly. Once the picture is taken, it is uploaded to the back office server under the right folder and using the right naming convention.

NOTE: The Camera feature is not available on Windows PC. It's only available on an Android/Apple device using the Chrome browser, the native application or an Enterprise Browser.

Deleting a picture

  • To delete a picture, select the picture in the list then open the top-right menu and press Delete image. A confirmation prompt will show asking the user to confirm the deletion.

Image location

When a picture is added and saved, it will be placed in the following location Storeman/BitMaps/ITEMS/[UPC_Part1]/[UPC_Part2]/[UPC_Part3].

The UPC will be broken up into three parts as seen below and this creates subfolder to avoid placing all the pictures into one single folder.

  • [UPC_Part1] is the first 3 digits of the UPC
  • [UPC_Part2] is the next 5 digits of the UPC
  • [UPC_Part3] is the final 5 digits of the UPC

For example if the item's UPC is 0000000004011 the file path would be Storeman/BitMaps/ITEMS/000/00000/04011

The filename will be as follow [UPC]_[Types]_[Angles]_[Size].jpg, these values refers to the detail selected while creating the image information above.
