eSuite - Future Redeem Table

eSuite >  Items > Batches > Price information

Future redeem table

The Future redeem table is used to change the future redeem information of the selected items.

Displaying the Future redeem table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Price information: the Price batch wizard opens in a new tab.
  3. In the Batch type field, select Redeem.
  4. In the Batch field, select a price redeem batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Price redeem batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Price batch, then Redeem: the Future price redeem table opens over the work area.

Layout

The Future price redeem table is divided in two parts: the main section and the Future price redeem table grid.

Main section

* Click on a field to get information on it.

Future price redeem table grid

* Click on a field to get information on it.

Content

Here is the description of the Future price redeem table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID of the Batch header table and is automatically displayed when a future redeem record is added, linking the future redeem record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future redeem record is added, the target value of the current price record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Level The Level field contains the price level number.
See the Level field section below.
Start date The Start date field contains the starting date for the future preset items points redemption, i.e. the date on which the point discount on the item can be given. The starting date is inclusive.
When a future redeem record is added, the start date of the Batch header table is automatically displayed in the Start date field. The starting date will become the redeem starting date in the corresponding price record.
End date The End date field is a date field that contains the ending date for the future preset items points redemption, i.e. the date aonwhich the point discount to be given on the item ends. The ending date is inclusive.
When a future redeem record is added, the ending date of the Batch header table is automatically displayed in the End date field. The ending date will become the redeem ending date of the corresponding price record.
Discount points The Discount points field contains the future number of points needed for the customer to get a discount on the item.
When a future redemption record is added, the discount points value of the current price record is automatically displayed in the Redeem field.
The Discount points field works in combination with the Discount_amount and Program fields.
Discount amount The Discount amount field contains the future amount of the discount given when the customer redeems points.
When a future redemption record is added, the discount amount of the current price record is automatically displayed in the Discount amount field.
Free points The Free points field contains the future number of points needed for the customer to obtain the item for free.
When a future redemption record is added, the free points value of the current price record is automatically displayed in the Free points field.
Clip points The Clip points field contains the future number of points obtained from clipped offers.
Program The Program field contains the future program that corresponds to the point program from which the points are redeemed.
When a future redemption record is added, the program value of the current price record is automatically displayed in the Program field.
The Program field works in combination with the Discount points, Discount amount and Free points fields.

The Level field

The Level field contains the price level number.

When a future redeem record is added, the level value of the current price record is automatically displayed in the Level field.

NOTE:
Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.
