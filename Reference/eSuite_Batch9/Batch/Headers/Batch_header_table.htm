eSuite - Batch Header Table

eSuite > Items > Batches

Information

The Batch header table is used to create batches. It allows the association of a batch with a unique ID. It contains general information on the batch: type, target, description, start and end dates and so on.

Displaying the Batch header table

To edit an existing batch:

  1. In the launcher, click the arrow in the Items entry to display its menu.
  2. Click on Batches: the Start window opens in the Batches work area.
  3. Select All: the list of in-store batches opens in the Batches window.
  4. Select a batch and click Edit in the bottom menu: the Batch header table opens on the desktop.
    It displays the existing batch information.

To create a batch:

  1. In the Launcher, click the arrow in the Items entry to display its menu.
  2. Click on Batches: the Start window opens in the Batches work area.
  3. Click on New to display the list of batch types.
  4. Make a selection: the Batch header table opens in the work area.
    It displays the default information.


NOTE: You may create batches from the Price grid and Cost grid entries. Click Add in the wizard to access the list of price-related and cost-related batch types.

Layout

The Batch header table displays an additional button in the toolbar menu: Open list . Disabled by default. Enabled when all mandatory fields are filled in and changes have been saved.

The Open list button opens the batch wizard, allowing the access to the list of items for selection and addition to the batch.

* Click on a field to get more information.

Content

Here is the description of the Batch header fields and their functionalities:

FIELD DESCRIPTION
Batch ID The Batch ID contains the batch identifier.
Not editable. Its value is automatically given by the system when a batch header record is added.
Target The Target field contains the store group.
Mandatory field.
Description The Description field contains the description of the batch.
Source The Source field contains the node number of the source of the batch.
Not editable. The value is automatically given by the system when a batch header record is created.
For an in-store batch, LOCAL is displayed instead of the node number.
State The State field contains the status of the batch.
See the State field section.  
Priority The Priority field contains the priority level for the batch.
Start date The Start date field contains the date on which the batch becomes effective.
See the Start date field section.
Time (Start date) The Time (Start date) field contains the time at which the batch becomes effective during the effective day.
If there is no specification on the start date time, the system uses 00:00.
End date The End date field contains the date on which the batch terminates.
See the End date field section.
Time (End date) The Time (End date) field contains the time at which the batch terminates during the ending day.
If there is no specification on the end date time, the system uses 23:59.
Type The Type field contains the name of the file that is called to execute the batch.
See the Type field section.
Count The Count field is user-defined.
It can be used by the user for any purpose necessary.
Action The Action field contains the code of the action to take when the batch is executed.
See the Action field section.
Default label template The Default label template field contains the name of the template to use for the labels.
Long description The Long description field allows entering comments on the batch.

The State field

The State field contains the status of the batch.

The following values are possible:

Value Description
[empty] The batch is open for modifications or adding new items.
Wait The batch will be manually executed.
Closed The batch is closed to modifications or adding new items.

The Start date field

The Start date field contains the date on which the batch becomes effective. It is a required field. The start date is inclusive.

By default, when a batch header record is created, the start date is automatically set by the system to the date of the first day of the next business week (next Sunday's date if the business week starts on Sunday, next Monday's date if the business week starts on Monday, etc.).

The business week is defined in the System.ini file. See the reseller for details.

Back to layout

The End date field

The End date field contains the date at which the batch terminates. The ending date is inclusive.

By default, when a batch header record is added, the ending date is automatically set by the system to the date of the last day of the next business week (the Saturday's date of the next business week if it starts on Sunday, the Sunday's date of the next business week if it starts on Monday, etc.).

The business week is defined in the System.ini. See the reseller for details.

TPR is for the future TPR batches.

Redeem is for the future redeem batches.
Cost is for the future cost batches.
General cost is for the future cost general batches.
Allowance is for the future allowance batches.
Rebate is for the future rebate batches.
Discount is for the future discount batches.
Cost break is for the future cost break batches.
Bill backs is for the future bill back batches.
Main item is for the future main item batches.
Point of sale is for the future point of sale (POS) batches.
Open batch is for the open batches.
Delete batch is for the delete batches (under development).

The Action field

The Action field contains the code of the action to take when the batch is executed.

The following values are available:

Action Description
Change New records are not added; changes are done on existing records only.
Add New records are added, but the changes are not done on existing records.
Add/Replace New records are added, and the changes are done on existing records.
Wait The batch will not be executed right away.
