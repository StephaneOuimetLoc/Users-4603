eSuite - Future Item Table

eSuite > Items > Batches > Item list

Information

The Future item table is used to enter the future general item information.

Displaying the Future item table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Item list: the Items - general list wizard opens in a new tab.
  3. Enter the search criteria for the item to add to the future main batch.
  4. Click on Execute: results will display in the Items - General list window.
  5. Right-click on a record and select Main batch: the Future item edit table opens on the work area.

Content

Here is the description of the Future item table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the identifier of the main item batch. Not editable.
Its value comes from the Batch ID field of the Batch header table. It links the future main item record to the current batch.
Description The Description field contains the future item description.
When a future main item record is added, the description of the current main item record is automatically displayed.
Brand The Brand field contains the name of the item manufacturer.
When a future main item record is added, the brand of the current main item record is automatically displayed.
Size The Size field contains the size of the future item.
When a future main item record is added, the size of the current main item record is automatically displayed.
Alternate brand The Alternate brand field contains an alternate name for the item manufacturer.
Mainly used for a second language.
When a future main item record is added, the alternate brand of the current main item record is automatically displayed.
Alternate description The Alternate description field contains additional details about the item.
Mainly used for a second language.
When a future main item record is added, the alternate description of the current main item record is automatically displayed.
Alternate size The Alternate size field contains an alternate size for the future item.
Mainly used for a second language.
When a future main item record is added, the alternate size of the current main item record is automatically displayed.
Category The Category field links the item to a future category. 
When a future main item record is added, the category of the current main item record is automatically displayed.
Family The Family field links the item to a future family group. 
When a future main item record is added, the family number and description of the current main item record are automatically displayed.
Report The Report field links the item to a future report group. 
When a future main item record is added, the report number and description of the current main item record are automatically displayed.
NACS The NACS field links the item to a future NACS code.
NACS (National Association of Convenience Stores) codes are used by gas stations to identify what is being charged to a fleet credit card.
Account The Account field contains the future general ledger account code associated to the item. 
When a future main item record is added, the general ledger account number of the current main item record is automatically displayed.
Section The Section field contains the future section grouping.
When a future main item record is added, the section code of the current main item record is automatically displayed.
Alias code The Alias code field contains afuture item UPC code used to group items. 
When a future main item record is added, the alias UPC code of the current main item record is automatically displayed.
Sell pack The Sell pack field contains the future number of units included in the item.
When a future main item record is added, the sell pack value of the current main item record is automatically displayed.
Volume/Weight The Volume/Weight field contains the future unit volume or unit weight of the item.
When a future main item record is added, the volume or weight value of the current main item record is automatically displayed.
Measure The Measure field contains the future unit of measure description.
When a future main item record is added, the measure value of the current main item record is automatically displayed.
Shelf tab
Graphic file The Graphic file field contains the future file name of the graphic that you want on the label.
When a future main item record is added, the graphic file name of the current main item record is automatically displayed.
Container The Container field contains the future identification number of the packaging format, used to determine the shape of the item.
When a future main item record is added, the container value of the current main item record is automatically displayed.
Width The Width field contains the future metric width value of the item, in millimeters.
When a future main item record is added, the width value of the current main item record is automatically displayed.
Height The Height field contains the future metric height value of the item, in millimeters.
When a future main item record is added, the height value of the current main item record is automatically displayed.
Depth The Depth field contains the future metric depth value of the item, in millimeters.
When a future main item record is added, the depth value of the current main item record is automatically displayed.
Long description The Long description field contains the future additional details about the item.
When a future main item record is added, the long description of the current main item record is automatically displayed.
Pharmacy tab
Medical code The Medical code field contains the future drug code of the item. Used for pharmaceutical items. It is given by the store.
When a future main item record is added, the medical code of the current main item record is automatically displayed.
NDC/DIN The NDC/DIN field contains the future standardized drug code of the item, given to pharmaceutical items by governmental institutions.
When a future main item record is added, the standardized drug code of the current main item record is automatically displayed.
Manufacturer code The Manufacturer code field contains the future manufacture code (for example, the 5 manufacturer's digits in the UPC code) used to group items.
When a future main item record is added, the manufacturer code of the current main item record is automatically displayed.
Competitive code The Competitive code field contains the future UPC of an item that is considered as a competitive item to the current item and could replace it.
Used for pharmaceutical items.
When a future main item record is added, the competitive code of the current main item record is automatically displayed.
Handling type The Handling type field contains the way the item must be handled and/or preserved.
When a future main item record is added, the handling method of the current main item record is automatically displayed.
Classification The Classification field contains the future keywords which are used to find items in the system.
When a future main item record is added, the classification of the current main item record is automatically displayed.
Owner The Owner field contains the store number of the store or host that has possession of the record. The other stores cannot modify the record unless they retake possession of the record.
The Owner field is not editable and used for information purposes only.
See the reseller for more details on how a store may retake possession of the records (System.ini file).
NuVal The NuVal field contains information on the future specific properties of the item (gluten free, organic, low sodium, etc.).
Marketing tab
Target customer The Target customer field contains the future target age group of the customers who use the item.
When a future main item record is added, the target age group of the current main item record is automatically displayed.
Target store The Target store field contains the future type of store in which the item is sold.
When a future main item record is added, the type of store of the current main item record is automatically displayed.
Marketing justification for this item The Marketing justification for this item field contains a future internal note on the item. This note can be used, for example, to enter more information on the item, to explain why the item is sold, etc.
When a future main item record is added, the internal note of the current main item record is automatically displayed.
Miscellaneous tab
Inter-department code The Inter-department code field contains the future code used for the inter-department transfers, i.e. to transpose the inventory when the original item code is sold.
When a future main item record is added, the inter-department code of the current main item record is automatically displayed.
Weight The Weight field contains the future net weight of the item. It is usually used in warehouses for shipping information.
When a future main item record is added, the weight value of the current main item record is automatically displayed.
Cubic The Cubic field contains the future metric volume of the item, in meters. It is usually used in warehouses for shipping information.
When a future main item record is added, the cubic value of the current main item record is automatically displayed.
Ship box The Ship box field contains the future number of boxes needed to ship the item. It is usually used in warehouses for shipping information.
When a future main item record is added, the ship box number of the current main item record is automatically displayed.
Operator owner The Operator owner field contains the future number of the operator who is responsible with the purchases of the item.
When a future main item record is added, the operator owner of the current main item record is automatically displayed.
Level The Level field contains the number of the minimum operator level needed to manage the current main item record.
When a future main item record is added, the level of the current main item record is automatically displayed.
Table grid
Batch column  The Batch column contains the identifiers of all main item batches the item belongs to.
Not editable.
