Global Changes

Global change is used for applying changes made to a single record to all the records selected in a list of items or batches (the global change function for batches is currently under development).


The following fields may not be changed with the Global change utility:


  • UPC code
  • Vendor ID
  • Price level
  • Batch ID
  • Target
  • Record status
  • Active price
  • Active price quantity
  • Active price changed date
  • Autorun descriptor
  • Buying format


The Global Change utility is available in the Context Tools window when Master and Batches are selected under Items in the Launcher:



The entry becomes expandable once records have been changed. It lists the names of the item tables impacted by the changes and gives access to the Apply global change window:



Displaying the Apply global change window


Click Apply global change on the right of any of the item-related tables: the Apply global change (Main item/POS/Price/Cost) window opens on the desktop.


NOTE: To globally change the value of a field in the Cost table, all items displayed in the item list must have the same vendor.

Layout

The Apply global change window is divided into several parts visible depending on the type of change applied on the item:

  • The main section, common to all Apply global change type of windows (Main item, POS, Price and Cost)
  • The Target section, common to Apply global change (POS, Price and Cost)
  • The Price level section, available in Apply global change (Price)
  • The button section, common to all Apply global change type of windows (Main item, POS, Price and Cost)


* Click on a field to get information on it


Here is the description of the Apply global change window fields and their functionalities:


FIELD DESCRIPTION 
Main section
Selection The Selection field contains the list where the changes are applied.
The default value is Items - General list.
Show changes only Show changes only allows displaying the fields and values that have been changed in the record (checked) or all fields available for change (unchecked).
For details, see the Show changes only field section below.

Checkboxes  The checkboxes in the main section allow selecting the fields to which global change will apply. 
Field The Field column contains the name of the fields affected by global change.
Before The Before column contains the old value of the respective field. 
After The After column contains the new value of the respective field.  
Changed The Changed column displays the fields that are subject to global change.
Checked and grayed out by default.  
Target section 
Filter The Filter field allows entering target IDs and refreshing the target list.
Checkboxes  The checkboxes in the target section allow selecting the targets to which global change will apply.  
Target The Target column contains the target IDs.

NOTE:Starting v350x,
Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.


The Show changes only field


Show changes only allows displaying the fields and values that have been changed in the record (checked).


When unchecked, it displays all fields in the respective item table that are available for global change. It allows the selection of any of these fields for the purpose of global change.


Example


Item 0000000002022 has changed the category. By default, the Apply global change (Main item) window will display the changes only.


When the Show changes only field is unchecked, all fields in the Main item table that are available for global change become visible and may be selected:



  That means that the values in the respective fields will globally apply to the records (3) that have been selected in the item list.

The Create as needed field


The Create as needed is used to apply the values that have been changed in the record and create a new record if it does not exist for an item.


Example


Item 0000000000021 has a single price level record, the Regular (1) one. This price level will change from $1.75 to $1.99.


The new value will also apply to the Super saver (5) price level that will be created for the item by checking the Create as needed field in the Apply global change (Price) window.


All these changes will globally apply to several other items as well. All these items will have two price level records, i.e. the regular (1) and the super saver (5) set to $1.99.

Applying global change after record modified in the POS table


In the following example, all Gatorade energy drinks are recalled.
The Not for sale case will be checked in the POS table and global change will be applied to all Gatorade energy drinks selected in the item grid.


  1. In the Launcher, click the arrow in the Items entry to display its menu.
  2. Click on Master: the Start window opens in the Items work area.
  3. Click on Item list: the Wizard tab opens in the same window, displaying the Items - General list.
  4. Enter the search criteria for the Gatorade items that are recalled and click Execute: results display in Items - General list.
  5. Select one of the Gatorade items.
  6. Click Edit in the bottom menu, then select Master tables and POS master.
  7. In the POS table, check the Not for sale box:
  8. Save the changes and close the POS table.
  9. In Items - General list, select all the Gatorade items the global change will apply to:
  10. In Context tools, click + next to Global change to expand the panel:
  11. Click next to POS: the Apply global change (Main item) window opens, displaying the before and after values of the modified field(s).
    The Apply [x records] button indicates the number of records to which the global change will apply.
  12. Click Apply [12 records].
    The following confirmation message is displayed:
  13. Click OK.
    The Apply global change (POS) window remains available and displays the "before" and "after" values until further changes are made and saved in the item grid or master tables. In this case, the Apply [records] button is disabled and indicates "0 records":
