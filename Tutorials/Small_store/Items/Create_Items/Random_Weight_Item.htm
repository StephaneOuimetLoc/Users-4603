Random weight items

Scaled coded random weight items are identified by:

  1. the digit 2 at the beginning of the UPC, this tells the scanner there could be a price embedded bar code.
  2. followed by the 5 digit product code ie: 12345, if it is not 5 digits it will be padded with leading zeros, ie: 01234
  3. followed by 5 zeros, this is where the price information is embedded in the package sticker

In eSuite, the Item creation table provides a tool that generates the random weight scale codes when creating the new item,  the forward slash on the numerical keyboard. For example, by typing 12345 in the Item number field and hitting / on the numerical keyboard, the code will automatically change for 0021234500000.

Note:  How your scale is programmed for labeling will have an impact on your point of sale.  If a lead digit in your scale is programmed to start with 02 the label can present a 5 digit price, if it starts with 2, the price is limited to 4 digits.   Please see your reseller to ensure eSuite is configured accordingly.

 

To enter a Random Weight item

From Items in the launcher menu select New in the Start wizard

By default item creation table opens, waiting for data input in Item Number field in the General Information table.

1. Enter your 5 digit  Scale code followed by /

2. If it is not found, you will be asked to confirm the creation of a new item:

2. Click yes, and you'll see that remainder of the 4 main information areas, General, POS, Price & Cost, have become available for data entry

Note: If the number entered is already in the database,  the table will populate with the details for that product.

3. Enter the item details in the General, POS, Cost and Price tables, paying particular attention to the FLAGS section of POS Information table.

4. The POS information table is where we would ensure the system knows that the item is to be sold by the weight and there are 2 ways to inform the point of sale that this is the case.

a) If the entire sub-department is all sold by the weight, and setup as such, you can leave the default Flag of Follow sub:, checked off, as demonstrated below.

b) If your sub-department has a mix of each and weight items, we must handle this setting item by item directly in the POS table.  Simply uncheck Follow sub: and check off scalable as shown.

4. To save, click the  in the tool bar of the Item Creation Table.

5. Once you have saved your item you'll see the information tables become available to fully expand, for example, clicking on the POS edit icon will provide access to additional fields if required.

6. When you have entered all the information required to manage your item, click the x to close the item creation table

Information:  If you are managing perpetual inventory by the weight for your scale coded items, the way the item weight is tracked, is the system takes embedded price on the label, divides it by the price per kg in the price table, and calculates the weight to be removed from inventory.  For this reason, the price per kg that the scale has programmed should be the price indicated in the price table, across all shopper levels.


To review your entry

1. From the Start wizard, choose the Items button instead of New 

2. Enter your 5 digit item code followed by /, into the Any code search field, and execute, be sure to check Show POS, Price and Cost if you wish to see those columns in the grid
