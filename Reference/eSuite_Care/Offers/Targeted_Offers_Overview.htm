eSuite Care - Targeted Offers Overview

eSuite Care > Offers > Offer Items

Information

Targeted offers are store and vendor coupons given by the system once the coupon criteria are reached.

When a loyalty card is entered at the POS, the targeted offers available for the customer are loaded into memory. With each item that is scanned, the system validates the applicable offers and adds them to the transaction. At the end of the sale, it updates the server with the targeted offers that have been used.

Offers in eSuite may be assigned to specific customers. They may also be available to anyone that clips them from the store website.

The following types of offers are available to specific customers:

  1. Targeted coupons, programmed through the coupon wizard utility and assigned to a single customer or to multiple customers using the Link offer table.
  2. GS1 Databar coupons, generated through the Create GS1 coupon table following the GS1 Databar coupon barcode rules.
    Currently under development in eSuite.
The offers available to everyone can only be targeted coupons working with the eStore and mStore options. As soon as a targeted coupon is created, it becomes available on the store website. Customers have to go online and clip the offers they would like to have attached to their account.

Targeted offers require the Bank_SMSPayment module.

Displaying the Link offer table

  1. In the Launcher, click on Offers: the Offers start window opens in the Offer tab.
  2. Click on Targeted offers: the Wizard tab opens in the same window, displaying the Targeted offers wizard.
  3. Enter the search criteria and click on Execute: results will display in a record grid, in the Offers  tab of the same window.
  4. Select a record and click Action, then Assign offer in the bottom menu: the Link offer table opens, overlapping the work area.

Content

Here is the description of the Link offer table fields and their functionalities:

FIELD DESCRIPTION
Offer code The Offer code field contains the UPC of the targeted coupon selected in the list of offers.
Type The Type field contains the type of offer.
The default value is ITM for 'item'.
Description The Description field contains the description of the offer.
Customer code The Customer code field contains the code of the customer to whom the offer should apply.
Reload Description to come
Offer quantity The Offer quantity field contains the number of times the customer can get the offer.
The default value is 1.
Clip needed The Clip needed field indicates if the offer should be clipped (checked) or not (unchecked) for the customer to get the offer.
Long description The Long description field contains the long description of the offer.
Create GS1 coupon The Create GS1 coupon button opens the table for the creation of a GS1 coupon.
Currently under development in eSuite.
Global The Global button is used to create an offer that is assigned to multiple customers.
Save The Save button finalizes the creation of the offer assigned to one specific customer.
