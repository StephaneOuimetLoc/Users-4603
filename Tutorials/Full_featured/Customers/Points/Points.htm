eSuite Care - Preferred Shopper Points

eSuite Care > Points

Information

The eSuite software provides different methods to give points directly from the points of sale on up to 35 point programs.

The points may be used (redeemed) to get discounts on items or items for free, as counters to the nth item for free, to get special discounts when a certain amount of purchases has been reached by the customer, etc.

Point programs

The system can manage up to 35 point programs. They are identified by a digit from 1 to 9 or by a letter from A to Z. A corresponds to point program 10, B corresponds to point program 11 and so on up to Z that corresponds to point program 35. Point program 1 is considered the main point program, all the others being secondary ones. They all work independently, according to their own methods, with no interaction between each other.

In the Shopper level table, the Point programs field is used to identify the point programs that the customers linked to the shopper level are members of. The field allows entering or selecting multiple point programs, with no space in between.

The way the points are redeemed for the main point program 1 is defined in the Shopper level table and may be different for each shopper level. The way the points are redeemed for the secondary point programs (2 to Z) are described in the System.ini file or directly on the items, since it is identical for every shopper level.

NOTE: In the Shopper level table, if the Point programs field is blank and a point method is defined for point program 1, the system considers that the customers who are linked to that shopper level are members of point program 1. This is due to the fact that when an upgrade is done from an old version, the main point program must continue to work.

Point methods

Points can be given according to four different point methods that can be combined:

  • Points on preset items
  • Points on entry amounts
  • Points on tenders
  • Points on the sub-total

For the main point program

The Actual method field of the Shopper level table is used to define the point method for the main point program (1).

For the secondary point programs

For the secondary point programs (point program 2 to Z), the system uses point method A (Preset item alone), unless a point method has been defined in the System.ini file under the Points section. When a point method is specified with a rate, the points are given on all items.

Here are the parameters to define:

Method Code Definition

POINTx_METHOD=

[character identifying the point method]

where x is the character that identifies the point program.
See the Actual method field topic for the list of the point methods available and their identifying characters.

POINTx_RATE=

[rate]

where x is the character that identifies the point program.
This parameter is needed with the Entry amount and Sub-total point methods. The rate corresponds to the number of pennies needed to get 1 point.

The points can also be given directly on the items, using the URL field of the POS table. Use the POINTx= parameter, where x is the character that identifies the point program. If no point program is specified, the POINT parameter applies to all point programs.

Here are its different values:

Value Definition

POINTx=N

No points are given on this item for the specified point program x.

POINTx=y

Points are given on this item for the specified point program x based on the y rate.

POINTx=Fz

A fixed number of points (z) are given on this item for the specified point program x.

NOTE: By default, the points given according to the Entry amount point method are given on the gross amounts. This setting can be changed for the net amounts. The points given according to the Tender alone and the Sub-total alone methods using a rate are given based on totalizer 3 (Net sales). This totalizer can be changed. See your dealer for details.

Fixed number of points using the Preset item point method

By default, when a number of points is defined in the Points field of the Price table (using the Preset item point method), the points are given on all relevant point programs indicated in the Shopper level table.

To give these points on a specific point program, this needs to be identified in the Method field of the Price table (P1 to PZ). More than one point program may be selected.

Restrict points

There are different ways of excluding items from points, each of them depending on the point method.

The most usual one is by using the Exclude from spending field (Exc spend) in the Price table. It works with the Entry amount and the Sub-total point methods. When it is checked for an item, this means that no points are given for this item for any of the point programs. However, since the Sub-total method calculates the points based on the amount of totalizer 3 (Net sales) by default, when excluding items with the Exclude from spending field, change this reference for totalizer 3415 (Admissible spending). This is done in the System.ini file, with the parameter CalculatePointTlz under the [CLIENT] section.

NOTE: To exclude all items of a sub-department, check the Exclude from spending field of the Price table for all items that are part of this sub-department and the Exclude from spending (Exc. spending) field of the Sub-department table. This field only excludes the direct sales performed in the sub-department.

To exclude points on some items for a point program or to only give points on some items for a point program when the point method is Entry amount, use the parameter POINTx= in the URL field of the POS table for each desired item, where x is the character identifying the desired point program. Parameter POINTx=N means that no points are given for the specified program. Parameter POINTx=[rate] or F[number of points] means that points are given according to the specified rate or fixed number of points for the specified program. If x is not specified, this applies to all point programs.

Based on the Tender only method, the amount paid with the tender is used to calculate the number of points given according to the defined rate. To use a different value, the POINTTLZ parameter can be set to specify the totalizer to use. For example, if items are excluded using the Exc spend field, the parameter POINTTLZ can be set to 3415 (Admissible spending).

The number of points given can be restricted by a minimum sale condition. If the customer does not buy for a minimum amount in the transaction, the points are not given. This restriction works only with the Sub-total point method on point program 1.

To set up a minimum sale condition, enter the minimum amount the customers must buy to obtain the points in the Minimum sale field of the Shopper level table. The amount must be a numeric value and must contain a decimal point as the separator between dollars and cents.

Increase points

The number of points given on point program 1 can be multiplied by a factor. For example, the points given can be doubled on any purchase for a special promotion. The multiplied factor is working with all four point methods on the point program 1 only.

To multiply the points given, enter the value of the multiplier factor in the Multiplier (points) field of the Shopper level table.

Increase points on specific items only

The Multiplier (points) field in the Shopper level table is used for multiplying all points given. The system does not have the possibility of multiplying points on specific items only. However, depending on the point method used, it may be possible to get a similar result.

If points are given based on the Preset item point method, the Points field of the Price table is used for determining the number of points to give. This value can be increased for specific items.

If points are given based on a rate with the Entry amount method, the POINTx parameter can be set in the URL field of the POS table to define a different rate for specific items.

If points are given based on the Sub-total method, additional points can be given using the Points field or the POINTx parameter.

Example

When the points are given based on the Sub-total method, the only thing that needs to be done to double the points on specific items is to set the value POINT1=100 in the URL field of the POS table for these specific items. This way, the points will be doubled on these items, since the points are already given based on the sub-total.

Let us say that an item is sold at $10.00 and another one at $5.00. Based on the Sub-total point method with a rate of 100 (1 point per $1.00), the customer will get 15 points for this transaction (10+5). To double the points on the item sold at $10.00, the POINT1=100 parameter is added in the URL field for this item. When this item is sold, 10 points are given at the moment it is sold and 15 points are given for the total of the transaction. At the end, a total of 25 points are given in this transaction, where points are doubled only for the item sold at $10.00.

Based on net or gross amount

By default, the points are awarded based on the gross amount (before discounts).

It is possible to modify the configuration of the system so that the points are awarded based on the net amount (after discounts). See your reseller for changes in the configuration.

NOTE: The net amount feature does not apply to the Tender alone and the Sub-total alone methods.
For items that are programmed to award a fixed number of points using the Preset item alone method, the points are not awarded if the item is sold at $0.00 based on net amount.

Point rounding

When the points are given according to a rate, it is possible to set how to round the number of points:

Rounding method Definition
Truncate = Always eliminates any decimal.
Round = Accounting rounding.
RoundUp = Always rounds up.
.5 down = Rounds down to the last 0.5 value.
.5 up = Rounds up to the next 0.5 value.

How to give points

The points are automatically given by the system according to the point method that is used, the restrictions and the configuration settings.

The points given are added to the point balance of the customer entered for the transaction. The system no longer needs a customer to be entered in the transaction to give points. In this case, the points given must be redeemed in the same transaction since they cannot be added to a customer's point balance. The unused points are lost.

The same thing occurs when a customer is entered in the transaction, except that the unused points are added to the point balance of the customer, or more points may be redeemed if the customer already has points in their point balance.

If the customer is entered at the end of the transaction, the system will automatically reset the points given for all items, sub-department sales and department sales. This does not apply to points given using a function (with the POINT parameter). In this case, the cashier needs to void the operation and perform it again. So the best way to proceed when points are given using a function is to enter the customer before the function is used.

NOTE: To give points without entering a customer in the transaction, point programs must be defined in the shopper level 1, which is the default shopper level.

Receipt

On the receipt, the points given and the point balances are automatically printed for all point programs the customer is member of.

Reports

The eSuite software has some reports related to points, like the point balance at a certain date, point movement, etc. These reports are related to the main point program (1), except for the point balance that shows the current point balance of all point programs at the specified date.

NOTE: Since the secondary point programs are generally used to count items to give the nth item for free or for special promotions like the Pampers program, there is no need for a statement on the point movement for these point programs.

Promotions related to points

Different promotions can be given according to the point balances of the customers. For example, the shopper level can be upgraded based on a customer's point balance, allowing special discounts. Electronic coupons can also be given according to the accumulated points. See the Preferred shopper - Shopper level upgrade and Electronic coupons - Examples topics for details on these promotions.

Notes

The points owned at the beginning of the transaction, the points given and the ones redeemed during the current transaction and the point balance at the end of the transaction are all printed on the receipt for all point programs. These values are all displayed in the Details window of the POS, but only the points owned at the beginning of the transaction for the main point program are displayed in the Header window of the POS. The total number of points given in the transaction can be displayed in the Total window. See your reseller for details.

For the point balance to be displayed in the Header window of the cashier screen (POS), a point program and a point method have to be set up in the Shopper level table. When working only with the Points adjustment functionality, we suggest that you use the Preset item alone point method to avoid giving unwanted points unless a value is set in the Points field of the Price table.

For details on points, see the Preferred shopper - Points - Examples and Preferred shopper - Point redemption topics.
