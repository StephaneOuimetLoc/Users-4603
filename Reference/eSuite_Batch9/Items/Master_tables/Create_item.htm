eSuite - How to Create Items

eSuite > Items > Master > Item list (New)

Information

The eSuite software allows creating items through the Item creation table, which includes the main information in the four master tables: Main item, POS, Price and Cost.

Once the new items have been created, changes in the database have to be deployed. The new information will be sent to the front-end workstations and the cash registers on the network.

Displaying the Item creation table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens on the Items desktop.
  2. Click on New.

     The Item Creation table opens in the work area.
    By default, the only enabled field is Item number. The other fields become available when the new number is entered.

Layout

The Item Creation table consists of four sections: General information, POS information, Price information and Cost information.

By default, when the Item creation table opens, all the fields are disabled except the UPC field. They become enabled once the new item UPC is entered.

Content

The table below briefly explains the fields in the Item Creation table and their functionalities.

FIELD DESCRIPTION
General information
Item number The Item number field contains the item code (UPC).
It is a required field used to uniquely identify records in the list of items.
Expanded description The Expanded description field contains the main description of the item.
Brand The Brand field contains the name of the item manufacturer.
Category The Category field allows displaying the list of categories and linking the item to the selected category.
It contains the category number and its description.
For details on categories in eSuite, see the Category table.
Size The Size field contains the item size (e.g., 125 ml).
POS information
Target The Target field contains the target identifier (the store group).
The default value is PAL.
The Target field is grayed out unless the system is set up for multi targets.
Description The Description field in the POS information section contains the short description used at the points of sale.
Sub-department The Sub-department field links the item to a sub-department.
For details, see the Sub-department field in the POS table.
Bottle link The Bottle link field links the item to a bottle link that contains a deposit value. The bottle links are displayed in the drop-down menu.
Records must already exist in the Bottle link table for the drop-down list to display choices.
Tare The Tare field links the item to a tare value that is used to subtract the item packaging weight from the total weight.
For details on tare links in eSuite, see the Tare table.
Follow sub The Follow sub-department field specifies whether the item follows the linked sub-department sales information programming (checked) or not (unchecked).
Selected by default.
For details, see the Follow sub-department field in the POS table.
Scalable The Scalable field specifies whether the item is sold by weight and scaled at the POS (checked) or sold by price (unchecked).
Food stamp The Food stamp field specifies whether the item can be purchased using food stamps as tender (checked) or not (unchecked).
For this field to work, the item must not be programmed to follow the sub-department settings.
The Food stamp flag may be specified for a whole sub-department or department keys.
WIC item The WIC item field specifies whether the item is WIC able (checked) or not (unchecked).
For this field to work, the item must not be programmed to follow the sub-department settings.
The WIC item flag may be specified for a whole sub-department or department keys.
Force price The Force price field specifies whether the item price must be manually entered when the item is sold at the POS (checked) or not (unchecked).
For details, see the Force price field section in the POS table.
Force quantity The Force quantity field specifies whether the item quantity must be entered before the item can be sold at the POS (checked) or not (unchecked).
For details, see the Force quantity field in then POS table.
Stop discount The Stop discount field specifies whether discounts may be applied on the item when it is sold at the POS (unchecked) or not (checked).
Taxable 1 The Taxable 1 field specifies whether the item has the first tax for the retail region (checked) or not (unchecked).
Taxable 2 The Taxable 2 field specifies whether the item has the second tax for the retail region (checked) or not (unchecked).
Taxable 3 The Taxable 3 field specifies whether the item has the third tax for the retail region (checked) or not (unchecked).
Taxable 4 The Taxable 4 field specifies whether the item has the fourth tax for the retail region (checked) or not (unchecked).
Price information
Target The Target field contains the target identifier (the store group).
The default value is PAL.
For details, see the Target field in the Price table.
Level The Level field contains the price level number.
The default value is 1.
For details, see the Level field in the Price table.
Retail The Retail field contains the retail price and the retail quantity for the item.
For details, see the Price field in the Price table.
Price package The Price package field contains the retail package price and quantity for the item. 
For details, see the Package field in the Price table.
Mix & Match The Mix & Match field links the item to a mix & match, which is used to mix items in order to match them to a price.
For details, see the Mix field in the Price table.
Cost information
Target The Target field contains the target identifier (the store group). 
The default value is PAL.
The Target field is grayed out unless the system is set up for multi targets.
Authorized The Authorized field specifies whether the current cost record is the official authorized cost (checked) or not (unchecked).
It is useful in multi-vendors cases, when an item has more than one cost record. The cost record officially authorized is the one used in the reports.
Vendor The Vendor field links the current item cost record to a vendor. 
For details, see the Vendor ID field in the Price table.
Format The Format field contains the buying format.
The default value is CASE.
Base The Base field contains the regular item cost per case.
Vendor code The Vendor code field contains the code used by the vendor to identify the item. 
It is generally used to order the item from the vendor.
Case The Case field contains the number of units per case for the item.

Creating an item

  1. Open the Item creation table.
    By default, the only enabled field is Item number.
  2. Type a new item code and press Enter on the keyboard.
  3. If the UPC already exists in the database, the Item Creation table displays the information of the corresponding item.
    If the UPC does not exist in the database, the following confirmation dialog is displayed:
  4. Click Yes to confirm.
    All fields are enabled for editing.
  5. Enter the item information.
  6. Save the changes and close the table.
  7. Refresh the item grid to display the new item.
  8. Deploy changes.

Item information may be further edited from the master tables.

Random weight codes

Random weight codes are identified by the digit 2 at the beginning of the UPC. They contain the product number on 5 digits and the price of the item encoded in the last 5 digits.

In eSuite, the Item creation table provides a tool that generates the random weight codes when creating the new item: the forward slash on the numerical keyboard. For example, by typing 12345 in the Item number field and hitting / on the numerical keyboard, the code will automatically change for 0021234500000.

Freeze current entry button

Currently under development.

The Freeze current entry button is used to freeze a part of the main code (UPC) in the Item number field.

It can be used for creating items from the same manufacturer. Since the first digits of the UPC are the same, they can be frozen and only the last digits will need to be entered for the new items.

To unfreeze the digits, click again on the Freeze current entry button.
