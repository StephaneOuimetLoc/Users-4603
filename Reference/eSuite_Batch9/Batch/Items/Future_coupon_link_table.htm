eSuite - Future Coupon Link Table

eSuite > Items > Batches

Information

The Future coupon link table is used to enter the future criteria of the electronic coupons. It contains the future coupon link information.

Displaying the Future coupon link table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batch work area.
  2. Click on Price grid: the Wizard opens in a new tab.
  3. Select a batch in the Batch field and enter all other necessary criteria. Press Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select a record and click right or press Edit on the bottom of the item grid.
  5. Select Price batch in the menu, then Future coupon link: the Future coupon link edit table opens overlapping the work area.

Content

Here is the description of the Future coupon link table fields and their functionalities:

FIELD DESCRIPTION
Type The Type field contains the abbreviation of the type of the future criterion which needs to be respected in order to get the future electronic coupon.
See the Type field of the Electronic coupon link table for a complete description of each available type.
Code The Code field contains the code associated to the selected type for the future criterion of the future electronic coupon.
A different code is associated to each type. See the Code field of the Electronic coupon link table for a complete description of each association.
The Cumulative (Cum.) field contains the cumulative identifier, used to specify that the minimum quantity, amount or weight required to get the future electronic coupon comes from a transaction or a cumulative of many transactions. 
Its default value is T (for transaction). See the Cumulative field of the Electronic coupon link table for the list of all cumulative identifiers.
NOTE: Be careful when you set the cumulative field to Daily, Weekly, Monthly and Grand total: since the POS needs to send a request to the backoffice database to get the movement for the specified items, this can slow down the tendering process. The system automatically cumulates the number of items purchased by each customer, when the cumulative identifier is different than Transaction (T). This means that the Capture data field in the Future POS table for these items does not need to checked.
Totalizer The Totalizer (Tlz) field contains the totalizer number used to target the future electronic coupon's criteria. Its default value is 3 (Net sales).
Specific totalizers may be used according to the type selected.
See the Totalizer field of the Electronic coupon link table for a complete description.
Quantity The Quantity field contains the minimum quantity required to reach the future criterion and get the future electronic coupon.
Depending on the type used, the Quantity field has different meanings. See the Quantity field of the Electronic coupon link table for a complete description.
Amount The Amount field contains the minimum amount required to reach the future criterion and get the future electronic coupon.
Depending on the type used, the Amount field has different meanings. See the Amount field of the Electronic coupon link table for a complete description.
Weight The Weight field contains the minimum weight required to reach the future criterion and get the future electronic coupon.
Depending on the type used, the Weight field has different meanings. See the Weight field of the Electronic coupon link table for a complete description.
Batch number The Batch number field contains the batch identifier.
Not editable.
Target The Target field contains the target identifier, i.e. the store group. Its default value is PAL.
When a future coupon link record is added, the target value of the current coupon link record is automatically displayed in the Target field.
Not editable unless the Multi target option in the System.ini file is selected.
Descriptor The Descriptor field contains the future description of the electronic coupon.
By default, when an ITM type is added, the description of the selected item is automatically copied in the Description field. Description or Descriptor?
