eSuite - Automatic Coupon Programming

eSuite > Items > Item auxiliaries

Information

Automatic coupons are store and vendor coupons given by the system when the specified criteria are met. Like any item in the system, the automatic coupons have their own UPCs; they are considered negative items.

By default, the automatic store coupons are linked directly to the items and they affect the net sales.
By default, the automatic vendor coupons are added as new registrations in the transactions and they do not affect the net sales.

Automatic coupons may be programmed in the classical way, namely by entering their POS and price information and defining the criteria by means of the Electronic coupon link table.

However, eSuite offers the Coupon wizard that facilitates the creation of coupons by grouping all criteria (type, method, quantity, amount, etc.) that must be respected for the coupon to be applied. Once the coupon information is entered and saved, the coupon wizard assigns it a unique UPC that starts with 096 followed by the promotion number, which stands on 10 digits.

For the automatic coupons to be issued, the Apply coupon link field of the Shopper level table must be checked in the corresponding shopper levels. The automatic coupons are given automatically by the system as soon as the criteria are met in the transactions.

Automatic coupons will be repeated by default, even when no repetition method is specified. Each item sold may be part of more than one automatic coupon, but only one automatic coupon discount may be applied on an item. However, some methods may be set in the automatic coupons definition to change these default behaviors. For details, see Coupons.

Since automatic coupons are given as soon as the criteria in the coupon wizard are met in the transaction, the order in which the items are scanned can have an impact on which coupons are applied and which ones are not.

Automatic coupon example

The Coupon wizard is used to program the criteria that must be respected for the automatic coupons to be applied.

Below are the steps for programming a Promotion ID automatic coupon. For more examples, see Automatic coupons - examples.

Buy 3 bags of Ruffles chips, get $1.00 off - limit of 2 automatic coupons per transaction

 1. Add the electronic coupon criteria in the main section of the coupon wizard.
  Value field: select Cents off. 
  Discount field: enter 1. 
  Coupon type field: select Automatic. 
  Description field: add a meaningful description. 
  Sub-dpt. field: select the right sub-department. 
   
 
2. In the Criteria tab, click Add criteria. 
  Type field: select Promotion ID sales. 
  Equal to field: enter the promotion identification number. 
  Minimum measure field: select Quantity. 
  Minimum value field: enter 3. 
  Select Make sure the discount affects the item(s) in these criteria.
This method forces the system to apply the automatic coupon discount on one of the items that is part of the Promo ID sales.
  Select The item(s) in this condition can only participate in one offer.
  Click OK to confirm the changes.  
   
 
3. In the Details tab: 
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
   
 
4. In the Descriptions tab:
Expanded description: enter a more detailed description of the promotion.  
   
 
 5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
   
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The customer gets $1.00 off when they buy 3 bags of Ruffles chips.
  • The customer may get up to $2.00 off when buying 6 bags of chips. The limit will then apply on the automatic coupon.
