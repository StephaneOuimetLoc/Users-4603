eSuite Buy - Suggestions Based on Sales

eSuite Buy > Buy > Order > Suggestions

Information

The Based on sales tool suggests items of the current vendor that have been sold to customers within a defined period. The suggested quantity for each item corresponds to the quantity sold during the specified period.

To get suggestions based on sales, access the list of documents in the Order entry of the launcher and then:

  1. Select a purchase order and right-click on it.
  2. Click on Open.
  3. From the start menu, click on Suggestions to open the list of ordering tools.
  4. Click on Based on sales: the Items - list of items wizard opens on the desktop:



    By default:
    • The Period field is set to Day.
    • The period range is set to today's date.
    • The Add minimum inventory section is set to None. No minimum values in inventory will be added to the results.
      Select On movement to add the minimum values in inventory to the items that have movement.
      Select All items to add the minimum values in inventory to all items in the results.
      NOTE: The minimum inventory values for each item are defined in the Minimum unit field of the Inventory table.
    • The Average on 1 period(s) checkbox is set to 1 period(s).
      The value is editable and adds to the one defined in the Period field.
    • The Totalizer to look for field is set to 3: Net sales.
    • The Vendors from current documents checkbox is selected and disabled. The resulting list will only display the items sold by the vendor in the current purchase order.
      NOTE: It is possible to select multiple vendors when the multi-document mode is on in the Context tools.
  5. If necessary:
    • Change the period and the date range.
    • Select a value in the Add minimum inventory field to add the minimum values in inventory to the results,
      or
      Select Subtract current inventory to subtract the current inventory values from the results.
  6. Select the Only items with suggestion checkbox.
  7. Click on Execute.
    The resulting grid dislays the items offered by the current vendor and sold to customers within the specified period. The suggested quantities expressed in cases correspond to the those sold to customers during that period, plus the minimum inventory value or minus the current inventory value for each item. The results are rounded in cases.

  8. Add the items and quantities to the purchase order.
  9. Save the changes.

NOTE: The date range is not restricted to a maximum number of days. It is possible to look for what was sold last Christmas and plan what needs to be ordered for next Christmas. Pay attention to the purge data: if purged after 90 days, for example, no resulting data will be displayed over these 90 days.

Example

The information is taken from Sample eSuite Base.

Kraft (vendor ID A765) sells white vinegar, Oreo cookies and corn cream.

During the first two weeks of May, a number of 76 white vinegar items, 51 Oreo cookies items and 51 corn cream items have been sold (net sales totalizer in item reports). According to these values, the Based on sales tool will give the following results:

The suggested quantities are expressed in cases:

  • White vinegar: 76/12 = 6.33 rounded to 6.
  • Oreo cookies: 51/10 = 5.1 rounded to 5.
  • Corn cream: 51/10 = 5.1 rounded to 5.

The suggested quantity may be used to order. If new values are entered in the Case, Unit and Weight fields of the Order column, they will be taken into consideration in the order.
