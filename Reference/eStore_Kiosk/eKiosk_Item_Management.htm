eKiosk - How to Manage Items

eKiosk

The Featured products

This parameter must be set for each item to display, on the Host or store's server, based on the brand's deploy management. Please contact the reseller for more information.

To display items in the Preferred section of the Start page on the eKiosk, follow these steps:

  1. Enter 'Featured' in the Disc desc. field of the price table.


  2. Deploy changes.

Search filters

Multiple search filters can be used to refine the search and find items more easily on the eKiosk application. When entering a keyword in the search bar, the query is executed on the following fields: F01, F29, F155, F255, F02.

The "Preferences" are used to filter items based on the classification field (OBJ.F1957).
Note that it is possible to set default preferences in the customer account that will be applied automatically when searching the catalog.
The "Select Category" filter will show a list of all the categories (OBJ.F17).
The "Brands" are the brands name (OBF.F155) associated to each items displayed.
This section contains other, more generic, filters.
"Featured items" will display featured items only (PRI.F903).
"Purchase with points" will display items that are available to redeem with points.
The All promotions section is used to filter items based on their active price :
Weekly : items with a SALE active price.
Long term : items with a TPR active price.
Manager : items with a STOR active price.

Preferences

The preferences section is used to filter items based on their classification field (OBJ.F1957) of the Main item table. Multiple preferences can be set by separating them by a ';'.

Item modifiers

It is possible to add extra fields to an item to specify preferences or change the behavior by adding a certain keyword in the Type field (F1785) of the POS table. Here is a list of the current modifiers and what they are used for:

F1785 Description
NOTE All items will have this modifier by default, even if F1785 is empty. It is used to add a basic note to an item.
CAKE Used for custom cake preparation. It will add extra fields related to the flavor, size, color, icing and sprinkles while also allowing to add a message and a picture to put on the cake.
FRUIT Used to specify the ripeness and the size of the item.
MEATBYQTY Used to sell meat by quantity. It will add extra fields related to the marbling, trimming and specify the quantity per package.
MEATBYWEIGHT Same use as MEATBYQTY and will also add fields to select the thickness or the weight by steaks.
MEATDELI Used for the deli meats. It allows to select the total weight, the thickness and the lbs per package.
MEATGROUND Add a total weight field and a lbs per package field.
MEATOTHER Add a quantity per package field.
MEATROAST Add the quantity per package and weight of each roast fields.
TRAYPARTY Used for ordering trays. It will add extra fields for the serving size and the type.
GIFT Allows the user to send the item as a gift by entering an email address.

Special order

When an item is a special order, it is possible to say so by setting the Status (POS.F08) field to SP of the POS_TAB. An item set as a special order will give a warning stating that special order items may take longer to arrive and the user will NOT be charged for that item if he adds it to the transaction.
