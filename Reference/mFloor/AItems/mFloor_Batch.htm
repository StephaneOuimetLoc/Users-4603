mFloor - mFloor Batches

mFloor Item Management

Information

mFloor provides tools to manage batches just as users do in LOC SMS or SMS Pro. Within the batches users can add, modify and activate items, and also modify existing batches and save them in the SMS database.

Creating batches in mFloor

Batches are created in two areas of mFloor, using the Select batch menu or as a Store scan, in both cases when batches are created users can view and access the batches in the Select batch menu.

Creating or modifying existing batches using the Select batch menu

mFloor supports four types of batches: Future TPR (Temporary Price Reduction), Future Sales, Future Price, and Future Instore price. The batch module also provides for immediate Item maintenance changes to fields in the following tables: Main Item, POS, Price, Cost, Shelf location, and Alternate code, these changes do not create batches. More details on Product Maintenance can be found in the mFloor Product module document.

To create or modify a batch using the Select batch menu:

  1. From the Products menu click or tap Select batch.
  2. On the Open batch screen:
    1.  Click or tap Add to create a new batch.
    2. On the New batch screen click or tap the batch type to create, (Future TPR , Future Sale, Future Price or Future Instore Price.)
      or
    3. Click or tap an existing batch to modify it.
  4. On the Batch header screen modify information as required (e.g.: Description, Start & End date, label, etc...) and click save .

  5. To add items to the batch click or tap the application menu and click or tap Add Item.
  6. Enter or scan an item code.
  7. The Item add screen displays the POS descriptor, several other item fields and an Add to batch on scan checkbox.

    The Add to batch on scan checkbox is used to add items to a batch using a barcode scanner. Selecting this checkbox removes the Add and Remove from the upper menu.

  8. Click or tap to add this item to the batch and the appropriate price field and additional information appears in the lower section as shown at right below.
          
  9. Modify the price and other fields as required and click or tap save .
  10. To add additional items, enter or scan code, enter information as above and click or tap save .
  11. The batch application menu provides actions to view items in the currently selected batch.
    • Add Item - Used to add an item to the current batch.
    • Activate Item - By default items in a batch are activated according to the date in the batch header or on each individual item. This action activates items immediately (i.e.: New sale price or TPR.).
    • View Added Items - Displays items added to the current batch.
    • View Items to Activate - Displays those items left to be activated.
    • Product Menu - Displays the Item information for the last item chosen
  12. Once all necessary items are added to the batch, click or tap exit  to return to batch header and again to return to Products menu.

Creating batches using Store scan menu

mFloor provides the  Store scan function enabling users to scan items, place them on the shelves, and create a Batch, which  can then be used to print labels and/or modified as an Instore Price batch from the SMS backoffice server or workstation.

To create a batch using Store scan menu follow the procedure below:

  1. From the Products menu click or tap Store scan.
  2. The first screen contains two sections, Set scan action and Initialize location as described below:
  3. Screen section Field Description



    Set Scan action    		 Use Batch: Determines how the LOCATION of an item is affected when it is scanned. There are three options:
    • No Batch - Does not create a batch, but allows use to make immediate changes to information on a scanned item using the product maintenance menu.
    • By User  - Creates a batch per employee when the first item is scanned and adds other items as they are scanned. Employees may scan the same location in each of their batches.
    • By location. Creates a batch per location when the first item is scanned and adds other items as they are scanned. All employees scan in the same batch for the same location.
    Verified : To verify the current information of an item in the database
    Label : If you want to add this item and print labels in Backoffice server likeInstant Label.
    Initialize Location Initialize loc. : Determines how the LOCATION of an item is affected whenever it is scanned. There are four options:
    • Do not touch - Does not change the location when the item is scanned.
    • Change - Changes the information within the same Shelf Id. I.E.: It will change the POS, Aisle, Side, Section, etc. but needs to be within the same Shelf Id.
    • Add replace - Adds a new location if the Shelf Id is different from the one currently in the database. If Shelf Id is the same as the one in database, it works just like Change.
    • Reset - Deletes the current Shelf Id and creates a new one with the new location.
    Shelf location fields Fill in the remainder Shelf Id, Position, Aisle, Section, and Side as appropriate and choose the corresponding Label template if printing labels for these shelf positions.
  4. Click or tap Next.
  5. At the Scan item screen, scan or type in an item code.
  6. Click or tap to add item to batch.
  7. If required, change the Instore price and Qty fields then click or tap save .

Activating Items in a batch

Batches are usually activated/executed by the scheduler and items in a batch are activated on the date of the batch or by the date on the item itself in the batch. mFloor provides the ability to manually activate individual items that are in the batch.

To activate items in a batch:

  1. Select an existing batch or create a new batch and add items as per section Creating batches using Select batch.
  2. At the Batch header screen click or tap Application menu and click or tap View Items to Activate.
  3. Click or tap an item to open it.
  4. At the Price activate screen click or tap Next to activate the item.
  5. Select Activate on scan to activate items by scanning them instead of choosing from the list.
