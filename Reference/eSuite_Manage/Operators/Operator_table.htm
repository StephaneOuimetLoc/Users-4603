eSuite Manage - Operator Table 

eSuite Manage > Operators > Operator List

Information

To use the eSuite software, users must have an operator number assigned to their profile in the system. The information on all system users, also known as operators, is kept in the Operator table.

Each operator is assigned a user level or security profile. The eSuite software can be configured to control operator information for more than one site. In multistore situations, each record has a Target field, linking the operator to a store or a group of stores.

The Operator table contains security information that requires a high security profile in order to be accessed. Users who have access to the Operator table cannot modify user profiles with a higher security level than the one assigned to them. Only a user with the programmer security level has the authority to modify profiles in the Operator table.

Special notes

To create, edit, modify or delete an operator in the Operator table, you need to have an equivalent or higher security profile than the user's profile you are creating/editing/modifying or deleting. This is necessary to prevent a user with a lower security profile from tampering with security levels and, consequently, confidential information.

Please be advised that by deleting an operator, you are deleting the historical information linked to that information. For that reason, LOC Software recommends modifying records instead of deleting them.

Sales personnel are defined in the system like any other operator. The difference between an operator and a salesperson is the way to log in. For salespersons, certain settings must be defined in the System.ini file. Contact your reseller for more information.

Displaying the list of operators

Operator profiles are created through the Operator table and displayed in the operator list.

To access the operator list, from the Manage module of eSuite:

  1. In the Launcher, click on Operators: the Start window opens in the Operators work area.
  2. Select Operator list: the grid of all operators in the system opens in the Operators window.

Displaying the Operator table

From the Manage module of eSuite:

  1. In the Launcher, click on Operators: the Start window opens in the Operators work area.
  2. Select Operator list: the grid of all operators in the system opens in the Operators window.
  3. To modify an existing operator record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Operator table opens, overlapping the work area.
  4. To create a new operator record, click on Add in the grid toolbar: the Operator table opens, overlapping the work area.

The Sequence field

The Sequence field contains the operator order number. It is auto-generated by the software when a new operator is created. It is grayed out and not editable.

The eSuite software uses the first record number available. If a record is deleted, eSuite will re-use its number the next time a record will be added.

On a multi-target layout, the sequence number must be different for each site to avoid conflicts at the Head office when the data is deployed. Thee sequence number must start by a different number at each site. See your dealer for more details.

The Number field

The Number field contains the number used by the operator to log into the system. This operator number must be unique.

If more than 8 digits are used, the default finish (to) numbers when running a report need to be changed. The default range is from 0 to 99999999; other 9's will need to be added to be able to display the information for the operator numbers that contain more than 8 digits.

The Number field is numeric-only, since some front-end workstations and cash registers may not have alpha keypads. The operator number may be different than the assigned sequence number.
