eCommerce - Deployment Methods by Assortment

eCommerce / eStore

Table of Contents

Information

There are different methods of deployment available with the e-commerce solution. Depending on the assortment method that was chosen during installation, here is the procedure to follow for items to be deployed.

For all methods, items must be for sale and active to be deployed.

Active and for Sale

All items for sale and active in the database are deployed. And when items in the eCommerce server database change to 'Not for sale' or the status 'Discontinued', these items are removed from the eCommerce server database.

Items flagged by Category

Only items with 'W' (E-Commerce) in the 'Filter' field of the Category table are deployed.

NOTE: This method is supported in the following scenarios: 'Virtual Lane deployment from the Store', 'Virtual Lane deployment from the Host' and 'Virtual Store deployment from the Host'.

Items flagged by Department

OnOnly items for sale, active and with 'W' (E-Commerce ) in the 'Show filter' field of the Department table are deployed.

NOTE: This method is supported in the following scenarios: 'Virtual Lane deployment from Store', 'Virtual Lane deployment from the Host' and 'Virtual Store deployment from the Host'.

Only Items 'Web Order enabled'

Only items for sale, active and with 'FE' (Web Order enable) in the 'Others' field of the POS table are deployed.

ALL Items except 'Web Order disabled'

All items for sale and active, except those with 'FD' (Web Order disable) in the 'Others' field of the POS table are deployed.
