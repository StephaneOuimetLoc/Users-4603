Shopper Level Upgrade Examples

eSuite > Customers > Customer auxiliaries> Shopper Levels

Information

The Shopper level table allows customers to upgrade to another shopper level in particular situations. This is useful for special promotions (discounts, more points, add-on amounts on vendor coupons, etc.) to customers when certain conditions are reached. This can be a good way of creating a certain movement in the store on slow moments.

NOTES:
  • The shopper level upgrade is valid for the current transaction only. The conditions are revaluated for each transaction.
  • The shopper level upgrade is performed when the Sub-total function is called in the transaction.

List of examples

  1. Regular customers (shopper level 1) upgrade to shopper level 4 (which gives 5% off) when they buy for $100.00 in the transaction.
  2. Preferred customers (shopper level 2) upgrade to shopper level 4 (which gives 10% off) when they have at least 5000 points in their points balances.
  3. Preferred customers (shopper level 2) upgrade to shopper level 5 (which uses price level 2) when they buy for $300.00 (net sales) during the month.
  4. Preferred customers (shopper level 2) upgrade to shopper level 6 (which doubles the points given) on Wednesdays from 8:00am to 12:00pm.
  5. Preferred customers (shopper level 2) upgrade to shopper level 4 (which gives 10% off) when they buy for $100.00 in the transaction and have at least 5000 points in their points balances.
  6. Regular customers (shopper level 1) upgrade to shopper level 7 (which doubles vendor coupons) on Mondays and Tuesdays from 12:00pm to 5:00pm.
  7. Regular customers (shopper level 1) upgrade to shopper level 7 (which allows item and sub-department discounts) when they buy for $50.00 in the transaction on Sundays.

1. Regular customers (shopper level 1) upgrade to shopper level 4 (which gives 5% off) when they buy for $100.00 in the transaction

 Set up the information for shopper level 1:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Type 100.00 in the Transaction total over field.
  3. Save the changes.

Set up the information for shopper level 4:

  1. Type 5 in the Discount % field.
  2. Save the changes.

In this example, no customer needs to be entered in the transaction since the shopper level 1 is the default shopper level used and the condition does not need any customer information.

The discount of the shopper level 4 is given when the Sub-total function is called, if the condition is reached (the customer buys for a minimum of $100.00 in the transaction).

2. Preferred customers (shopper level 2) upgrade to shopper level 4 (which gives 10% off) when they have at least 5000 points in their points balances

Set up the information for shopper level 2:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Enter 5000 in the Points balance over field.
  3. Save the changes.

Set up the information for shopper level 4:

  1. Type 10 in the Discount % field.
  2. Enter all other necessary values from shopper level 2.
  3. Save the changes.

In this example, a customer must be entered in the transaction since the condition needs customer information (i.e. the customer's point balance).

The discount of the shopper level 4 is given when the Sub-total function is called, if the condition is reached (the customer has at least 5000 points in his points balance at the beginning of the transaction; the points given during the transaction are not taken into consideration).

3. Preferred customers (shopper level 2) upgrade to shopper level 5 (which uses price level 2) when they buy for $300.00 (net sales) during the month

Set up the information for shopper level 2:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Type 300.00 in the Period spending field.
  3. Select Current month in the Period field.
    The displayed value is M.
  4. Type 3 (Net sales totalizer) in the Totalizer field.
  5. Save the changes.

Set up the information for shopper level 5:

  1. Type 2 in the Price level field.
  2. Enter all other necessary values from shopper level 2.
  3. Save the changes.

In this example, a customer must be entered in the transaction since the condition needs customer information (i.e. the total amount of the customer's purchases during the current month).

The prices of the items bought in the transaction are changed when the Sub-total function is called, if the condition is reached (the customer bought for a minimum of $300.00 (Net sales) during the current month.

4. Preferred customers (shopper level 2) upgrade to shopper level 6 (which doubles the points given) on Wednesdays from 8:00am to 12:00pm

Set up the information for shopper level 2:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Type 0800 in the Wednesday (Wed) Start field.
  3. Type 1200 in the Wednesday (Wed) Stop field.
  4. Save the changes.

Set up the information for shopper level 6:

  1. Type the identifier of the points programs in the Point programs field:
  2. Select the desired point method in the Actual method field:
  3. If necessary, type the desired penny per point rate in the Penny/point field:
  4. Type 2 in the Multiplier (points) field:
  5. Save the changes.

In this example, a customer must be entered in the transaction since the condition needs customer information (i.e. the points programs the customer is member of).

The points given are doubled when the Sub-total function is called, if the condition is reached (the customer buys on Wednesdays between 8:00am and 12:00pm).

5. Preferred customers (shopper level 2) upgrade to shopper level 4 (which gives 10% off) when they buy for $100.00 in the transaction and have at least 5000 points in their points balances

Set up the information for shopper level 2:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Type 100.00 in the Transaction total over field.
  3. Type 5000 in the Point balance over field.
  4. Save the changes.

Set up the information for shopper level 4:

  1. Type 10 in the Discount % field.
  2. Enter all other needed values from shopper level 2.
  3. Save the changes.

In this example, a customer must be entered in the transaction since one of the conditions needs customer information (i.e. the customer's points balance).

The discount of the shopper level 4 is given when the Sub-total function is called, if all conditions are reached (the customer buys for a minimum of $100.00 in the transaction and they have at least 5000 points in their point balance).

6. Regular customers (shopper level 1) upgrade to shopper level 7 (which doubles vendor coupons) on Mondays and Tuesdays from 12:00pm to 5:00pm

Set up the information for shopper level 1:

  1. In the Redeem tab, type 4 in the Upgrade to level field.
  2. Type 1200 in the Monday (Mon) Start field.
  3. Type 1700 in the Monday (Mon) Stop field.
  4. Type 1200 in the Tuesday (Tue) Start field.
  5. Type 1700 in the Tuesday (Tue) Stop field.
  6. Save the changes.

Set up the information for shopper level 7:

  1. Type 2 in the Coupon multiply field.
  2. Save the changes.

In this example, no customer needs to be entered in the transaction since the shopper level 1 is the default shopper level used and the condition does not need any customer information.

The discount of the shopper level 7 is given when the Sub-total function is called, if the condition is reached (the customer buys on Mondays and Tuesdays between 12:00pm and 5:00pm). 

Go to the list of examples

7. Regular customers (shopper level 1) upgrade to shopper level 7 (which allows item and sub-department discounts) when they buy for $50.00 in the transaction on Sundays

Set up the information for shopper level 1:

  1. In the Redeem tab, type 7 in the Upgrade to level field.
  2. Type 50.00 in the Transaction total over field.
  3. Type 0000 in the Sunday (Sun) Start field.
  4. Type 2359 in the Sunday (Sun) Stop field.
  5. Save the changes.

Set up the information for shopper level 7:

  1. In the Discounts tab, check the Apply item discount % field.
  2. Check the Apply item discount $ field.
  3. Check the Apply subdepartment discount field.
  4. Save the changes.

In this example, no customer needs to be entered in the transaction since the shopper level 1 is the default shopper level used and the conditions do not need any customer information.

The allowed discounts of the shopper level 7 are given when the Sub-total function is called, if the conditions are reached (the customer buys for $50.00 in the transaction on Sundays).
