eSuite - Change Log

eSuiute > Items > Master > Change log

Information

The Change logs feature allows displaying changes that have been made to any of the four item master tables: price, cost, POS and main (OBJ). These changes refer to the updating, deletion and addition of information.

To access the change logs:

  1. In the launcher, click on Items.
  2. Click on Master: the Start window opens, displaying the item-related options.
  3. Click on Change log.

The Items - display changes wizard

The Change log entry opens the Items - display changes wizard, with criteria selection that includes start and end date, price level, type of price event, item code, POS description, target and the Not for sale option for the items that cannot be sold at the POS. Criteria related to the item code range may be added with the Add criteria button.

Start and end dates

The start and end dates allow defining the period when changes have been made to the master tables. Both fields must be filled out for the query to be executed.

The Start date and the End date fields allow you to enter the date and time or select them from the picker:

The following buttons are available:

  • OK confirms the time and date selection.
  • Today sets the current date and time.
  • Now sets the current time and date.

Required date format: mm/dd/yyyy.

Required time format: hh:mm:ss.

The Items - display changes grid

The Items - display changes grid shows the price, cost, POS and OBJ changes within the selected period and according to the criteria set in the wizard.

In the bottom menu:

  • Analyze allows performing item subqueries and linked queries.
  • Action allows deploying the changes.

Price changes

The following changes are displayed by default in the Price changes section of the grid:

Further columns may be added from the column headers:

Here is the description of the columns and the information they may contain.

COLUMN DESCRIPTION
Price The active price of the item.
Event source The type of pricing even that triggers the change: regular, sale, TPR or store.
Start date The date on which the pricing event triggering the change will be effective.
End date The date on which the pricing event will no longer be effective.
Margin The active margin, calculated using the active price.
Type of operation Update: existing information that has been modified.
Insert: new information added to the price table.
Delete: information that has been removed from the price table.
Next price The next price of the item, based on the changes made in the pricing columns of the Price table.
Next price date changes The date and time when the future pricing even will take place.
Fields of the changes The number of the fields that have been changed.
Level The price level.
Responsible for the changes The identifier of the user that has made the change (operator number in SMS or user code in eSuite).
Target The target identifier.

Reading the grid

For each change in any of the four master tables (price, cost, POS and OBJ), the grid will display the first change in the three other master tables where the date of the change is equal to the one in the master table.

For example, if the TPR is changed in the Price table, all changes related to cost, POS and OBJ that match the start date for the TPR event will be displayed in the change grid. When there are no matching changes, the fields will be left emtpy.
