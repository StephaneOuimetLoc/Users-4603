ePos - Reverse Attributes

Epos operations

Table of content

Information

Reverse attributes consist of shift functions. These functions will perform operations to inverse specific flags on items according to the function.

The following are the shift functions available:

  1. Function 740 Tax shift
  2. Function 742 Scale shift
  3. Function 744 Discount shift
  4. Function 746 Food stamp shift
  5. Function 747 WIC shift
  6. Function 748 FSA shift 

Tax Shift

The Tax shift function manually adds or removes the taxability of an item sold. If an Item is flagged taxable(x) on the POS_TABLE, the tax shift will remove the tax(x). If an Item does not have a flag taxable(x) on the POS_TABLE, the tax shift will add the tax.

To temporarily add or remove the taxability of an item, follow these steps:

  1. Open an epos session and login
  2. Select the tax shift button.
  3. A message will pop up in the Entry window  "Tax shift?".
  4. In the Entry window, enter the tax number(s) that must be added or removed.
  5. Press the Enter key or click on the E button to validate your entry; The "Tax shift [numbers]" message is displayed in the Entry window.
  6. Enter the item.

Note: The Tax shift function must be used before entering the item.
The Tax shift is effective for one (1) item only. This means you have to use the Tax shift function before each item for which the taxability must be added or removed in the transaction.

Scale shift

The Scale shift function manually adds or removes the scalable flag of an item sold.

To temporarily alter the scalability of an item that is not flagged scalable, follow these steps:

  1. Open an epos session and login.
  2. Select the Scale shift button.
  3. The "Scale shift" message is displayed in the Entry window.
  4. Enter the item. 

Notes :
The Scale shift function must be used before entering the item.
The Scale shift is effective for one (1) item only. This means you have to use the Scale shift function before each item for which the scalability must be added or removed.

Discount shift

The Discount shift function manually removes the discount ability of an item sold and sets the item to
"None discountable" on the item line. The discount shift can also temporarily remove the "Stop discount" flag of an item and allow it to receive discounts.

To temporarily alter the discount ability of an item, follow these steps:

  1. Open an epos session and login.
  2. Select the Discount shift button.
  3. The "Discount shift" message is displayed in the Entry window.
  4. Enter the item.

Notes:
The Discount shift function must be used before entering the item.
The Discount shift is effective for one (1) item only. This means you have to use the Discount shift function before each item for which the discount ability must be altered.
The Discount shift function affects manual item discounts, proportional discounts, global discounts and shopper level discounts.

FSA Shift

The FSA shift function manually adds or removes the FSA ability of an item sold, whether the item could be purchased using a FSA card as tender or not.

To temporarily alter the FSA ability of an item, follow these steps:

  1. Open an epos session and login.
  2. Select the FSA shift button.
  3. The "FSA shift" message is displayed in the Entry window.
  4. Enter the item.

Important: By default the FSA Shift function does not allow access to any operator. Make sure you have configured the proper access rights before using this function.

Notes: The FSA shift function must be used before entering the item. The FSA shift is effective for one(1)  item only. This means you have to use the FSA shift function before each item for which the FSA ability must be altered.
