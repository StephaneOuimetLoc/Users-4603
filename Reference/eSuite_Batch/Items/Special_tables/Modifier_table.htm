eSuite - Modifier Table

eSuite > Items > Master > Item list

Information

The Modifier table is used to assign modifier comments to an item. It works with the other item tables, as its entries are related to a UPC.

It is possible to have as many notes as required and they can be linked to any item. Notes will appear on the kitchen receipt.

It is currently used with the fast food kiosks to request condiments when ordering sandwiches. After the customer selects the sandwich they want to order, the system asks them for the condiments they want. In this example, the sandwich is considered the main item and the different condiments are the modifiers. As many modifiers as needed may be linked to the same main item.

Some modifiers are compulsory, meaning that they have to be selected for the main item be sold. Others are considered as extras.

Also, some modifiers change the price and some do not. In the example above, an extra bacon may charge an additional amount.

NOTE: Even if the Modifier table is an item table, it is deployed to the front-end workstations as an Auxiliary table, using the Deploy auxiliaries job.

Displaying the Modifier table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens in the Items work area.
  2. Click on Item list: the Wizard tab opens in the same window, displaying the Items - General List.
  3. Enter the search criteria and click Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select a record and click on Edit on the bottom of the item grid: the option menu is displayed.
  5. Select Special items and then Modifier: the Modifier edit table opens overlapping the work area.

Layout

The Modifier table is divided in two parts: the main section and the Modifier table grid.

Main section

* Click on a field to get information on it.

Modifier table grid

* Click on a field to get information on it.

The Modifier table grid offers quick information on the modifiers that are linked to the item. The following columns are displayed: Sequence, Required, and Type.

Columns are movable, resizable and may be selected/deselected for a customized experience. They allow sorting the records in an ascending or a descending order.
By default, records are displayed in ascending order by sequence. Selecting a record allows displaying and editing its information in the main section of the Modifier table.

Content

Here is a description of the item Modifier table fields and their functionalities:

FIELD DESCRIPTION
Main section
Sequence The Sequence field contains the sequence number that determines the order in which the modifier will be asked to the customer.
Description The Description field contains the description of the modifier.
Required The Required field contains the identifier letter defining the nature of the modifier.
Compulsory (C) means that the customer must make a choice on this modifier.
Extra (E) means that the customer can make a choice on this modifier.
Type The Type field contains the identification letters of the type used to determine how the modifier will be filtered, i.e. to determine the choice to display for the modifier.
The following options are available: Category, Family code, Report code, Sub-department and Department.
Link The Link field contains the code of the selected type.
It displays the list of modifiers, according to the selected type.

Item modifier setup

A modifier is an item that affects another item during a sale. If the item modifier has a price, it will affect the price of the main item it is linked to. The best example is in a restaurant environment where modifiers such as ham, lettuce, tomatoes, etc. extra bacon are are added to the main item, a sandwich.

More than one modifier may be linked to an item.
More than one modifier group may be linked to an item.

Item modifiers and modifier groups are listed and linked to the main item through the Modifier table. See the example below.

Application

The item modifiers are used in one of these options/plugins:

  • Kiosk_FastFood
  • Plugin_POS_Delivery
  • Plugin_POS_FastFood

See the reseller for more details on these options/plugins.

Linking an item to a modifier

This example illustrates how to link an item to modifiers grouped by category.

  1. Create the main item (Chicken sandwich).
  2. Create the category 500000 called Extras.
  3. Create the items that are extras (cheese, tomatoes, lettuce, etc.).
  4. From the main item (Chicken sandwich), open the Modifier table and set the following values:
    Description field: add a relevant description.
    Required     field: select Extra.
    Type field: select Category.
    Link field: select 5000000.

The extras are now linked to the Chicken sandwich item.

Required item modifier group

An item modifier group can be Compulsory (C) or Extra (X). The main difference between these two kinds of requirements is that you absolutely need to make a choice for the Compulsory item modifier, like a yes/no question that can have more than 2 choices (for example, the type of bread of a sandwich). On the other hand, the Extra item modifier can have several item modifiers selected or none of them (for example, the different toppings of a sandwich).

Each item modifier can be grouped by category, family, report, sub-department or department. Once each group is created, they can be associated with more than one main item.

Through the eKiosk application, it is possible to have an image attached to an item in the Modifier table. The image must be saved in the \Storeman\Bitmaps directory, under the UPC of the item (for example, 0000000006000.jpg). The image file can be of any HTML valid type such as .bmp, .jpg, .gif or .png.

Tips

To have a Yes/No choice like "Do you want an extra bacon? (Yes/No)", two items have to be created: one for the YES option and one for the NO option. These two items have to be grouped under the same category, such as Extra Bacon.

Fees

To have a modifier that affects the price of the current item, the configuration needs to be done accordingly.

See your reseller for the details.

Ability to multiply extra modifiers

Through the eKiosk application, it is possible to multiply the extra modifiers by a quantity of 2 or 3 without having to create separate modifiers. For example, to have a multiplier of 2 (x2) and 3 (x3) for the extra bacon modifier, the value 3 has to be set in the Qty rent field of the POS table for the extra bacon item.
