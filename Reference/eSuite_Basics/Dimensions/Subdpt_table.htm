eSuite - Sub-department Table

eSuite > Dimenstions > Sub-Departments > List

Information

The Sub-department table contains information for groups of items. The eSuite uses sub-departments as a way of grouping both coded items and direct key sales for reporting.

Instead of linking coded items directly to departments, eSuite uses sub-departments to group items, and then links the sub-departments to departments. This allows for another report level, regrouping item codes in a more specific manner.

The sub-departments also have their own settings, such as taxability, maximum and minimum entries for sales and voids etc., used when direct key sales are performed. The items can also be programmed to follow these settings. The Sub-department table has to be deployed to the points of sale. The eSuite software does not support sub-department programming for all types of points of sale. The sub-department information may not get through to all points of sale; however, the Sub-department table must be programmed to use the sub-departments for reporting purposes.

Displaying the Sub-department table

In the Launcher, click the arrow in the Dimensions entry to display its menu and then:

  1. Click on Sub-departments: the Start window opens in the Sub-depart work area.
  2. Click on List: the Sub-depts tab opens in the same window, displaying the Sub-department List.
  3. To modify an existing sub-department, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit - Sub-department F3 command in the bottom menu: the Sub-department table opens, overlapping the work area.
  4. To create a new sub-department, click on Add in the bottom menu of the grid: the Sub-department table opens, overlapping the work area.

Content

Here is a description of the Sub-department table fields and their functionalities:

FIELD DESCRIPTION
Main section
Number The Number field contains the sub-department number.
It is a required field used to uniquely identify records in the sub-department list.
Description The Description field contains the description of the sub-department.
It is not a required field; however, a description should be entered to identify the sub-department in the sub-department reports.
Department The Department field links the sub-department to a department for global sales reports.
It is a required field. There is no limit as to how many sub-departments can be linked to the same department.
Regardless of the sub-department number, the grouped sales reports combine all sub-departments having the same department at the report generation time.
Max. entry The Maximum entry field contains the maximum amount the cashiers can enter for the sub-department key sales.
It prevents cashiers from entering too high a value in the sub-department key.
Min. entry The Minimum entry field contains the minimum amount the cashiers can enter for the sub-department key sales.
It prevents cashiers from entering too low a value in the sub-department key.
Max. void The Maximum void field contains the maximum amount the cashiers can enter as a void (correction) for the sub-department key sales.
It prevents overpunching void amounts at the points of sale.
Max. refund The Maximum refund field contains the maximum amount the cashiers can refund for the sub-department key sales.
It prevents cashiers from refunding too high an amount in the sub-department key.
Cust. age The Customer age field contains the minimum age the purchasers must be to buy any item in the sub-department.
For details, see the Customer age field section below.
Cash. age The Cashier age field contains the minimum age the cashiers must be to perform sales in the sub-department.
For details, see the Cashier age field section below.
Margin The Margin field contains the profit margin on the items of the sub-department, in percentage.
Discount The Discount field contains the percentage of discount applied when a sub-department sale is performed.
The Apply sub-dept disc field must be checked in the corresponding shopper levels for the Discount to work.
If an item discount (discount $ or discount %) is also applicable on the item sold, the item discount is applied before the sub-department discount.
Restrictor The Restrictor field links the sub-department to a restriction that the items in the sub-department and the sub-department sales must follow.
For example, in areas where stores are allowed to sell beer, wine or alcohol, sales might be legally restricted to certain hours of the day.
Commission The Commission field contains the salesperson's commission rate for the sub-department.
Shrink The Shrink field contains the percentage that is allowed to be lost on a regular basis for the items in the sub-department.
It is mostly used with perishable products.
Other flags The Other flags field contains additional sub-department sales information.
For details, see the Other flags field section below.
Mem. no. The Memory number field contains the memory number that links the keyed sub-department sales to the scales or the cash registers.
Priority The Priority field contains the order in which the sub-department must appear in the specified modules.
Show filter The Show filter field contains the identification letters of the modules where the sub-department must appear.
For details, see the Show filter field section below.
URL The URL field contains special parameters used when a sale is performed in the sub-department.
Scalable The Scalable field specifies whether the sub-department sales are performed by weight, scaled at the POS (checked) or not (unchecked).
Manual The Manual field specifies whether the weight of the scalable sub-department sales can be manually entered using the POS keyboard (checked) or not (unchecked).
WIC item The WIC item field specifies whether the sub-department sales are WIC able (checked) or not (unchecked).
Food stamp The Food stamp field specifies whether the sub-department sales can be performed using food stamps as tender (checked) or not (unchecked).
FSA eligible The FSA eligible field specifies whether the sub-department sales can be performed using a FSA card as tender (checked) or not (unchecked).
Not in net The Not in net field specifies whether the sub-department sales are added to the store net sales (unchecked) or not (checked).
Exc. spending The Exclude from spending field specifies whether the keyed sub-department sales are excluded from discounts and points programs (checked) or not (unchecked).
Taxable 1 The Taxable 1 field specifies whether the sub-department sales have the first tax for the retail region (checked) or not (unchecked).
Taxable 2 The Taxable 2 field specifies whether the sub-department sales have the second tax for the retail region (checked) or not (unchecked).
Taxable 3 The Taxable 3 field specifies whether the sub-department sales have the third tax for the retail region (checked) or not (unchecked).
Taxable 4 The Taxable 4 field specifies whether the sub-department sales have the fourth tax for the retail region (checked) or not (unchecked).
Taxable 5 The Taxable 5 field specifies whether the sub-department sales have the fifth tax for the retail region (checked) or not (unchecked).
Taxable 6 The Taxable 6 field specifies whether the sub-department sales have the sixth tax for the retail region (checked) or not (unchecked).
Taxable 7 The Taxable 7 field specifies whether the sub-department sales have the seventh tax for the retail region (checked) or not (unchecked).
Stop discount The Stop discount field specifies whether sub-discounts may be applied on the sub-department sales at the points of sale (unchecked) or not (checked).
For details, see the Stop discount field section below.
Stop return The Stop return field specifies whether the sub-department sales should be returned (unchecked) or not (checked).
Stop refund The Stop refund field specifies whether the sub-department sales should be refunded (unchecked) or not (checked).
Bottom section
 Shopper The Shopper field contains the number of the shopper level to which a discount is applied when a sub-department sale is performed for the customers that are part of that shopper level.
The discount given is specified in the Discount field.
For details, see the Shopper field section below.
Discount The Discount (Shopper) field contains the percentage of discount applied when a sub-department sale is performed for the customers that are part of the shopper level specified in the Shopper field.
For details, see the Discount field section below.

The Customer age field

The Customer age (Cust. age) field contains the minimum age the customers must be of to buy any item of the sub-department. For example, in areas where stores are allowed to sell beer, wine or alcohol, customers must be of legal adult age for that area in order to buy these items.

NOTES:
The Customer age may also be specified for the department key or an item only.
If there is no customer age specified at the item level, the system automatically looks for the customer's age at the sub-department level ONLY if the Follow sub-department field is checked.
In the case where a customer's age is specified at the item level AND at the sub-department level, the system verifies both values. If these values are different, the customer age specified at the item level has precedence over the one at the sub-department level.

When a sale is performed in the sub-department, the cashier is prompted to enter the customer's birth date at the POS. The system automatically calculates the customer's age and compares it to the value in the Cust. age field. If the customer is under the minimum age required, a warning message is displayed on the POS screen.

NOTES:
If the operator is a level 5 or higher, this restriction will be bypassed.
See your reseller regarding the possible alternate configurations (System.ini file).

The Cashier age field

The Cashier age (Cash. age) field contains the minimum age that the cashiers (operators) must be of to sell the items in the sub-department and perform sub-department sales. For example, in areas where stores are allowed to sell beer, wine or alcohol, cashiers must be of legal adult age for that area in order to sell these items.

The operator's age is verified based on the information entered in the Birth date field of the Operator table. If no birth date has been specified in this field, the system does not perform any age validation.

NOTE: The cashier's age can also be specified for the department key or for an item only.

The Other flags field

The Other flags field contains information on additional sub-department sales. A sub-department may have more than one value in the Other flags field.

NOTES:
For this field to apply to the items in the sub-department, they must be programmed to follow the sub-department settings.
The Other flags information may also be specified for the department keys or for an item only.

To enter additional sub-department sales information, type the abbreviation of the desired information in the Other flags field or click in the Other flags field to select the desired information from the list of choices:

OTHER FLAGS INFORMATION ABBREVIATION DESCRIPTION
Enable zero price EZ The sub-department sales accept zero prices in the price fields of the Price table ($0.00).
No cost plus price NC No cost plus price can be calculated for the sub-department sales.
WIC 2 W2 The sub-department sales are WIC able 2.
WIC 3 W3 The sub-department sales are WIC able 3.
WIC 4 W4 The sub-department sales are WIC able 4.
Web Order FE The item can be ordered from the Web.
Scale fix weight FW The scaled sub-department's sales have fixed weights (e.g. store-made muffins). See the scale manual for more information.
Scale fix weight by count FC The scaled sub-department sales have fixed weight and is By Count.
Scale fix price SF The scaled sub-department sales have fixed prices (e.g. store-made pizzas). See the scale manual for more information.
Scale special weight SW The scaled sub-department sales are 'weighable' and 'by count'.
Scale open price SO The scaled sub-department sales allow an operator to enter a price per pound/kilogram and the system will calculate the price based on the weight of the item.
Scale force tare FT The scaled sub-department sales have a tare that is forced in the scale. See the scale manual for more information.


NOTES:
 More than one type of information may be entered, with no space in between.
For more detailed information on the Scale settings, please refer to the Scale - Over and Deploy document.

The Show filter field

The Show filter field contains the identification letters of the modules where the sub-department must appear.

The sub-department will only appear in the specified modules, or in any of the modules if the Do Not Show value is entered.

To enter the modules where the sub-department must appear, type the identification letters of the desired modules in the Show filter field or click in the Show filter field to select the desired modules from the list of choices.

The Stop discount field

The Stop discount field specifies whether discounts may be applied on the sub-department sales at the points of sale (unchecked) or not (checked).

It does not allow manual item discounts, proportional discounts, global discounts, customer discounts (shopper level discounts) and sub-department discounts on the sub-department sales.

NOTES:
For this field to apply to the items of the sub-department, they must be programmed to follow the sub-department settings.
The Stop discount behavior may also be specified for the department keys or an item only.

To prevent discounts on the sub-department sales, check the Stop discount field.

The Shopper field

The Shopper field in the bottom section displays the shopper level to which a discount is applied in that sub-department. Customers that are part of that shopper level will then benefit from the discount that is specified in the Discount (Shopper) field

To add a shopper level:

  1. Once the Sub-department table is open, pin it to the desktop.
  2. On the bottom of the Launcher, click on the Add to current desktop icon.
  3. Open the list of shopper levels from the Clients Auxiliaries entry and select the shopper level to which you want to add a shopper discount.
  4. In the Sub-department table, click on +: the selected shopper level is automatically displayed in the Shopper field.

To delete a shopper level, follow these steps:

  1. Select the shopper level in the grid and click on -.
  2. A message will display, asking you to confirm your choice:

Deleting the shopper level will also delete the discount applied to it.

NOTE: The shopper discount supersedes the sub-department discount. A shopper discount can be entered for each shopper level in each sub-department.

The Discount (Shopper) field

The Discount (Shopper) field contains the percentage of discount applied when a sub-department sale is performed for the customers that are part of the shopper level specified in the Shopper field.

A shopper discount can be entered for each shopper level in each sub-department.

To add a discount in the bottom section, follow these steps:

  1. Select the Shopper level for which the discount will apply
  2. Double-click in the Discount field to enable editing
  3. Enter the percentage of discount to apply
  4. Save the changes

NOTE: The shopper discount supersedes the sub-department discount.
