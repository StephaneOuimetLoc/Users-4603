eSuite - UPC Entries

eSuite > Items 

Information

The item UPC (Universal Product Code) uniquely identifies items in eSuite. The system allows different types of UPCs, including five that are commonly used. They are entered according to the following guidelines:

  • UPC-A barcode
  • UPC-E barcode
  • EAN-13 barcode
  • EAN-8 barcode
  • GS1 Databar barcode

NOTE: The system will automatically expand the UPC to 13 digits by adding nine zeros (000000000) at the beginning when the Execute button is clicked in the wizard.

UPC-A barcode

The UPC-A barcode is the most common and well-known symbol in USA and Canada. It is the grocery standard barcode, commonly called 'UPC barcodes' or 'UPC symbols'.

The UPC-A barcode has 12 numeric-only characters, divided in four areas:

  1. The number system (1 digit)
  2. The manufacturer code (5 digits)
  3. The product code (5 digits)
  4. The check digit (1 digit)

Enter only 11 digits in the system maintenance. Start with the first digit on the left (even if it is zero) and enter all following digits except the check digit (the last one). The system will automatically expand the UPC to 13 digits by adding two zeros (00) at the beginning when the Execute button is clicked in the wizard.

Bulk scale barcode

The Bulk scale barcode is a UPC-A barcode for which the number system is 2. It is used for pre-packed products (also called random weight items). The prices of the pre-packed products are directly printed in the Bulk scale barcodes.

The Bulk scale barcode has 12 numeric-only characters, divided into four areas:

  1. The number system (1 digit) = 2
  2. The scale product code (5 digits)
  3. The scale product price (5 digits)
  4. The check digit (1 digit)

In the system maintenance, enter 2 followed by the five digits of the scale product code and five zeros (00000). The five zeros are used to create the code, but when a pre-packed item is scanned, the price is read by the scanner. The system will automatically expand the UPC to 13 digits by adding two zeros (00) at the beginning when the Execute button is clicked in the wizard.

You can also enter the 5 digits of the scale product code followed by *. When you click on the Filter button, the system will add the two zeros (00) and the number system (2) at the beginning of the UPC as well as the five zeros (00000) at the end. For example, if you enter 65432*, it will become 0026543200000 when the Execute button is clicked in the wizard.

UPC-E barcode

The UPC-E barcode is a variation of the UPC-A barcode. It allows for more compact barcodes, used on products with small packaging where a full UPC-A barcode could not reasonably fit. The UPC-E barcode reduces the 10 digits of the message characters (manufacturer code - 5 digits and product code - 5 digits) of the UPC-A barcode into only six digits.

The UPC-E barcode has eight numeric-only characters, divided into three areas:

  1. The number system (1 digit)
  2. The message characters (6 digits)
  3. The check digit (1 digit)

The number system digit and the check digit are printed as human-readable numbers, but are encoded in the parity of the other six digits.

In the system maintenance, enter six digits. Start with the second digit on the left and enter all following digits except the check digit (the last one). The system will automatically convert the UPC-E code (6 digits) to a UPC-A code (11 digits) and then expand the UPC to 13 digits by adding two zeros (00) at the beginning when the Execute button is clicked in the wizard.

EAN-13 barcode

The EAN-13 barcode is the international grocery standard barcode. It was implemented mostly because the UPC-A barcode was not designed for international use.

The EAN-13 barcode has 13 numeric-only characters, divided into four areas:

  1. The number system (2 digits)
  2. The manufacturer code (5 digits)
  3. The product code (5 digits)
  4. The check digit (1 digit)

The EAN-13 barcode is a superset of the UPC-A barcode. Any software or hardware capable of reading a EAN-13 barcode is automatically able to read a UPC-A barcode. In fact, a UPC-A barcode is an EAN-13 barcode with the first digit of the number system set to zero.

Visually, the human-readable check digit is placed below the barcode and not on its right. This does not make any difference on the encoding.

In the system maintenance, enter 12 digits. Start with the first digit on the left (even if it is zero) and enter all following digits except the check digit (the last one). The system will automatically expand the UPC to 13 digits by adding a zero (0) at the beginning when the Execute button is clicked in the wizard.

EAN-8 barcode

The EAN-8 barcode is the EAN equivalent of the UPC-E barcode in the sense that it provides a short barcode for small packages. However, the EAN-8 barcode is not compatible with the UPC-E barcode (like the EAN-13 barcode is compatible with the UPC-A barcode). The EAN-8 barcode has no EAN-13 equivalence; it is directly assigned by the numbering authority.

The EAN-8 barcode has 8 numeric-only characters, divided into three areas:

  1. The number system (2 or 3 digits)
  2. The product code (4 or 5 digits)
  3. The check digit (1 digit)

Unlike  the UPC-E barcode in which only 6 digits are explicitly encoded, the EAN-8 barcode explicitly encodes all eight digits.

In the system maintenance, enter 7 digits. Start with the first digit on the left (even if it is zero) and enter all following digits except the check digit (the last one). The system will automatically expand the UPC to 13 digits by adding six zeros (000000) at the beginning when the Execute button is clicked in the wizard.

GS1 Databar barcode

The GS1 Databar is a family of barcode symbologies. Some of them are small, intended for produce and small consumer packages. Some others are larger, intended to carry more data.

See the GS1 Databar barcode topic for details.
