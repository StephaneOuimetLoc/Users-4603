eSuite - Scale COOL Text Table

eSuite > Items > Item auxialaries

Information

The Scale COOL text table stores the texts that will be printed on the labels for the country of origin, the production method, etc.

Each COOL text contains a number to which an item is linked. This number can also be used in the Scale COOL short list table.

To link an item to a given COOL text, the Text and/or the Pre-text fields of the Scale information table are used, depending on whether the COOL text is a regular text or a pre-text. Typically, the pre-text is used to print production information in front of the COOL text.

NOTE: Even if the Scale COOL text table is an auxiliary table, its content is deployed to the front-end workstations using the "Deploy changes process".

Displaying the Scale COOL text table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Scale Tables and then on the Scale COOL Text: the Scale COOL Text tab opens in the same window, displaying the Scale COOL Text List.
  3. To modify an existing scale COOL text record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Scale COOL text table opens, overlapping the work area.
  4. To create a new scale COOL text record, click on Add in the bottom menu of the grid: the Scale COOL text table opens, overlapping the work area.

Content

Here is a description of the Scale COOL text table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier, i.e. the store group.
It is a required field used to uniquely identify records in the Scale COOL Text List.
The default value of the Target field is PAL.
Number The Number field contains the identifier of the scale COOL text information. This identifier is used to link scale information to scale COOL text information.
It is a required field used to uniquely identify records in the Scale COOL Text List.
S-Dept The Sub-department field links the scale COOL text information to a sub-department.
For details, see the Sub-department field section below.
OperLevel The Operator level field contains the number of the minimum operator level needed to manage the scale nutrifacts records.
Pre-text The Pre-text field specifies whether the scale COOL text record is a pre-text (checked) or not (unchecked).
The pre-text is typically used to print production information in front of the COOL text.
Font size The Font size field specifies the font size to use for the COOL text.
Descriptor The Descriptor field contains the description of the COOL text.

The Sub-department list

The Sub-department (S-Dept) links the scale COOL text information to a sub-department. 

In most POS systems, item sales are linked to sale categories for reporting purposes. Because different POS systems use different terminology for sale links, it is important to understand the LOC SMS eSuite terminology and how it relates to the POS system. The eSuite software uses a three-tier sale links level organization:

  1. Item sales are accumulated in sub-departments;
  2. Sub-department sales are accumulated in departments (a department sales total contains sales from one or more sub-departments);
  3. Department sales are accumulated in group sales (a group sales total contains sales from one or more departments).

The eSuite software sale link levels (sub-departments, departments and groups) are defined in the Sub-department and Department tables.

Whether or not the POS system uses the same terms, eSuite requires that items be linked to a sub-department or to a first sale link level.
