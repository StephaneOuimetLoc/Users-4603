The Advertising table contains the configuration of the different dynamic advertising slides that are displayed by eSuite.

The advertising slides use HTML to provide rich content. They can be downloaded from the internet. Any HTML pages with almost every Internet Explorer feature are supported by the advertising windows.

The advertising slides may be created from a template or using the eSuite HTML file designer (L_HTM.exe), accessed by selecting the Edit HTML templates option from the Execute pull-down menu of the SMS Pro application. To correctly fit in the eSuite default screen size of 1024 x 768 screen the advertising slides must be 1021 x 570.

Notes:
The default advertising file size is 1021 x 570, which is optimal for a 1024 x 768 screen. The screen size can be customized (see your reseller for more details).
You have to ensure the advertising page does not get focus as this would make the cursor go off the Entry window for every advertisement page that follows.
The HTML pages used as advertising slides can contain texts, videos and images. The images can be any format type supported in an HTML page (.jpg, .gif and .png images are preferred to .bmp images as these files are usally smaller in size.). Keep in mind that video files can often be quite large and may use up network bandwidth during loading.

The URL programmed are shown on the Screen saver, the Advertising window and the Customer window.

Displaying the Advertising table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Advertising: the Advertising tab opens, displaying the Advertising list.
  3. To modify an existing advertising, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Advertising table opens, overlapping the work area.
  4. To create a new advertising, click on Add in the list toolbar: the Advertising table opens, overlapping the work area.

Content

Here is a description of the Advertising table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier, i.e. the store group.
The default value is PAL.
When the advertising information is deployed from the head office, the target identifier specifies the stores to which it is deployed. 
Ordering name The Ordering name field contains the name or the description of the advertising slide.
Its value must be unique.
Duration The Duration field contains the time, in seconds, the advertising slide stays on the screen.
The default value is 10.
Form The Form parameter identifies which form to use for the advertising slide.
The default value is PUB, which is the main advertising form. If necessary, add section on how to change the advertising slide form.
URL The URL field contains the file name or the URL of the advertising slide to display.
The complete file name must be entered, including the extension.
For details, see the URL field section below.
Trigger type The Trigger type field contains the type of element (item, department, etc.) for which the advertising slide is automatically triggered.
It works in combination with the Trigger key field.
For details, see the Trigger type field section below.
Trigger key The Trigger key field contains the code of the element (item, department, etc.) for which the advertising slide is automatically triggered when it is sold.
It works in combination with the Trigger type field.
For details, see the Trigger key field section below. 

Note:

The URL field

The URL field contains the file name or the URL of the advertising slide to display. The complete file name must be entered, including the extension.

When URL have scripted content, the browser may return a scripting error. If the advertising slide is generated from a template, the file name is automatically entered by the system in the URL text box.

If no path is specified for the file, the system loads the file from the Storeman/Office/Htm directory. Make sure the file is located in that directory or specify the complete path. If Internet access is available, a valid URL may be entered and the system automatically loads the web page from the location specified by the URL.

The Trigger type field

The Trigger type field contains the type of element (item, department, etc.) for which the advertising slide is automatically triggered.

It works in combination with the Trigger key field. For example, when Trigger type = SDP and Trigger key = 3 (so Meat sub-department), the advertising slide is automatically triggered when an item of the Meat sub-department is sold.

Note:
The eSuite software needs to be restarted for the trigger to take effect.

The Trigger key field

The Trigger key field contains the code of the element (item, department, etc.) for which the advertising slide is automatically triggered when it is sold.

 

It works in combination with the Trigger type field. For example, when Trigger type = SDP and Trigger key = 3 (so Meat sub-department), the advertising slide is automatically triggered when an item of the Meat sub-department is sold.

Note:
The eSuite software needs to be restarted for the trigger to take effect.
