Release Notes NAVET

Important information that must be known before upgrading to version 4.5.0 of the LOC Software Management Suite (SMS).

System Requirements When upgrading to LOC SMS V4 (eSuite), the LOC SMS version 3.5.0 is required as a base software. This also means that all the necessary steps, security and changes done with version 3.5.0 also applies to LOC SMS V4.

Legacy

With the release of LOC SMS V4, some options from LOC SMS version 3.5.0 are not compatible anymore. Please make sure to review the list here below in order to find out which ones are not compatible anymore and to see what is replacing them.

Legacy options Not Compatible with LOC SMS V4

Option name Description Replacement
Device_PDA The PDA/Http option for PDA devices. LOC SMS mFloor
Device_EMU_R16X20 The RF/Telnet option for PDA devices. LOC SMS mFloor

Legacy Options Compatible With LOC SMS V4

  • Device_MK - The Price Checker option.

Upgrade Order

When upgrading to LOC SMS V4 within a store chain, it is possible to upgrade the stores first (including the lanes), and then the host, or the opposite as well.

To take advantage of the upgrading flexibility, the chain must be running at minimum the version 3.5.0 of SMS on the server and lanes.

NOTE: If the chain is using SMSPayment, it is mandatory to update the host first as ApiGenericIP cannot communicate with a service of a lower version.
