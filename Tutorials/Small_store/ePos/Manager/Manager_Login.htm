Information

Some operations of the selling screen need the supervision of the manager. For example; Refund, Price modification, Cancel transaction, Cancel previous, Delete all...etc. In this case, the "Please login with the right access level" message will be displayed in the entry window, indicating that the supervisor, usually the manager, must log in and supervise the operation.

To log in as manager on the front-end workstation (ePos), please follow the next steps:

First way:

  1. Log in as a Manager from the ePos main screen.
  2. "Login manager (date of login)"will be displayed
  3. Click on Pos to access the enter screen.
  4. You will now have access to all the manager functions.

Second Way:

  1. Log in as a cashier
  2. Select the Access option from the entry window; The "login access?" message will be displayed in the entry window.
  3. In the entry window, enter the manager login, and press the enter key or click on the E button. "Access login manager" will be displayed. 
  4. After you run a transaction, it will log back to the last cashier that logged in.

The "Access login [operator]" message is displayed in the entry window, indicating that the supervisor is now logged in. However note that the cashier will still be responsible for the transaction.



Note: By default, the Login access mode is valid for one scan of an item. For example; The return of one item, if scanning a second item, the manager access must be re-entered. The setting for the AutoManagerLogout is in system.ini > Company section, choices are to automatically log out the manager after one scan (checked) or at the end of the transaction (not checked).

Third way:

  1. Log into ePos. Click on Login and enter manager access code
  2. Cashier: Manager will be displayed on the entry window and any transactions made after that will be listed under the manager until he/she logs out and a cashier logs in after them.

Warning: The Login access function is also used to lock and unlock the operator. If the POS is locked by an operator a manager cannot login to the POS with the login access function. (This behaviour may be changed, please contact your reseller.)
