Module - SMSPayment Centralized Account Receivable - POS Operations

Bank_SMSPayment Unmanaged

Information

The ability to do AR transactions using SMSPAY allows us to manage the balance of the customer in SMSPayment.

Centralized account creation

Before being able to use a centralized A/R account, the first thing to do is to activate it to create a new account:

  1. In Option, select SMSPayment
  2. Select Centralized account creation (FCT 6130)
  3. Enter Card number

Centralized balance inquiry

Any time after a centralized AR account is created, it is possible to get its balance. The centralized balance inquiry will retrieve the account balance:

  1. In Option, select SMSPayment
  2. Select Centralized balance inquiry (FCT 6131)
  3. Enter Card number
  4. The bank stub should show the 'current balance'

Centralized payment

A centralized AR account can be used for a partial or complete payment of a sale. For configuration on partial payment and risk limits, please see the members document SMSPayment - Centralized Account Receivable:

  1. Begin the transaction and sell item(s)
  2. In Option, select SMSPayment
  3. Select Centralized payment (FCT 6132)
  4. Enter amount of payment to be applied to the account
  5. Enter Card number
  6. If a partial payment, select tender to be used to pay the balance

Centralized Adjustment

This function is used if adjustments need to be done to a centralized AR account.

  1. In Option, select SMSPayment
  2. Select Centralized adjustment (FCT 6133)
  3. Enter amount of adjustment
  4. Enter car number
  5. Select Tender (cash)
  6. Bank stub should show the adjustment amount and new balance

Centralized pre-auth

This function temporarily adds the pre-authorised amount to the account balance. The pre-authorization can either be completed (FCT 6136) or canceled by means of a timeout or by cancelling the order. For configuration on timeout, please see the members document SMSPayment - Centralized Account Receivable for information on:

  1. In Option, select SMSPayment
  2. Select Centralized pre-auth (FCT 6135)
  3. Enter amount of pre-auth

 Centralized completion

Following a pre-authorization, you can complete the pre-auth using the Centralized Completion as long as you have not surpassed the timeout configured or it has not been cancelled. Once the pre-auth is completed, the account balance will be updated with the actual charge amount instead of the pre-authorization charge amount.

  1. Start a transaction
  2. In Option, select SMSPayment
  3. Select Centralized payment (FCT 6136)
  4. Receipt should show updated balance with charged amount (not pre-auth amount) and will be type 'Pre-auth complete".
