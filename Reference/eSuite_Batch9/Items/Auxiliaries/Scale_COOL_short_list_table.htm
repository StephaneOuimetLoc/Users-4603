eSuite - Scale COOL Short List Table

eSuite > Items > Item auxialaries > Scale Tables

Information

The Scale COOL short list table is used to display popups on the scale to manually select the COOL text to be printed.

The short list of COOL texts is built in advance and then called from the scale. The short list can be called manually or automatically by linking an item to a short list number, so that when the item is weighed, a prompt automatically displays the content of its short list.

NOTE: Even if the Scale COOL short list table is an auxiliary table, its content is deployed to the front-end workstations using the "Deploy changes" process.

Displaying the Scale COOL short list table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Scale Tables and then on the Scale COOL Short List: the Scale COOL Short tab opens in the same window, displaying the Scale COOL Short List.
  3. To modify an existing scale COOL short list record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Scale COOL short list table opens, overlapping the work area.
  4. To create a new scale COOL short list record, click on Add in the bottom menu of the grid: the Scale COOL short list table opens, overlapping the work area.

Content

Here is a description of the Scale COOL short list table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier, i.e. the store group.
It is a required field used to uniquely identify records in the Scale COOL Short List.
The default value of the Target field is PAL.
For details, see the Target field section below.
Number The Number field contains the identifier of the scale COOL short list information. This identifier is used to call on the proper popup.
It is a required field used to uniquely identify records in the Scale COOL Short List.
Sequence The Sequence field contains the sequential number.
This sequential number works in combination with the Number field.
It is a required field used to uniquely identify records in the Scale COOL Short List.
S-Dept The Sub-department field links the scale COOL short list information to a sub-department.
For details, see the Sub-department field section below.
Text # The Text # field contains the related COOL text number. This text number is used to link the scale COOL short list information to a COOL text information.
OperLevel The Operator level field contains the number of the minimum operator level needed to manage the scale COOL short list records.

The Target field

The Target field contains the target identifier, i.e. the store group. It is a required field, meaning that new COOL short list records must have a valid entry in the Target field for the record to exist. It is used as a key field, along with the COOL short list number and the COOL short list sequence to uniquely identify records in the Scale COOL Short List table. The default value of the Target field is PAL.

The eSuite software is designed to support multi-stores and regional pricing. The Target field is grayed out and not editable unless the system is set up for multi targets.
See the reseller for more details.

Target identifiers can be modified in a multi-stores setting by typing the right information in the Target field or by selecting a target identifier in the drop-down list.

A warning message is displayed when an invalid target identifier is entered.

The Sub-department field

The Sub-department (S-Dept) links the scale COOL short list  to a sub-department.

In most POS systems, item sales are linked to sale categories for reporting purposes. Because different POS systems use different terminology for sale links, it is important to understand the eSuite software terminology and how it relates to the POS system. The eSuite software uses a three-tier sale links level organization:

  1. Item sales are accumulated in sub-departments;
  2. Sub-department sales are accumulated in departments (a department sales total contains sales from one or more sub-departments);
  3. Department sales are accumulated in group sales (a group sales total contains sales from one or more departments).

The eSuite software sale link levels (sub-departments, departments and groups) are defined in the Sub-department and Department tables.

Whether or not the POS system uses the same terms, eSuite requires that items be linked to a sub-department or to a first sale link level.

To link the scale COOL short list information to a sub-department, type the sub-department number in the S-Dept field or select a sub-department from the Sub-department table.
