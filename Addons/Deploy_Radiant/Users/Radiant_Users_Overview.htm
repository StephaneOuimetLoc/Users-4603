Radiant Interface - Users Guide Overview

Radiant Unmanaged

Information

The Deploy_Radiant option is used to communicate with the Radiant interface. These documents assume that the operator performing maintenance is familiar with the following concepts and software:

  • Radiant Interface
  • Multi Target (Host)
  • Auxiliary deployment
  • Item deployment
  • Mix&Match deployment
  • Combo deployment
  • Customer Maintenance
  • Operator Maintenance

Requirements

  • SMS Https service (32-bit or 64-bit depending on the OS of the system, Install\SMS v4)

Table Maintenance

The maintenance of tables is done on the LOC SMS back office Server and deployed to the Radiant back office Server. The Radiant back office Server takes care of the Radiant lanes. Essentially, the Radiant back office Server controls the registers' hardware and network. LOC SMS manages all tables needed to run the POS transactions.
