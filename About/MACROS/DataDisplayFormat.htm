Data display formats & text operation

This section describes data display formats and operations to apply to text before formatting.

Data display format depends on the type of data to be displayed.
Here is the list of data display formats:

<TextDisplayFormat>
<NumericDisplayFormat>
<DateDisplayFormat> 		To format a string.
To format a string representing a number.
To format a string representing a date or a time.

<DataDisplayFormat> represents all the different formats.

Data display format will be used in several macros like :

@dbRow(<ResultSetId>,<FieldName>,<DataDisplayFormat>,<TotalId>)
@dbHot(<FileName>,[<Section>]<Parameter>,<TextDisplayFormat>)
@dbHot(REG,<RegistryKey>,<TextDisplayFormat>,<DefaultValue>)

A text operation is used to manipulate text and it can be used in a macro or combined with a <TextDisplayFormat>.
Here is a the list of text operation :

<TextOperationAlign>
<TextOperationMultiLines>
<TextOperationSubString>
<TextOperationFind>
<TextOperationDate>
<TextOperationTime>
  

For example, a text operation can be used in a macro like @Fmt(<TextOperationAlign>,<Text>) and in all macros using a <DataDisplayFormat> or a <TextDisplayFormat>

TextDisplayFormat: <TextFormat>-<TextOperation>

Format for displaying a text value. A predefined <TextOperation> can be combined to the <TextFormat>.
Separate <TextFormat> and <TextOperation> with a dash if both are present.

This text display format can be used in a macro like @dbHot(<FileName>,[<Section>]<Parameter>,<TextDisplayFormat>) and in all macros using a <DataDisplayFormat>.

Parameter Description Required/Optionnal
TextFormat
RAW
MACRO
TXT



empty
 Text as is. Without transformation. The macros in the text are parsed.
Text as is. Without transformation. The macros in the text are NOT parsed.
Substitution of the following characters:
( by [
) by ]
The macros in the text are parsed.
If the format is empty then text is encoded in HTML by replacing:
< by &lt;
> by &gt;
& by &amp;
( by [ and ) by ] for macro parentheses. i.e. @WizGet(A) will become @WizGet[A]
Optionnal
Default:empty

<TextOperationAlign>
<TextOperationMultiLines>
<TextOperationSubString>
<TextOperationFind>
<TextOperationDate>
<TextOperationTime>		 Align a text value.
Extract a string in a multi lines text
Extract substring from a string.

Format a time from HHMMSS format
Optionnal

Examples
Key x of section [100] content <AA>@WizGet(V1);ZZ
@dbHot(popup.ini,[100]x,Macro)    will return <AA>@WizGet(V1);ZZ
@dbHot(popup.ini,[100]x,RAW)      will return <AA>ZZ
@dbHot(popup.ini,[100]x,TXT)       will return <AA>@WizGet[V1];ZZ
@dbHot(popup.ini,[100]x)             will return &lt;AA&gt;@WizGet[V1];ZZ

@dbHot(popup.ini,[100]x,RAW-R2) will return ZZ

@dbHot(popup.ini,[100]x,RAW-R20:*) will return ********Loc Software
@dbHot(popup.ini,[100]x,RAW-L20:*) will return Loc Software********

NumericDisplayFormat: <SpecialFormat><NumericFormat><LengthInCharacter>:<NumberOfDecimal>

Format for displaying a numeric value.

The <NumericDisplayFormat> can be used with some macros :
   @FMT(<NumericDisplayFormat>,<Value>)
   @FMT(COMA,<ParamName>,<NumericDisplayFormat>,<TotalId>,<ParamList>) 
   @dbRow(<ResultSetId>,<FieldName>,<DisplayFormat>,<TotalId>)
   @dbFind(<ResultSetId>,<SearchFieldName>,<SearchValue>,<ReturnFieldName>,<DisplayFormat>,<TotalId>)
   @dbRow(<ResultSetId>,FIELD,<ColumnIndex>,<DisplayFormat>)

Parameter Description Required/Optionnal
SpecialFormat Z      Return empty string if value equal zero.
RAW  Value as is. Without transformation.		 Optionnal
NumericFormat $    Return the value as amount starting with a dollar sign .
#    Return the value as quantity.
%   Return the value as percentage ending with a percent sign.		 Required if SpecialFormat<>RAW
LengthInCharacter Maximum length for the formatted value (including $ or % if present).
The value is aligned to the right side.		 Required if SpecialFormat<>RAW
NumberOfDecimal Number of decimal of the formatted value. Required if SpecialFormat<>RAW
Examples
@FMT(Z$10:2,0.0)       will return ""
@FMT($10:2,10.253)    will return " $10.25"
@FMT(#10:2,10.253)    will return " 10.25"
@FMT(%10:2,10.253)   will return " 10.25%"
@FMT(Z%10:2,10.253) will return " 10.25%"

<TextOperationAlign>:   <Alignment><Length><:PadCharacter>

Operation on text to align a text value.

Parameter Description Required/Optionnal
Alignment
L

R

C
 Text is left-justified
If the length of the text is larger than <Length> then the text will be truncated on the right.
Text is right-justified
If the length of the text is larger than <Length> then the text will be truncated on the left.
Text is centered
<PadCharacter> is not used for center alignement. A space character is always used for padding.
A trim right is done after centering the text.
If the length of the <Text> is larger than <Length> then the text will not be truncated. All the <Text> will be returned.
Required
Length Number of characters of the returned text in which. Optionnal
Default: All characters
:PadCharacter The character use to pad left or right. Default padding character is the space character.
Use SPACE for HTML space character: &nbsp;		 Optionnal
Default: Blank
Examples
@dbHot(VARSET,Key,One Two)

@dbHot(REG,Key,RAW-L3)        will return One
@dbHot(REG,Key,RAW-L)         will return One Two
@dbHot(REG,Key,TXT-L10:#)   will return One Two###

@dbHot(REG,Key,R3)               will return Two
@dbHot(REG,Key,RAW-R10:#)  will return ###One Two

@dbHot(REG,Key,C15)             will return "    One Two"  

<TextOperationMultiLines>:   M<LineNumber>:<MaxLength>

Operation on text to extract a string in a multi lines text. Formatting with this concept will allow to word-wrap the strings and insure that none of the words will be cut.
The formatting text support an unlimited number of lines.

Parameter Description Required/Optionnal
LineNumber Line number based on lines of maximum <MaxLength> characters. Required
MaxLength Maximum length of the text to return. Required
Examples
@WizSet(TEXT=Computer programming is a way of telling a computer how to do certain things.);
@Fmt(M1:25,@WizGet(TEXT)) will return Computer programming is
@Fmt(M2:25,@WizGet(TEXT)) will return a way of telling a
@Fmt(M3:25,@WizGet(TEXT)) will return computer how to do 

@dbHot(VARSET,Key,Computer programming is a way of telling a computer how to do certain things.)
@dbHot(REG,Key,RAW-M2:25) will return a way of telling a

<TextOperationSubString>:   S<Index><:Length>

Operation on text to extract substring from a string.

Parameter Description Required/Optionnal
Index Starting character position in <Text>. Required
:Length Number of characters to return.
If no length is specified, all text from the index will be returned.		 Optionnal
Examples
@Fmt(S5:4,Loc Software)   will return  "Soft"
@Fmt(S5,Loc Software))    will return  "Software"

<TextOperationFind>:   F:<Name>

Operation on text to retrieve text value from a list of key-value.

Each key-value can be separated by character ampersand (&) or a comma(,) or a space.
To use one of the following characters: ampersand (&) comma (,) space in the value; you must set the value between quotation marks. Ex : 'One&Two and Three'
At the beginning of the name, the following characters will not be considered: backslash (\) dash (-)

Parameter Description Required/Optionnal
Name Name of the key whose value will be returned. Required
Examples
@WizRpl(KEY="/Item1=AA /Item2=BB /Item3=CC");
@Fmt(F:Item1,@WizGet(KEY)) will return AA
@Fmt(F:Item2,@WizGet(KEY)) will return BB
@Fmt(F:Item3,@WizGet(KEY)) will return CC

KEY Registry contains -Item1='A,&B',/Item2='BBB BB'&Item3=CCC
@dbHot(REG,Key,RAW-F:Item1) will return A,&B
@dbHot(REG,Key,F:Item2)        will return BBB BB
@dbHot(REG,Key,F:Item3)        will return CCC

@Fmt(F:PIN,PIN=1 CARDTPYE=D)            will return 1
@Fmt(F:CARDTYPE,PIN=1 CARDTYPE=D)   will return D
