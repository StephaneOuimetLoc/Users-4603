eSuite - Scale COOL Country List Table

eSuite > Items > Item auxialaries > Scale Tables > Scale COOL Country

Information

The Scale COOL Country List table is used to declare the country of origin for the products.

It is not possible to link an item to a specific COOL country list. Instead, the COOL country lists are set with a given class, and that class is linked to items through the Scale information table. The scale should then display all country lists linked to that class.

NOTE: Even if the Scale COOL country list table is an auxiliary table, its content is deployed to the front-end workstations using the "Deploy changes" process.

Displaying the Scale COOL country list table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Auxiliary tables: the Start window opens in the Auxiliary work area.
  2. Click on Scale Tables and then on the Scale COOL Country List: the Scale COOL Country tab opens in the same window, displaying the Scale COOL Country List.
  3. To modify an existing scale COOL country list record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit command in the bottom menu: the Scale COOL country list table opens, overlapping the work area.
  4. To create a new scale COOL country list record, click on Add in the bottom menu of the grid: the Scale COOL country list table opens, overlapping the work area.

Content

Here is a description of the Scale COOL Country List table fields and their functionalities:

FIELD DESCRIPTION
Target The Target field contains the target identifier, i.e. the store group.
It is a required field used to uniquely identify records in the Scale COOL country list.
For details, see the Target field section below.
Number The Number field contains the identifier of the scale COOL country list information.
It is a required field used to uniquely identify records in the Scale COOL country list.

Description The Description field contains the description of the COOL country list.

The Target field

The Target field contains the target identifier, i.e. the store group. It is a required field, meaning that new COOL production list records must have a valid entry in the Target field for the record to exist. It is used as a key field, along with the COOL country list number, to uniquely identify records in the Scale COOL country list table. The default value of the Target field is PAL.

The LOC SMS eSuite software is designed to support multi-stores and regional pricing. The Target field is grayed out and not editable unless the system is set up for multi targets.
See the reseller for more details.

Target identifiers can be modified in a multi-stores setting by typing the right information in the Target field or select a target identifier in the drop-down list.

A warning message is displayed when an invalid target identifier is entered.

The Sub-department field

The Sub-department (S-Dept) field links the scale COOL country list information to a sub-department.

In most POS systems, item sales are linked to sale categories for reporting purposes. Because different POS systems use different terminology for sale links, it is important to understand the eSuite terminology and how it relates to the POS system. The eSuite software uses a three-tier sale links level organization:

  1. Item sales are accumulated in sub-departments
  2. Sub-department sales are accumulated in departments (a department sales total contains sales from one or more sub-departments)
  3. Department sales are accumulated in group sales (a group sales total contains sales from one or more departments)

The eSuite software sale link levels (sub-departments, departments and groups) are defined in the Sub-department and Department tables.

Whether or not the POS system uses the same terms, eSuite requires that items be linked to a sub-department or to a first sale link level.

To link the scale COOL country list information to a sub-department, type the sub-department number in the S-Dept field or select a sub-department from the Sub-department table.
