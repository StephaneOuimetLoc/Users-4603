eSuite - Printed Coupons Examples

eSuite > Items > Master > Coupon list

Information

Printed coupons are store and vendor coupons given by the system when the specified criteria are met.

Here are some examples of how to create printed coupons.

List of examples

  1. Buy 7 bags of Oreo cookies, get an extra 3 for free
  2. Buy for $20.00 in the Produce sub-department, get $1.00 off and 2 additional points in the main point program (point program 1) if the customer is shopper level 3
  3. Buy for a total of $100.00 (excluding tobacco and alcohol purchases) in the transaction, get $15.00 off
  4. Buy 2 items of the Grocery sub-department, get 15% off on these 2 items; buy 4 items of the Grocery sub-department, get 20% off on these 4 items

Buy 7 bags of Oreo cookies

Get an extra 3 for free

 1. Add the electronic coupon criteria in the main section of the coupon wizard.
  Value field: select Buy X get Y free. 
  Buy X field: enter 10. 
  Get Y field: enter 3.
  Coupon type field: select Printed. 
  Description field: add a meaningful description. 
  Sub-dpt. field: select a sub-department. 
 
 
2. In the Criteria tab, click Add criteria to add the coupon criteria.
  Type field: select Item sales. 
  Equal to field: enter the Oreo bag UPC. 
  Minimum measure field: select Quantity. 
  Minimum value field: enter 10. 
  Select Make sure the discount affects the item(s) in these criteria.
This method forces the system to apply the printed coupon discount on the Oreo bags. 
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
3. In the Details tab: 
  Funding field: select Store or Vendor.
  Coupon template field: click and select the coupon format.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
 
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code.
 
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • If the customer buy 6 bags of Oreo cookies, they will not get any discount.
  • If thye buy 10 bags of Oreo cookies, they will get 3 of them for free.
  • If they buy 15 bags of Oreo cookies, they will get 3 of them for free.
  • If they buys 22 bags of Oreo cookies in the same transaction, he will get 3 of them for free, etc.
  • For the customer to get 6 bags of Oreo cookies for free when they buy 20 bags of Oreo cookies in the same transaction, add Based on validated criteria (R) in the Multiplication and accumulation field of the Add criteria window. In this case, 3 bags of Oreo cookies will be given for free for each 10 bags of Oreo cookies bought by the customer in the same transaction.

Buy for $20.00 in the Produce sub-department

Get $1.00 off and 2 additional points in the main point program (point program 1) if the customer is shopper level 3 

 1. Add the electronic coupon criteria in the main section of the coupon wizard.
  Value field: select Cents off. 
  Discount field: enter 1. 
  Coupon type field: select Printed. 
  Description field: add a meaningful description. 
  Sub-dpt. field: select the Produce sub-department. 
 
 
2. In the Criteria tab, click Add criteria to add the first criteria.
  Type field: select Customer level. 
  Equal to field: enter the customer level. 
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
2. In the Criteria tab, click Add criteria to add the second criteria.
  Type field: select Sub-department. 
  Equal to field: enter the produce sub-department number. 
  Minimum measure field: select Dollar. 
Minimum value field: enter 20. 
  Select Make sure the discount affects the item(s) in these criteria.
This method forces the system to apply the printed coupon discount on the items in the Produce sub-department.
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
3. In the Details tab: 
  Funding field: select Store or Vendor.
  Coupon template field: click and select the coupon format.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
  Bonus field: enter 2. 
  Point program field: enter 1. 
 
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code.
 
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The shopper level 3 customer gets $1.00 off and 2 points in the main point program when they buy for $20.00 in the Produce sub-department in the transaction.
  • Even if the shopper level 3 customer buys for $40.00 in the Produce sub-department, they will only get $1.00 off and 2 points in the main point program.
  • To repeat the electronic coupon, add the Multiply using criteria method (R) to the SUB criterion. Then the shopper level 3 customer will get $2.00 off and 4 points in the main point program when they buy for $40.00 in the Produce sub-department; $3.00 off and 6 points in the main point program when he will buy for $60.00 in the Produce sub-department, etc.

Buy for a total of $100.00

 (excluding tobacco and alcohol purchases) in the transaction, get $15.00 off .

 1. Add the electronic coupon criteria in the main section of the coupon wizard.
  Value field: select Cents off. 
  Discount field: enter 15. 
  Coupon type field: select Printed. 
  Description field: add a meaningful description. 
  Sub-dpt. field: select a sub-department. 
 
 
2. In the Criteria tab, click Add criteria to add the first criteria.
  Type field: select Total sales. 
  Totalizer field: enter 3. 
  Minimum measure field: select Dollar. 
  Minimum value field: enter 100. 
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
3. In the Criteria tab, click Add criteria to add the second criteria.
  Type field: select Sub-department. 
  Equal to field: enter the beer sub-department number. 
  Multiplication and accumulation field: select Exclude this from accumulation.
This method forces the system to exclude any sale from the beer sub-department from the required $100.00 (X).
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
4. In the Criteria tab, click Add criteria to add the third criteria.
  Type field: select Sub-department. 
  Equal to field: enter the wine sub-department number. 
  Multiplication and accumulation field: select Exclude this from accumulation.
This method forces the system to exclude any sale from the wine sub-department from the required $100.00 (X).
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
5. In the Criteria tab, click Add criteria to add the fourth criteria.
  Type field: select Sub-department. 
  Equal to field: enter the tobacco sub-department number. 
  Multiplication and accumulation field: select Exclude this from accumulation.
This method forces the system to exclude any sale from the tobacco sub-department from the required $100.00 (X).
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
6. In the Details tab:
  Funding field: select Store or Vendor.
  Coupon template field: click and select the coupon format.
  Event type field: select Sale.
  Enter a Start date.
  Enter an End date.
 
 
7. Add any necessary description in the Descriptions tab.
 
8. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code.
 
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The customer gets $15.00 off when they buy for at least $100.00 in the transaction.
  • If the customer buys any tobacco or alcohol (beer and/or wine) item, they must buy for an additional amount in order to get the discount, since the amount of these items does not count in the required $100.00 minimum purchase.

NOTE: For the discount be added as a new entry in the transaction instead of being applied on an item, add the WIT=NEWREG or WIT=NEWCPN parameter in the Registration type field.

Buy 2 items of the Grocery sub-department

Get 15% off on these 2 items; buy 4 items of the Grocery sub-department, get 20% off on these 4 items

This example includes two printed coupons that work identically (buy a number of items in the same sub-department and get a certaint percentage off on these items).

However, the required quantities differ from one coupon to the other, and therefore the discount percentage is different.

The two printed coupons may exclude each other: when the customer buys 4 items of the Grocery sub-department, only 1 of these 2 discounts should be given, namely the 20% off on the 4 items. To get both discounts, the customer should buy 6 items of the Grocery sub-department.

Create the first printed coupon (2 items, 15% off)

 

 1. Add the electronic coupon criteria in the main section of the coupon wizard.
  Value field: select Percent. 
  Discount field: enter 15. 
  Coupon type field: select Printed. 
  Description field: add a meaningful description. 
  Sub-dpt. field: select the Grocery sub-department. 
 
 
2. In the Criteria tab, click Add criteria to add the first criteria.
  Type field: select Sub-department. 
  Equal to field: enter 1. 
  Minimum measure field: select Quantity. 
  Minimum value field: enter 2. 
  Multiplication and accumulation field: select Accumulate for bundle or percent.
This method calculates the percentage discount on the 2 items of the Grocery sub-department.
  Select Make sure the discount affects the item(s) in these criteria.
This method forces the system to apply the printed coupon discount on the items in the Grocery sub-department.
  Select The item(s) in this condition can only participate in one offer.
This method limits the participation of the 2 items to this electronic coupon only.
  Add a Manual description (optional).
  Click OK to confirm the changes.  
 
 
3. In the Details tab:
  Funding field: select Store or Vendor.
  Coupon template field: click and select the coupon format.
  Event type field: select Sale.
  Enter a Start date.
  Enter an End date.
 
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code.
 
  The Criteria tab will display the following information (Text view and Data view).

  Select Make sure the discount affects the item(s) in these criteria.
This method forces the system to apply the printed coupon discount on the items in the Grocery sub-department.
  Select The item(s) in this condition can only participate in one offer.
This method limits the participation of the 4 items to this electronic coupon only.
  Add a Manual description (optional).

3. In the Details tab:
  Funding field: select Store or Vendor.
  Coupon template field: click and select the coupon format.

4. Add any necessary description in the Descriptions tab.

In this example:

  • If the customer buys 1 item of the Grocery sub-department, they will not get any discount.
  • If they buy 2 items of the Grocery sub-department, they will get 15% off on these 2 items.
  • If the customer buys 3 items of the Grocery sub-department, they will get 15% off on 2 items.
  • If they buy 4 items of the Grocery sub-department, they will get 20% off on these 4 items
  • If the customer buys 5 items of the Grocery sub-department, they will get 20% on 4 items.
  • If they buy 6 items of the Grocery sub-department, they will get 20% on 4 items and 15% on the 2 others
  • Iif the customer buys 7 items, they will get 20% on 4 items and 15% on 2 others.
  • If they buy 8 items, they will get 20% on 4 items and 15% on 2 others, etc.

Since the items of the Grocery sub-department have different prices, an average price is defined by the system, which is then multiplied by the quantity of items required (2 for the 15% coupon and 4 for the 20% coupon) to determine the amount from which the percentage discount is calculated.
