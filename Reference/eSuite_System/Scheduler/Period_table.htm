eSuite System - Period Table

eSuite System > Scheduler > Periods

Information

The Period table defines the different periods (fiscal and regular periods) of the store. Periods are used in reports.

The Period table requires a high security level access (owner and programmer).

  • By default, the period types are not displayed in the Period report parameters. See the Application.ini file documentation for more details.

Displaying the Period table

In the Launcher, click the Scheduler entry:

  1. The Start window opens in the Scheduler work area.
  2. Click on Periods: the Periods grid opens in the List tab.
  3. To modify an existing record, select it in the grid and double-click it, press the F3 key on the keyboard or select Edit in the bottom menu: the Period edit table will open, displaying the existing information related to that period.
  4. To create a new record, click Add in the toolbar: the Period table will open for editing.

Content

Here is a description of the fields and their functionalities in the Period table:

FIELDS DESCRIPTION
Target The Target field contains the target identifier (the store group).
It is a required field.
The default value is PAL.
Type The Type field contains the type of period (Period or Fiscal year).
Start date The Start date field contains the date at which the period starts.
The starting date is inclusive.
End date The End date field contains the date on which the period ends.
It is a required field.
The ending date is inclusive.
Number The Number field contains the number of the period, according to the type of period defined.
Description The Description field contains the description of the period.
