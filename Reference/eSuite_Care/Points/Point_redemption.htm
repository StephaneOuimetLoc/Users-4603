eSuite Care - Point Programs

eSuite Care > Points

Information

The eSuite software provides different methods to redeem points directly from your points of sale on up to 35 point programs.

The points may be used (redeemed) to get discounts on items or items for free, as counters to the nth item for free, to get special discounts when a certain amount of purchases has been reached by the customer, etc. The discount given using points is called "point discount".

The system does not need a customer to be entered in the transaction to give and redeem points. In this case, the points given in a transaction must be used (redeemed) in that transaction since they cannot be added to the customer's point balances. The unused points are lost.

The same thing occurs when a customer is entered in the transaction, except that the unused points are added to the point balances of the customer, or more points may be used (redeemed) if the customer already has points in the point balances.

Point programs

The system can manage up to 35 point programs, identified by a digit from 1 to 9 or a letter from A to Z. For example, the letter A corresponds to point program 10, B corresponds to point program 11 and so on up to Z that corresponds to point program 35.

Point program 1 is considered the main one, all the others being secondary point programs.

In the Shopper level table, the Point programs field is used to identify the point programs that the customers linked to the shopper level are members of. The field allows entering or selecting multiple point programs, with no space in between.

The way the points are redeemed for the main point program 1 is defined in the Shopper level table and may be different for each shopper level. The way the points are redeemed for the secondary point programs (2 to Z) is described in the System.ini file, since it is identical for every shopper level.

Redemption methods

The points can be redeemed according to three different point redemption methods:

  • Preset point redemption
  • Variable point redemption
  • Point redemption on functions

The point redemption can only be done according to one point program at a time.

Preset point redemption

The Preset point redemption method uses the point redemption values entered in the Discount points and Free points fields of the Price table. The point discount given corresponds respectively to the amount defined in the Discount amount field, or the item for free. The Program field is used to identify the point program from which the points are redeemed.

The number of redeemed points needed is calculated by multiplying the active point value with the quantity of items sold.

To set up the Preset point redemption:

  1. Open the Shopper level table for the right shopper level.
  2. Select the Allow redeem on item point method in the Redeem method field.
    The displayed value is 1.

  3. Save the changes.
  4. Open the Price table for the selected item.
  5. To give a rebate on the item, enter the number of points needed in the Discount points field and the amount of the rebate in the Discount amount field.
  6. To give the item for free, enter the number of points needed in the Free points field.
  7. In the Program field, select the point program on which the points are redeemed.

Repeat these steps for all the items needed.

Variable point redemption

The Variable point redemption method uses a rate to calculate the point discount based on a number of points.

The amount of the point discount given is calculated by dividing the number of points used (redeemed) by the point redemption rate. If an item has a point redemption defined in the Price table and the Preset point redemption method is activated in the Shopper level table, the system uses the point redemption values instead of the point redemption rate for that item.

For the main point program

The Red/penny field of the Shopper level table is used to define the point redemption rate for the main point program (point program 1).

For the secondary point programs

For the secondary point programs (2 to Z), the rates are defined in the System.ini file, under the Points section.

Here is the parameter to define:

REDEEMx_RATE=

[rate]

where x is the identifier character that identifies the point program.
The rate corresponds to the number of points needed to get 1 penny point discount.

Point redemption on functions

The Point redemption on functions method uses the REDEEM alpha parameter to define the point redemption rate of the functions.

When the function is used, the points are redeemed according to the value of the REDEEM alpha parameter defined in the function.

Here are its different values:

Parameter Description

REDEEMx=Y

where x is the identifier character that identifies the point program. If x is not specified, it is applied to point program 1.
The system redeems points according to the point redemption rate defined in the Shopper level table (Reduction/penny field).

REDEEMx=##

where x is the identifier character that identifies the point program. If x is not specified, it is applied to point program 1.
The system redeems points according to the rate ##. The system takes the amount of the registration to calculate the number of points that must be redeemed.

REDEEMx=##:$$

where x is the identifier character that identifies the point program. If x is not specified, it is applied to point program 1.
The system redeems points according to the rate ##. The point discount associated the rate (##) to the amount ($$).
For example: 10:1.00 means that 10 points are redeemed for a point discount of $1.00. So, if an item is sold at $2.60, 26 points are redeemed for a point discount of $2.60

REDEEMx=F##

where x is the identifier character that identifies the point program. If x is not specified, it is applied to point program 1.
The system redeems a fixed number of points (##).

REDEEMx=F##:$$

where x is the identifier character that identifies the point program. If x is not specified, it is applied to point program 1.
The system redeems a fixed number of points (##) for a specified point discount ($$).

The REDEEM alpha parameter can be used to reduce the use of a function, like a tender, since it costs points to use that function.

NOTE: If the customer does not have enough points, the function cannot be used.

Restrict the point redemption

Some restrictions may be added to the point redemption. These restrictions are:

  • Minimum points needed
  • Restricted to a period
  • Limitations on the REDEEM alpha parameter

Minimum points

The point redemption may be restricted to a minimum point condition. If the customer does not have a minimum number of points in their point balance, the point discounts are not given.

The minimum point condition does not concern the point discounts given on the preset point redemption.

To set up a minimum point condition:

  1. Open the Shopper level table for the right shopper level.
  2.  In the Minimum points field, enter the minimum number of points that the customers must have accumulated for the system to allow a point discount.

Period

The number of points redeemed can also be restricted to a period. The point discounts are given only during a specific period.

The restriction periods only concern the point discounts given on the preset point redemption.

To set up a restriction period:

  1. Open the Price table for the right item.
  2. Enter the starting date of the period in the Start (Redeem) field.
  3. Enter the ending date of the period in the End (Redeem) field.

Limitations on the REDEEM alpha parameter

It is possible to set some limitations when using the REDEEM alpha parameter to define the point redemption rate. These limitations are specified as suffixes to the rate value.

Here are the different suffixes values:

Value Description

;Rxxx

Redeems points based on the specified rate to a maximum of xxx points redeemed per item or entry.

;Qxxx

Redeems points based on the specified rate to a maximum of xxx units or weight.

;Axxx

Redeems points based on the specified rate to a maximum of xxx dollars per item or entry.

;Pxx

Redeems points based on the specified rate to a maximum of xx percent of the item sale.

Decrease points

The number of points redeemed can be divided by a factor. For example, the divisor can be used to reduce to half the number of points needed for a point discount.

To divide the number of points redeemed:

  1. Open the Shopper level table for the right shopper level.
  2. In the Multiplier (redeem) field, enter the value of the divisor redemption factor.

Point expiration

For the main point program 1 only, it is possible to expire points that have not been used after a certain time. Two steps are necessary: determining the points that are to be expired and establishing how to expire them.

Determining the points that are to be expired

The point expiration is determined by a date defined by a number of days. This number of days is specified in the Exp days field of the Shopper level table.

A different number of days may be defined for each shopper level.

Expiring points

The point expiration is done using a script. A scheduler job can be created for the expiration to run automatically. All unused points at the expiration date (today's date - number of days specified) are expired, meaning that an adjustment transaction is performed.

NOTE: Before expiring points, it is important to run the Calculate cumulative job and the Compare points calculated vs Current balance report.

How to redeem points

The point redemption can be done manually or automatically. The system behaves differently, depending on the point redemption method used.

Manual point redemption

To manually perform a point redemption, the Point redemption function must be manually called for each item by selecting the Point redemption option in the POS (the eSuite Sell module). Make sure the cashiers have access to that function.

If points are redeemed based on the Preset point redemption method, the number of points redeemed and the point discount value come from the Price table. If a rebate and a free value are defined for the item, the system prompts the cashier to know which one to use. The points are redeemed in the point program specified in the Program field. The point discount decreases the item amount.

If points are redeemed based on the Variable point redemption method, the system prompts the cashier for the number of points to redeem. Then, the system calculates the point discount given according to the number of points entered and the point redemption rate defined in the Shopper level table. Based on this point redemption method, the Point redemption function redeems points only on point program 1. To redeem on another point program, a new function must be created. The point discount decreases the item amount.

Automatic point redemption

When the automatic point redemption is set up, the Point redemption function is automatically called by the system each time an item is sold.

In the Shopper level table, select Automatic redeem in the Redeem field. Then, in the System.ini file, specify the point programs for which the auto-redeem process has to be called automatically.

When the automatic point redemption is set up, the system first uses the redemption information defined in the Price table. If no redemption information is set up in the Price table, the system will use the defined point redemption rate. If no point redemption rate is defined, the system does not automatically give any point discount on the item.

Redemption information in the Price table

The corresponding point discount defined in the Price table is automatically given when the item is sold if the customer has enough points. If a discount and a free value are defined for the item, the system automatically gives the best point discount (item for free or redemption discount) according to the number of points the customer has.

For example, an item gives a discount of $1.00 for 100 points or the item for free for 400 points. If the customer has 600 points, they will automatically have the item for free (and 400 points will be redeemed).
If the customer has 200 points, they will automatically have a point discount of $1.00 (and 100 points will be redeemed).
If the customer has 75 points, they will not have any point discount.

The point discount decreases the item amount. The points are redeemed in the point program defined in the Program field.

Point redemption rate

The maximum point discount is automatically given when an item is sold according to the number of points the customer has and the point redemption rate defined (if there is no point discount defined in the Price table).

For example, the price of an item is $2.00 and the point redemption rate is 10. If the customer has 3000 points, they will automatically have a point discount of $2.00 (thus the item for free, and 2000 points will be redeemed). If the customer has 750 points, they will automatically have a point discount of $0.75 (and 750 points will be redeemed), etc.

The point discount decreases the item amount. The points are redeemed in the point program used.

NOTES:
The points redeemed on functions (like tenders) are automatically redeemed when the functions are called.
A point discount cannot be given more than once on an item (or an entry).
If the REDEEMx parameter is set up in the URL field of the POS table for an item, points are redeemed automatically on this item even if the automatic point redemption is not set up.

Customer

The points redeemed are subtracted from the point balances of the customer entered for each transaction. The Point redemption is allowed if the customer has enough points. If points are given even if no customer is entered in the transaction, the Point redemption has to be done in the same transaction since the unused points cannot be kept in a point balance. If they are not used, they are lost.

If the customer is entered at the end of the transaction, the system will automatically reset the points redeemed for all items, sub-department sales and department sales. However, this does not apply to points redeemed using a function (with the REDEEM parameter). In this case, the cashier needs to void the operation and reperform it. So the best way to proceed when points are redeemed using a function is to enter the customer before the function is used.

Receipt

The points redeemed and the point balances are automatically printed on the receipt for all point programs the customer is member of. The information is also available in the Electronic journal.

Reports

The eSuite software has some reports related to points, like the point balance at a certain date, point movement, etc. These reports are related to the main point program (1), except for the point balance that shows the current point balance of all point programs at the specified date.

NOTE: Since the secondary point programs are normally used to count items and give the nth item for free or for special promotions like the Pampers program, there is no need for a statement on the points for these point programs, neither explicit details on the points given and redeemed.

Customers

The system no longer needs a customer to be entered in the transaction to give and redeem points. In this case, the points given in a transaction must be redeemed in that transaction, since they cannot be added to the customer's point balances. The unused points are lost.

The way the system works when no customer is entered in the transaction is that the points given on the previous items can be redeemed on the current item. This means that the points given on an item cannot be redeemed on it. As soon as a second item is sold, the points given by the first item can be redeemed on this second item, and so on. However, if points are redeemed when a function is used, like a tender, the points given by all items sold can be redeemed, including the last one.

In an automatic point redemption mode, the points given by the first item are automatically redeemed for a point discount on the second item, the points given by the second item are automatically redeemed for a point discount on the third item and so on , until the end of the transaction.

The same thing occurs when a customer is entered in the transaction, except that the unused points are added to the point balance of the customer, or more points may be redeemed if the customer already has points in their point balance.

Customer change

If points are redeemed for point discounts and the cashier changes the current customer entered in the transaction, the system automatically resets the point discounts that affect the items, the sub-department entries and the department entries. However, the point redemption done using a function is not reversed, because to remove the point redemption, the operation must be voided. So the new customer has to assume this point redemption. For example, a point redemption is set up on the store coupon tender, meaning that it costs points to get a store coupon. If the customer is changed, the store coupon stays in the transaction along with its associated point redemption.

NOTE:For details on points, see the Preferred shopper - Point redemption - Examples and Preferred shopper - Points topics.
