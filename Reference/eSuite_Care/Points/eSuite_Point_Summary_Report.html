eSuite_Care_Point_Summary_Report

Point_Summary_Report

Information

This report calculates points given, expired, adjusted, and redeemed at a date range and compares the points with the current balance.

Provides drill down to view the points transaction details information.

Navigation:

The user can access it under eSuite Care / Points / Point summary

Details of wizard fields
 Date panel


This component is the standard date panel used in eSuite in all other wizards.

For this report, by default the period Month is selected.  The detailed periods Week by day, Month by day Month by week Year by week and Year by month are not used since the result is generated based on the chosen dates only.

Customer ID

We can display results for specific customers, just by entering their ID (F1148) in this field, separated by comma.  

If using the More criteria button with a customer range instead, or if we want to have all customers, we should leave it empty.

Company/Legal name

Allows to search for customers having the same company name, (F1155)

Points balance:

Allows to search for all customers for a point program. 

 You can put many programs, separated by a comma.

Customer type:


When using this report on a host with SMS Payment, the points should be in Centralized account.  If using this on a server store using native points in SMS, we should use Local Account.

  •  Centralized account: the results will be taken from tables RPT_PAY adding current period details from TRS_PAY.
  • Local account: the results will be taken from native tables of SMS RPT_CLT_M  adding current period details from RPT_CLT_D.

NOTE: There is no way actually in eSuite to automatically select one choice or the other based on SMS options that are installed.  We should do this in the future so when SMS Payment is installed and this report is run from a host.

Target / More criteria / Layout (Standard in eSuite)

Target works the same way than all other reports in SMS and eSuite based on target that are for Reports.

The button More criteria allows searching from many other fields. It is a condition added to the other fields listed above. When using a customer range in addition to putting also text in the Customer ID field, the range must include this customer to get all the conditions working together.

Breakdown

By date

The results will be generated by date and should have 1 line by date (or more if other breakdowns are used).  The date column should appear first in the result grid.

By store

The results will be generated by store and should have 1 line by store (or more if other breakdown used) The store column should appear first in the result grid.

By customer

The results will be generated by customer and should have 1 line by customer.  The customer Id column should appear first in the result grid.

Show only difference between calculated and current

When it is checked, the results will only be displayed if there is a difference between calculated amount and totals.

Execution

When executing this wizard, the result grid is displayed in another tab on the top of the current window with the title Summary
