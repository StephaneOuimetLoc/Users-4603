eSuite - Future General Price Table

eSuite > Items > Batches > Price information

Information

The Future general price table is used to change the future general price information of the selected items.

How to display the Future general price table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Price information: the Price batch wizard opens in a new tab.
  3. In the Batch type field, select Price general.
  4. In the Batch field, select a price general batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Price general batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Price batch, then General: the Future general price table opens over the work area.

Layout

The Future general price table is divided in two parts: the main section and the Future general price table grid.

Main section

* Click on a field to get information on it.

Future general price table grid

* Click on a field to get information on it.

Content

Here is the description of the Future price redeem table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future general price record is added, linking the future general price record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future general price record is added, the target value of the current price record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Level The Level field contains the price level number.
See the Level field section below.
Like code The Like code field contains the future code used to group items.
When a future general price record is added, the like code value of the current price record is automatically displayed in the Like code field.
Item link The Item link field contains the future UPC code of a linked item.
When a future general price record is added, the item link value of the current price record is automatically displayed in the Item link field.
Alternate price The Alternate price field contains the future alternate price of the item, which is an alternative to the retail price.
Quantity The Alternate quantity field contains the future alternate quantity for the alternate price.
Add totalizer The Add totalizer field contains the future number of an additional totalizer in which the value of the item sold must be added.
When a future general price record is added, the add totalizer value of the current price record is automatically displayed in the Add totalizer field.
Time included The Time included field contains the number of days the item is rented at this price (or at this price level).
Discount description The Discount description field contains the future description of the discount given. It may appear on the receipts and invoices when customized.
When a future general price record is added, the discount description value of the current price record is automatically displayed in the Discout description field.
Commission The Commission field contains the future salesman's commission amount for the item.
When a future general price record is added, the commission value of the current price record is automatically displayed in the Commission field.
Exclude spending The Exclude from spending (Exc. spend) field specifies whether the future sales of the item are excluded from the points calculation (checked), or not (unchecked).
When a future general price record is added, the exclude from spending value of the current price record is automatically displayed in the Exclude from spending field.

The Level field

The Level field contains the price level number.

When a future general price record is added, the level value of the current price record is automatically displayed in the Level field.

NOTE:
 Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.
