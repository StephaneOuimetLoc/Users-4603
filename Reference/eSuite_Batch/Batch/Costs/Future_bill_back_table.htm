eSuite - Future Bill Back Table

eSuite > Items > Batches

Information

The Future bill back table is used to enter the future bill backs given on the selected items. It contains the future bill back information.

Displaying the Future cost bill back table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Cost information: the Cost batch wizard opens in a new tab.
  3. In the Batch type field, select Bill back.
  4. In the Batch field, select a bill back batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Cost bill back batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Cost batch, then Bill back: the Future cost allowances table opens over the work area.

Layout

The Future cost bill back table is divided in two parts: the main section and the Future cost bill back table grid.

Content

Here is the description of the Future cost bill back table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future bill back record is added, linking the future bill back record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future bill back record is added, the target value of the current cost record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Format The Format field contains the future buying format description.
When a future bill back record is added, the format of the current cost record is automatically displayed in the Format field.
Vendor The Vendor field contains the vendor ID and name. It links the item to a vendor.
When a future bill back record is added, the vendor ID and the vendor name of the current cost record are automatically displayed in the Vendor field.
Dollar 1 The Dollar 1 field contains the future value per case of the bill back 1, expressed in dollars.
When a future bill back record is added, the bill back 1 value of the current cost record is automatically displayed in the Dollar 1 field.
Dollar 2 The Dollar 2 field contains the future value per case of the bill back 2, expressed in dollars.
When a future bill back record is added, the bill back 2 value of the current cost record is automatically displayed in the Dollar 2 field.
Dollar 3 The Dollar 3 field contains the future value per case of the bill back 2, expressed in dollars.
When a future bill back record is added, the bill back 3 value of the current cost record is automatically displayed in the Dollar 3 field.
Dollar 4 TThe Dollar 4 field contains the future value per case of the bill back 2, expressed in dollars.
When a future bill back record is added, the bill back 4 value of the current cost record is automatically displayed in the Dollar 4 field.
Percent 1 The Percent 1 field contains the future value per case of the bill back 1, expressed as a percentage.
When a future bill back record is added, the bill back 1 value of the current cost record is automatically displayed in the Percent 1 field.
Percent 2 The Percent 2 field contains the future value per case of the bill back 1, expressed as a percentage.
When a future bill back record is added, the bill back 2 value of the current cost record is automatically displayed in the Percent 2 field.
Percent 3 TThe Percent 3 field contains the future value per case of the bill back 1, expressed as a percentage.
When a future bill back record is added, the bill back 3 value of the current cost record is automatically displayed in the Percent 3 field.
Percent 4 TThe Percent 4 field contains the future value per case of the bill back 1, expressed as a percentage.
When a future bill back record is added, the bill back 4 value of the current cost record is automatically displayed in the Percent 4 field.
Descriptor 4 The Descriptor 4 field contains the future description of the bill back 4.
When a future bill back record is added, the bill back 4 description of the current cost record is automatically displayed in the Descriptor 4 field.
