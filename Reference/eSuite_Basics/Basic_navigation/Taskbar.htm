eSuite - Taskbar

eSuite

Table of contents

Informationsiiiiiiiiiiiiii

The eSuite interface includes a taskbar whose layout and behaviour are identical across modules.

The eSuite taskbar is located at the top of the eSuite application. It is configured to switch between modules, navigate through views and provide session information.

Layout

The eSuite taskbar includes the following components:

  1. LOC logo
  2. Views area
  3. Control menu

Behavior

This section focuses on the behavior of the taskbar components and on the actions they trigger.

The LOC logo

The LOC logo is located on the taskbar, at the top left side of the eSuite application.

It is a clickable field that opens the Home window with access to all the eSuite modules.

The following fields are available in the Home window:

FIELD DESCRIPTION
Home Displays the window's name.
Refresh Refreshes the window.
Pin Pins the window to the desktop.
Close Closes the window.
eSuite Care Opens the eSuite Care module in a new browser tab.
eSuite Batch Opens the eSuite Batch module in a new browser tab.
eSuite Buy Opens the eSuite Buy module in a new browser tab.
eSuite Sell Opens the eSuite Sell module in a new browser tab.
eSuite Manage Opens the eSuite Manage module in a new browser tab.
eSuite System Opens the eSuite System module in a new browser tab.

The views area

The views area is located on the taskbar, at the top of the eSuite application.

The Home tab is active by default when you open eSuite.

The views area shows the name of the views that open on the desktop when an entry is selected in the launcher.

For example, when you select the Customers > Master entry in the launcher, the taskbar will open the Customers view and it will activate it. That means that it adds a blue line on the top of the Customers view in the taskbar. It will also open the customer view on the desktop, with the Start window displaying the customer-related options.

When you click on a view in the views area:

  • The view becomes active.
  • It displays a blue horizontal line.
  • It shows a close button when you hover the mouse over it.
  • The desktop shows all the windows that have been opened in the active view.
  • The Context Tools window displays information specific to the view.

The views area allows opening multiple views. When more views are open than the allotted space, a button consisting of three horizontal lines appears to the right of the views area. When expanded, this button displays a list of the hidden views. The most recent one is checked.

You can select any view in the list to activate it in the taskbar and to display the associated windows on the desktop.

The control menu

The control menu is located at the top right of the eSuite application.

The following controls are available:

  • Server
  • Help
  • Settings
  • Desktop
  • Session

The Server button opens the Process status window, with information on the background processes being executed on the processors in SMSHttps.


The following fields are available:

FIELD DESCRIPTION
Process status Displays the window's name.
My process Shows the processes that the user started executing using the session.
All process Shows the system processes that are being executed on all processors.
Process Shows the name of the script that is being executed.
Status Shows the status of the running process that can be queued, started, running and finished.
Progress Shows the progress bar for the process execution.
Message Shows the file that the processor is working or processing on.
Errors will also be displayed here.
Last update Shows the date and time of the process execution.

The Help button provides access to the 3.5 and 4.5 Users and Members help files.

The Settings functionality in eSuite allows you to set and save the user's session parameters. It opens the Configuration window.

The following fields are available in the Configuration window:

FIELD DESCRIPTION
Configuration Displays the window's name.
Touch keyboard When checked, it allows using a touch keyboard with eSuite.
Default keyboard Sets a different keyboard layout or input method.
Default value: US.
Top pivot records Sets the number of pivot records that can be viewed.
Default value: 2000.
Top result records Sets the maximum number of results that can be viewed.
Default value: 10000.
Default query timeout Sets the time that the query operation can take before the SQL server times out.
Default value: 40 seconds.
Extended query timeout Extends the default query timeout to a higher time in seconds.
Default value: 1200 seconds.
Main server Sets the main server for the system (host) or the local area network (store).
Web socket Shows the name of the connection between the client and the server.
Not editable.
Default value: sms.

The Desktop button lists the windows that are open on the active desktop.

It allows you to:

  • Clear hot-indexes: Deletes the index on the table that is active on the desktop.
  • Close desktop: Closes the active desktop or any of its associated windows.
  • Close any of the windows on the active desktop.

The Session button displays a menu with information about the active session. It provides access to different sub-menus.

The following options are available in the Session menu:

OPTION DESCRIPTION
Logout Signs off the current user.
Prompts the confirmation message Logout ok.
Base rights Shows the basic user level permissions. By default, they are set with minimal access rights.
Settings Provides tools for setting and saving the session parameters.
Information Offers details on the active session: server, session number, store number, terminal number, user level, profile, transaction date, hardware profile link.
About Shows information on the eSuite version and licence.
Session info Shows the login information on the session: ID/level, name, date, store, module, version.
Desktop layout Shows the list of the saved desktop layouts.
Save desktop Allows you to create and save desktop layout or to delete them.
Cursor location Provides the tools for defining the movement sequence of the cursor while it's switching fields.
Next cursor location Changes the cursor focus to the next field defined in the Cursor location sequence.
It works with the Save desktop and Cursor location functions.
