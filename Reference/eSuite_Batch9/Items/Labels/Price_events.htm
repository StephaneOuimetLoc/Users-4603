eSuite - How to Generate Labels Based on Price Events

eSuite > Tools > Labels > Generate based on price events

Information

The Labels - Price Events utility helps identifying the items for which the system will print labels on a specific date. The system manages all the batches and the events defined in the items, and compiles the next event for the selected date.

This feature is used to compile the very next event, not the one scheduled in three weeks. This means that if the utility is due to run for today and changes have to be printed, when the utility will be run again for the next week, the changes done today will be shown again for the next week until the changes have been activated. So the following steps are recommended: price changes for instance, print labels, activate and deploy the changes to avoid them to be printed again on the next use of this utility.

How to Generate Labels Based on Price Events

In the Launcher, click the arrow in the Tools entry to display its menu and then:

  1. Click on Labels: the Start window opens in the Labels work area.
  2. Click on Generate based on price events: the Labels - price events wizard opens in a new tab.
*Click on a field to get more information.

The Next price source method field

The Next price source method specifies the method to use when printing labels.

Four methods are available:

  • See all events and detect retail price change
    Let's say there is a sale that goes effective today. The active price will be the sale price. A next event will be generated at the end of the sale to have the item get back to the regular price. This means that the regular price does not really change and so no label should be printed.
    This method will recall the last regular price that was printed and will compare it with the next regular event. Only the real price changes will be shown.
    The key in Application.ini file in label section is LABEL_REGGEN_METH=0.
  • Include Instore price change as a retail price
    This method acts exactly as the first one except that the INSTORE event is considered as a regular price change.
    Let's say there is an INSTORE for the next three months. The same label is used for a regular price change, so you will remove the item label and put this one instead. In fact, this is like a regular price change, so there is no special sign printed for it.
    This method will recall the price that was printed.
    The key in Application.ini file in label section is LABEL_REGGEN_METH=1.
  • Include Instore price and TPR change as a retail price
    This method works as the previous one, except that the INSTORE and the TPR event is considered as a regular price change.
    The key in Application.ini file in label section is LABEL_REGGEN_METH=2.
  • See all events including the return to regular price
    This method does not remember the last regular price printed. So ALL next events will be shown here. The key in Application.ini file in label section is LABEL_REGGEN_METH=3

See the Labels - examples topic for examples for each case.

The Price level fields

The Price level start and the Price level stop fields allow setting the price level range of the items for which labels will be printed.

NOTE:
Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.

Labels - Event summary results

When the Labels on price events utility is launched, it opens the Labels - Event summary results wizard:

*Click on a field to get more information.

The Event summary results wizard is used to report, generate, print and activate the items that have a next event.

Here is the description of the fields and functionalities in the Labels - Event summary results wizard:

FIELD DESCRIPTION
Keep items having a predefined label When checked, the Use template from shelf value is added to the menu in the Label format field.
Event list Shows all next events grouped by event.
The number displayed between parentheses is the number of items that have a selected event to print. After selecting an event to print, the Aisle field is refreshed to show these items grouped by aisle (this information is coming from the Shelf table). All aisles or only a specific one may be selected for printing. The label format to use for this event is defined by the value set in the Label format field.
Aisle Groups by event all items with a selected event to print.
All is the default value.
Label format Contains the label format. Provides a drop-down menu with the predefined label formats.
Like code Gives the option of printing labels without using like codes or printing a single label for a group of items that share the same like code.
In a sale event, it may happen that all flavors of a product go on sale. Generally, one sign is printed for each flavor, but it may be possible to use like codes and print only one sign for all flavors.
The Like code field in the Price table is used to group items together. To make it work with the label utility, the UPC code of the master item of the group must be entered in the Like code field for all flavors of this product. Only the label of the master item will be printed when the like code feature is used.
Select items by price variation Gives the option of printing labels for all items or only items whose price has gone up or only items whose price has gone down.
When printing labels for items whose price has changed, the engine will compare the actual regular price with the last regular price print to determine which label to print. If there is no last regular price print, then this label will qualify only when the All Items option is selected.
Action to do The Print report option allows printing a report before printing the labels and make sure the right labels are printed for the right items.
The Generate label option allows generating a label batch to be printed and it recalls the regular price (if a regular price event is printed) to detect the real price changes.
The Activate item option takes the printed items from the batches and applies them to the items.
NOTE: The Activate item option will not send the changes to the lanes. A deploy change process is still needed. Also, when items are activated and transferred from the batches to the items, there might still be items in the batches. In this case, the batches are flagged as Closed, so no more items can be added to these batches from a host deploy change.
Print this quantity for each label Allows entering a quantity of labels to be printed.
Use quantity set in item location Allows printing the same label according to the quantity set in the Shelf location table.
When selected, the Print this quantity for each label field is disabled.
Reset Allows resetting the wizard values to the default ones.
Execute Launches the search and opens the list of labels that are ready to print.
Enabled when the Event list and the Label format fields are populated.
Settings Opens the Execute window.
Allows changing the search parameters.
