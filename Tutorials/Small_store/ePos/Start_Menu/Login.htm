User Login

Information

Login procedure

All applications of the LOC Store Management Suite software are protected by a security system. Users must login to access data and / or functions. You can use numeric user codes and passwords to access applications for workstations having only numeric keyboards. If the workstation has an alphanumeric keyboard, the operator can log in using their email. The password can also be alphanumeric.

NOTE: When a user is logged in, they are using a session. Only one session is allowed to be logged into any of the applications at any one time, with the exception of the SMS Front-end workstation that may have a secondary user log in temporarily using the Login Access function. 

Operator login

  1. Open a ePOS session
  2. Click on Login
  3. Enter user code (in this example we will use operator 2 - cashier)
  4. Click on E (Enter)
  5.  Enter password and click on E (Enter)

NOTE: Depending on the operator level and security profile set up for the operator being used when logging into the POS, the system will either bring you to the POS main screen or you will need to select the module you would like to access i.e. POS or Back-office. For example: A cashier with a level 2 having only access to normal transaction functions and pricing information, the system will bring you start to the POS main screen.

Login access

Information

Some operations of the Selling screen need the supervision of the manager (refund, price modification, etc.). In this case, the "Please login with the right access level" message is displayed in the Entry window, indicating that the supervisor, usually the manager, must log in and supervise the operation.

In this example, we will use the Login access for a price modification:

Login using a level 2 Cashier operator. From the POS main screen:

  1. Enter PLU
  2. Select Access login
  3. You will see the message:
  4. Enter operator 5
  5. Enter password

Once you have logged in using the operator 5 - Manager, you will be able to select the Modify price function and change the price before completing the transaction.
