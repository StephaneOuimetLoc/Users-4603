eSuite Manage - Generic Accounting Export

Manage Export Accointing 

Information

The Accounting module in LOC SMS eSuite Manage is used to export the store's data to an accounting software. Sales and accounts payable can be exported

The LOC SMS eSuite Manage offers a Generic Accounting Export for G/L and A/P. The data exported can then be imported in an accounting package using its import option.

WARNING:
  • To be able to program your General Ledger accounts in the LOC SMS eSuite Manage an Export terminal 961 must be added in the Terminal table (Export to = Generic Export GL).
  • To be able to program your Account Payable accounts in the LOC SMS eSuite Manage, the Export terminal 965 must be added in the Terminal table (Export to = Generic Export AP).

General Ledger

The General Ledger data is exported by a script named Export_Accounting_GL.sql. The filename will be SMSGL_ttt_yyyymmdd where t=target followed by the date. The exported file will be created in both:

  • The folder specified in the Sending driver field of the terminal 961 (the default value is C:\SMS_EXPORT\). One file per day is created.
  • In the eSuite > Manage > Accounting > Ledger > Export file where the General Ledger file can be selected and dowloaded. One file per day is created.

Example of file format:

20200914,3,Grocery/Net Sales,D1,,9.00,
20200914,1301,Dwr Cash,T1301,9.00,,
Parameter number Description DataType
1 Date of the transaction Date (yyyymmdd)
2 Totalizer number Numeric
3 Totalizer descriptor Char
4 GL Account number Char(20)
5 Debit Amount Numeric (99999.99)
6 Credit Amount Numeric (99999.99)

Accounts Payable

The Account payable data is exported by a script named Export_Accounting_AP.sql. The filename will be SMSAP_ttt_yyyymmdd where t=target followed by the date. The exported file will be created in both:

  • The folder specified in the Sending driver of the terminal 965 (the default value is C:\SMS_EXPORT\).
  • In the eSuite > Manage > Accounting > Payable > Export file where the General Ledger file can be selected and dowloaded.

Example of file format:

H,20090914,001,6225,B620,,
D,8201,Grocery/Total purchased,123456,74.67,
D,13910,Drop payment,123456369,,49.78
D,13920,Charge to account,555666,,24.89

The file is created with 2 different records, H= header and D=detail. For each transaction, an H record will be created:

Parameter number Description Data Type
1 H (Header) Char(1)
2 Date of the transaction Date (yyyymmdd)
3 Store Number Char(5)
4 Transaction number Numeric
5 Vendor code Char(14)
6 PO Number * Char(8)
7 Invoice Number Char(20)

* The PO number will only be available if the final document to export was originally done from a Purchase Order type of document. If the A/P export is done at a head office, the billing document must be transferred to the host before the export to get the PO number exported, otherwise, this field will remain empty.

For each transaction, a detailed record will be created for each different AP account number.

Parameter number Description Data Type
1 D (Detail) Char (1)
2 Totalizer Numeric
3 Totalizer descriptor Char
4 Accounting account Char(20)
5 Debit Amount Numeric (99999.99)
6 Credit Amount Numeric (99999.99)

Technical notes

  • When adding the Export terminal, the default sending driver (export path) is derived from the System.ini [GENERAL] ReportExportDir value.
  • If this parameter is empty, its default value is C:\SMS_EXPORT\.

How to program and post the GL and AP accounts

  • For a detailed step by step on how to program and post the GL and AP accounts, see the Accounting module documentation in the SMSUsers help.
