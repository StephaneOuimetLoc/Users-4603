mFloor - Buying Module Configuration

mFloor  Configuration

Information

The Buying Module Config screen enables users to set the configurations for the three main Buying sub-modules Purchase Order, Receiving and Deliveries.

Setting configurations in the Buying module configuration screen

To set the configuration, follow these steps:

  1. Click or tap Buying module then click or tap Config.
  2. Enter the password, (the default value is "999") and click or tap OK.
  3. The Config buy screen displays three configuration sections, one for each of the buying sub modules (Purchase order, Receiving, and DSD settings.)

  5. For each sections listed, there are two settings:
    • Order qty - This is the default number of cases that displays when ordering an item, it can be changed by user as required.
    • Goto exist - This numeric field is equivalent to the Go to item line field in PDAs and essentially this field changes the behavior of how an item is added to the document, it either creates a new record (1) or is placed in an existing record (2). For further information refer to the SMSMembers.chm file under the section Hardware/PDA/Advanced/Profiles.
  7. When all settings have been changed, as required click or tap save and a save profile dialog box appears. To discard changes and return to Products menu, click or tap Cancel .
  8. Enter the profile name associated to the user/operator or enter "1" and click or tap OK, to save the profile to the Default profile used by ALL users. More information on profiles can be found in the Profile - Operator table field topic in the SMS Users help file.
  9. Upon successfully saving the profile, a Profile saved: dialog box displays the saved profile name for confirmation. In the example below settings were saved to the "DEFAULT" profile for all users.
