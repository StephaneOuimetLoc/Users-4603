The Sub-department table contains information for groups of items. The eSuite software uses sub-departments as a way of grouping both coded items and direct key sales for reporting.

Instead of linking coded items directly to departments, eSuite uses sub-departments to group items, and then links the sub-departments to departments. This allows for another report level, regrouping item codes in a more specific manner.

Sub-departments also have their own settings, such as taxability, maximum and minimum entries for sales and voids etc., to be applied when direct key sales are performed. Coded items can also be programmed to follow these settings using the POS Master table.  

Note: 
The eSuite software does not support sub-department programming for all types of points of sale, as such sub-department information may not get through to all points of sale

Sub-department table must be programmed to use the sub-departments for reporting purposes

Displaying the Sub-department table

From the launcher in eSuite Care, Manage, Buy, Sell or Batch, click the down arrow in the Dimensions entry to display Sub-departments menu:

Click on Sub-departments: the Wizard Start window opens in the Sub-departments work area.

Click on List: the Sub-dept tab opens in the same work area, displaying the Sub-department List grid

    

Click on  in the grid toolbar to display the table

The Sub-department Edit table is displayed

To create a Sub-department

1. To create a new Sub-department

click on  in the grid toolbar

or click on  in the Sub-department Edit toolbar from a populated table

2. The Sub-department Edit table opens in the work area

3. Enter number, description & choose the department desired from the department drop down field (minimum requirements)

4. Apply the desired attributes to the Sub-department by checking off selections and entering values

Note:  Attributes that can be set via Sub-department could also be implemented by individual coded item using the fields in the POS Master table

Information:  Please see the Reference Table for programming details on certain attributes, for example: Cashier age, Customer age, Others Flags, Stop discount etc

5. Click the  in the toolbar or hit enter on your keyboard to save

6. From the bottom menu, refresh   the grid to see your entry displayed

    

Warning: The Sub-department table must be pushed to the points of sale using Deploy Auxiliary 

To modify a Sub-department

1. From the Sub-department List grid, select/highlight the entry you would like to edit

2. Click Edit on the bottom menu & choose Sub-Department (F3) or press F3 on your keyboard

3. The Sub-department Edit table opens

4. Modify as required

5. Save when complete

Warning: Once created, Sub-departments should not be deleted as the link to its historical reporting data will be lost
