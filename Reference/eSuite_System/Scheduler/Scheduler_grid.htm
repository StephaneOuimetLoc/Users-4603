eSuite System - Scheduler Grid

eSuite System > Scheduler

Information

The Scheduler grid contains default jobs that are already configured in the system. Some of them are automatically activated at installation time.

By default, the list of jobs contains five columns: Host target, Number, Name, Execution target, Last automatic execution and Next automatic execution. The grid layout can be customized. For details, see the Grids in eSuite topic.

The Scheduler grid displays records in ascending order by host target number.

Displaying the Scheduler grid

In the Launcher, click the Scheduler entry:

  1. The Scheduler window opens in the Scheduler work area.
  2. Click on Scheduler: the Scheduler grid opens in the List tab.
  3. To edit an existing record, click on Edit in the bottom menu to open the Scheduler table.
  4. To create a new record, click Add data in the toolbar: the Scheduler table will open for editing.

Executing a job manually from the Scheduler grid

Sometimes, it may be necessary to execute a scheduled job manually. For example, when a process does not succeed during an automatic execution. The scheduler can also contain jobs that are not set for an automatic execution, using the scheduler as a configurable menu that contains jobs that have to be executed manually.

To manually execute a job from the scheduler grid:

  1. In the Scheduler grid, select the job to run.
  2. Click on Menu at the bottom of the grid: the Edit (F3) and Execute options are displayed.
  3. Click the Execute button.
    A dialog box appears, allowing users to pick and confirm the date of manually executing the job:


    NOTE: If the selected job does not allow manual execution, user receives a message that the command cannot be executed manually.
     
  4. Set the date and click Execute. A message opens, confirming that the task has been added to the queue.
  5. Click OK to complete the manual execution.

NOTE: The Execute option in the Menu is only available when the selected task can be manually executed. If not, it is greyed out and disabled.

Scheduled jobs

Here is the description of the jobs that are executed on the back office server:

Job name Job number Description
Deploy end of day 10 The Deploy end of day job polls the NRGT (Non-Resettable Grand Total) value of each front end workstation.
Close events 15 The Close events job polls the net sales in the daily item movement table into different totalizers for items that were on sale, on TPR or in-store.
Deploy to head office 20 The Deploy to head office job sends the sales from the store to the head office. This job can be executed while the store is running.
Calculate live counters 25 The Calculate live counters job updates the current quantities on order (purchase orders) and on process (sale orders) for all items. This job may be executed many times during the day.
Calculate cumulatives 30 The Calculate cumulatives job updates weekly, monthly and yearly totals in every report table (RPT_???). This job may be executed at any time during the day.
Set effective date 35 The Set effective date job changes the soft date. This job is used for stores that do not end the day at midnight.
Purge old promo price 37 The Purge old promo price job removes all past promotions (TPR, Sale and In-store prices).
Import files 39 The Import job imports data into the system.
Activate accept files 40 The Activate accept files job automatically execute the external batches received from the head office as a SIL file import.
Activate instore batch 45 The Activate instore batch job automatically execute the in-store batches, received from the head office (Accept files), and/or created locally.
Deploy changes 50 The Deploy changes job sends the item changes from the back office server to the front end workstations. This job can be executed while the store is running. See your reseller for a customized setup.
Deploy auxiliary 52 The Deploy auxiliary job sends the auxiliary tables data from the back office server to the front end workstations. This job is never scheduled since it needs answers from the operator. If the Deploy auxiliary job is used while the store is running, do not select Clean and Load because the job will cause all the tables to clear. See your reseller for a customized setup.
Deploy images and templates 53 The Deploy image and template job sends images (bitmaps, advertising images and pages, etc.) and templates (receipt template, etc.) from the back office server to the front end workstations and the cash registers. See your reseller for a customized setup.
Check database sync 55 The Check database sync job is used to compare the back office database and the front end workstations databases. This job should never be performed at the head office.
Backup program 60 The Backup program job backups the complete database by exporting the data in a SIL file format in the Office/Load directory. The database is backed up by table (one table = one file). This job also backups the main directories (Storeman, Options, Drivers, Office, Load) by creating zip files in the backup directory (by default C:\Storeman\Backup\Program). See the reseller for a customized setup.
Backup on device 62 The Backup on device job backups the zip files of the backup directory (by default, C:\Storeman\Backup\Program) on an external device. See the reseller for a customized setup.
Backup history 64 The Backup history job backups the sales tables (TRS_, RPT_) by creating zip files in the backup directory (by default, C:\Storeman\Backup\Program).
Purge all report data 90 The Purge all report data job purges the LOC Store Management Suite report files. See the reseller for a customized setup.
Automatic update download 120 The Automatic update download job is used to download automatically the latest version of the eSuite software from the www.locsoftware.com website. The files downloaded are copied in the Storeman/Install directory of the back office server. See the reseller for a customized setup.
Execute task and cube watch 130 The Execute task and cube watch job is used to automatically email the selected eSuite reports to operators and execute the cube watches.
Set date range restriction 140 The Set date range restriction job is used to restrict the number of days it is possible to work on old documents from their creation dates. This job avoids posting old documents that may cause potential issues.
