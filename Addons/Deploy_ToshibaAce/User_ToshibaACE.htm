Module - ToshibaACE Interface User Guide

ToshibaACE Unmanaged

Information

The Toshiba ACE Interface - Users Guide document describes how to use LOC SMS with the Toshiba ACE interface in order to manage a store running with a Toshiba ACE controller. The Toshiba ACE controller controls the registers' hardware and network, and LOC SMS manages all tables needed to run the POS transactions. Therefore, maintenance of items, coupons, customers and operators is done on the LOC SMS back office server or back office workstations. Once deployed to the Toshiba ACE controller, the controller will delegate data to the Toshiba ACE lanes.

Deployment

Changes done in the LOC SMS system have to be deployed to the Toshiba ACE controller. Below are the different deploy processes that can be performed:

  • Deploy Auxiliary, used to send bottle link and sub-department data to the Toshiba ACE controller;
  • Deploy Change, used to send item, coupon, operator and customer data to the Toshiba ACE controller;
  • Deploy End of Day (EOD), used to poll the transactions from the Toshiba ACE controller (TLog).

Deploy Auxiliary

The deploy auxiliary process is only used to send the bottle link or sub-department information to the Toshiba ACE controller.

To execute the deploy auxiliary process, follow the steps below:

  1. In the eSuite/Manage application, expand the Tools drop-down menu (1), click Deploy (2) and select Deploy Auxiliaries (3).
  2. Expand the Target drop-down menu (1) and select Show Search Tool (2).

  3. Select DACE (1) and click OK (2).

  4. Expand the Deploy Type drop-down menu (1), select Sub-Dept OR Bottle Link (2), then click Execute (3).

Deploy Change

The Deploy change process sends item and coupon changes, item and coupon link deletions, along with customer and cashier data to the Toshiba ACE controller.

To execute the deploy change process, follow this step:

  1. In the eSuite/Manage application, expand the Tools drop-down menu (1), click Deploy (2) and select Deploy Changes (3).

Some fields are validated during the deploy process. In most cases, items will be deployed to the Toshiba ACE lanes without issue. If there is an issue with deployment an urgent message is received. See examples below.

Example:

  • If an item contains an invalid price (null or zero).
  • If the UPC code contains a character other than a number.
  • If an item has the same UPC code as a bottle link, urgent messages will be created.

NOTE: It is important to check for urgent messages after a deploy change is executed to ensure there is no error pertaining to item programming.

Deploy End of Day (EOD)

The deploy end of day process sends a request to the Toshiba ACE controller to poll the transaction log file, referred to as the TLog file. This process is normally executed automatically through the Deploy end of day job in the Scheduler once a day, but can also be executed manually.

Miscellaneous Item

Miscellaneous items are Paid In or Paid Out items. To create an item linked to a miscellaneous account, follow the steps below:

  1. For a Paid In item, check the Not in net field in the POS table; for a Paid Out item, check the Not in net and Bottle return fields;
  2. Enter the Toshiba ACE account number for that function in the M. Fam field of the POS table.
    NOTE: The account must already exist in the Toshiba ACE system; the SMS deploy process will not create the account itself.

Account Receivable (AR)

The Toshiba ACE system has different risk levels. By default, the LOC SMS risk level 1 is used for a $0.00 credit limit, corresponding to the risk level 55 in the Toshiba ACE system. All other risk levels, having a credit limit higher than $0.00, are linked to Toshiba ACE risk level 50. All customers assigned to a risk level, possessing a credit limit, will be verified by the Toshiba ACE system. To stop certain customer accounts from being charged, the risk level needs to be changed to 1, followed by a Deploy Change.

All LOC SMS Customer reports will work (Account Statement, Interest Charge, etc.), as if a LOC SMS-only system was used. An analysis tool named Customers Over Limit is available. The tool will help to manage customers going over the limit, at which point their risk levels may be re-assigned to a credit level of $0.00 to stop charges. The tool can be found in eSuite Manage\Accounting\Receivable\Customer Accounts menu.=

Positive and Negative Check Files

For positive check lookups, it is suggested to assign customers whose checks are accepted at Risk Level 1. Negative check lookups are Risk Level 2.

The negative check file gets populated by setting the Accept check field of the Risk level table.

  • The check number needs to be used as the customer number in LOC SMS.
  • It is possible to create a generic 'Bad check' account, add the check account number in the Customer alternate code table using the CK (check) type.

Report files

The LOC SMS system polls the transaction log file (TLog) from the Toshiba ACE controller every day. It is then processed to populate the proper Report tables (RPT_).

Temporary Markdown

When the TLog is received from ACE, the price found within will be compared to the 'Active Price' in SMS. The difference will show as a 'Markdown' value will show on the report(s).
