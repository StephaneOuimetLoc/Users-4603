eSuite Buy - Transaction Header Table

eSuite Buy > Buy

Information

The Transaction header table allows editing the general information in the buying documents. It opens with the Edit header button located in the header section of the document.

Layout

The Transaction header table is divided in two parts: the main section and the tab section.

Main section

*Click on a field to get more information.

Tab section

*Click on a field to get more information. *Click on a field to get more information. *Click on a field to get more information.

Content

Here is the description of the Transaction header table fields and their functionalities:

FIELD DESCRIPTION
Main section
P/O # The PO # field contains the purchase order number.
Generated by the system for the new purchase orders only.
Note: Receiving, billing and delivery documents may automatically get a purchase order number if the buying process starts from a purchase order.
Doc # The Doc # field contains the number of the open document.
Generated by the system for new documents.
Order The Order field contains the date on which the order has been placed.
Generated by the system for the new purchase orders only.
Based on the delivery schedule defined for this vendor in the Vendor table or on today's date.
Store ID The Store ID field contains the number of the store for the delivery.
Vendor ID The Vendor ID field contains the vendor identifier.
The value is taken from the Vendor table.
Name The Name field contains the vendor's company name.
Address The Address field contains the vendor's address.
City The City field contains the name of the place where the vendor's company is located.
Contact The Contact field contains the name of the salesperson to contact at the vendor's.
Phone The Phone field contains the vendor's phone number.
Fax The Fax field contains the vendor's fax number.
Terms The Terms field contains the description of the payment terms.
Account # The Account # field contains the vendor's account number.
Buyer The Buyer field contains the user level that creates the document.
Reference The Reference field contains the invoice number given by the vendor to your bill (refer to the physical bill received from the vendor).
Customer ID The Customer ID field contains the customer code.
Shipping tab
Arrive The Arrive field contains the date on which the merchandise will arrive at the customer's location.
Delivery The Delivery field contains the date on which the merchandise will arrive at the store.
Set by default when the document is created.
Price event The Price event field contains the date used to calculate the selling price.
Works with the PDA only, so that the clerk knows if the items will be on special that day and adjust the ordered quantity consequently.
FOB The FOB field contains the name of the location where the merchandise will be shipped by the vendor.
Shipper ID The Shipper ID field contains the shipper's identifier.
Ship via The Ship via field contains the shipper's name.
Address The Address field contains the shipper's address.
Order notes The Order notes contains special information on the order.
Shipping notes The Shipping notes field contains special information on the shipment.
Vendor tab
Internal note The Internal note field contains additional information on the vendor.
Minimum order The Minimum order field contains the minimum quantity on cases that must be ordered for the vendor to deliver the merchandise.
Minimum cubic The Minimum cubic field contains the minimum volume that must be ordered for the vendor to deliver the merchandise.
It comes from an agreement between the store and the vendor. The system does not control this volume, it is used for information purposes only.
Delivery days The Delivery days field contains the number of days needed to receive the current item from the vendor after the order.
BO days The BO days contains the number of days the items of a purchase order can stay in back order for the vendor.
Categories The Categories field contains the types of items the vendor offers.
Special entries may be used:
  • SHIPVIA: the vendor is a carrier.
  • ANY: any item can be received from the vendor.
Broker The Broker field contains the name of the freight company that ships the goods if the vendor does not ship directly to the store.  
Shipper tab
Name (Address) The Name field contains the shipper's name.
Contact (Address) The Contact field cotains the name of the person to contact at the shipper's location.
Weight (Freight) The Weight field contains the rate of the shipping fees charged by weight for the delivery of the merchandise by the shipper.
Cubic (Freight) The Cubic field contains the rate of the shipping fees charged by volume for the delivery of the merchandise by the shipper.
Phone (Shipper) The Phone field contains the shipper's phone number.
Fax (Shipper) The Fax field contains the shipper's fax number.
