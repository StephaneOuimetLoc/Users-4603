Preferred shopper - Points - Examples

Points are automatically given by the system according to the point method used. The points given are added to the point balance of the customer entered for each transaction. If no customer is entered for a transaction, the points are not added.

List of examples

  1. Points are given on all items, except on cigarettes and alcohol, at the 1 penny per point rate.
  2. Points are given on all items at the 1 penny per point rate, plus 250 points are given when the payment is made by credit card.
  3. Points are given on all items at the 1 penny per point rate, plus on the sub-total amount at the 10 penny per point rate.
  4. Points are given on all items at the 1 penny per point rate, on the entries for which the amount is higher or equal to $1.00.
  5. Points are given on all items at the 1 penny per point rate when the customers buy for a minimum of $100.00 in the transaction.
  6. A fixed number of points is given for each item. These points are doubled for the customers that are part of a particular shopper level.
  7. A fixed number of points is given for each item. These points are doubled on Wednesday from 8:00am to 12:00pm.

1. Points are given on all items, except on cigarettes and alcohol, at the 1 penny per point rate

This example uses the Entry amount and preset item point method with item exclusions.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Entry amount and preset item point method in the Actual method field.

  4. Type 1 in the Penny/point field.

  5. Save the changes.

In the Base tab of the Price table, set up the information for each item:

  1. In the Launcher, click the arrow in the Items entry to display its menu.
  2. Click on Master.
  3. In the Start window, click on Item list.
  4. In the wizard, enter the search criteria corresponding to all cigarettes and alcohol items and click Execute.
  5. For all cigarettes and alcohol items, type 0 in the Points field.

  6. Save the changes.

A point will be given for each penny of each item sold, except for the cigarettes and alcohol items.

The points are added to the point balance of the customer entered in the transaction, if they are part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.

2. Points are given on all items at the 1 penny per point rate, plus 250 points are given when the payment is made by credit card

This example uses the Entry amount, preset item and tender point method with a fixed number of points for the credit card tender.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Entry amount, preset item and tender point method in the Actual method field.

  4. Type 1 in the Penny/point field.

  5. Save the changes.

In eSuite System, set up the credit card tender function (104) in the Function table:

  1. Type POINT=F250 in the Alpha parameter field.

  2. Save the changes.

A point will be given for each penny of each item sold, plus 250 points if the payment is made by credit card.

The points are added to the point balance of the customer entered in the transaction, if they are part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.

3. Points are given on all items at the 1 penny per point rate, plus on the sub-total amount at the 10 penny per point rate

This example uses the Entry amount, preset item and sub-total point method with 2 different penny per point rates.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Entry amount, preset item and sub-total point method in the Actual method field.

  4. Type 1 in the Penny/point field.

  5. Save the changes.

In eSuite System, set up the sub-total function (720) in the Function table:

  1. Type POINT=F250 in the Alpha parameter field.

  2. Save the changes.

A point will be given for each penny of each item sold, plus 1 point for each 10 pennies on the net sales amount of the transaction.

The points are added to the point balance of the customer entered in the transaction, if they are part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.

4. Points are given on all items at the 1 penny per point rate, on the entries for which the amount is higher or equal to $1.00

  2. Click Edit in the bottom menu to open the Shopper level table.

  4. Type 1 in the Penny/point field.

The points are added to the point balance of the customer entered in the transaction, if they are part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.

6. A fixed number of points is given for each item. These points are doubled for the customers that are part of a particular shopper level.

This example uses the Preset item alone (A) point method multiplied for some customers.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Preset item alone point method in the Actual method field.

  4. Type 2 in the Multiplier (points) field.

  5. Save the changes.

In the Base tab of the Price table, set up the information for each item:

  1. Type the number of points given when the item is sold at a price event in the Points fields.

  2. Save the changes.

The corresponding points will be given for each item sold, doubled for the customers that are part of that particular shopper level.

The points are added to the point balance of the customer entered in the transaction if they are part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.

7. A fixed number of points is given for each item. These points are doubled on Wednesdays from 8:00am to 12:00pm

This example uses the Preset item alone point method multiplied during a specific period. It uses the upgrade-to-level concept. The shopper level contains the period for which the points are doubled. For this period, the customers upgrade to another shopper level that doubles the points.

In this example, the shopper level upgrades to shopper level 4 to double the points.

See the Preferred shopper - Shopper level upgrade topic for details on the upgrade-to-level concept.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the points will double.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. In the Upgrades tab, type 4 in the Upgrade to level field.

  4. Type 0800 in the Wednesday (Wed) Start field.
  5. Type 1200 in the Wednesday (Wed) Stop field.

  6. Save the changes.

Set up the points information in the upgraded shopper level 4:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select shopper level 4 and open the Shopper level table.
  2. Select the Preset item alone point method in the Actual method field.

  3. Type 2 in the Multiplier (points) field.

  4. Save the changes.

In the Base tab of the Price table, set up the information for each item:

  1. In the Points fields, type the number of points given when the item is sold at a price event.

  2. Save the changes.

The corresponding points will be given for each item sold, doubled on each Wednesday from 8:00am to 12:00pm.

The points are added to the point balance of the customer entered in the transaction if he is part of the targeted shopper level. If no customer is entered or if the customer is not part of the targeted shopper level, the points are not added.
