eSuite - Coupons Overview

eSuite > Items > Master > Coupon list

Information

The eSuite software offers the coupon wizard utility aimed at simplifying the creation of coupons by grouping all their criteria (type, method, quantity, amount, etc.).

Once the information is gathered and saved, the coupon is assigned a unique UPC that starts with 096 followed by the promotion number (10 digits). It is possible to manually enter coupon UPCs: automatic coupons may, for example, start with 095 followed by the promotion number and target coupons may start with 097 followed by the promotion number.

Below is a description of the Coupon wizard fields and functionalities and an example of how to create a coupon.

Displaying the coupon wizard

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens in the Items work area.
  2. Click on New next to the Coupon list option: the Coupon wizard opens overlapping the work area.

Coupon methods

The coupon methods dictate the coupon behavior. They may be selected from the following fields in the Add criteria window:

  • Multiplication and accumulation
  • Make sure the discount affects the item(s) in these criteria
  • The item(s) in this condition can only participate in one offer

The method identifier is displayed in the Method column of the Criteria tab, in Data view.

The Multiplication and accumulation field

The Multiplication and accumulation field contains the method used for the coupon:

NOTE: When the Type is set to Total sales, this field is called Multiplication.

The available methods are:

Method Identifier Behavior
No multiplication (empty)

Basically, a coupon is only given once in a transaction, even if its criteria are matched several times. Each item sold may participate in more than one coupon, but only one (1) coupon discount may be applied on an item (or an entry).

Examples:
Coupon #1: 2 Sprites                          = $0.30 off
Coupon #2: 1 Coke + 1 bag of chips    = $0.50 off
Coupon #3: 1 Coke + 1 chocolate bar  = $0.40 off

Customer #1 buys 2 Sprites. They will get $0.30 off.

Customer #2 buys 4 Sprites. They will get $0.30 off.

Customer #3 buys 1 Coke, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.

Customer #4 buys 2 Cokes, 2 bags of chips and 2 chocolate bars. They will get $0.90 ($0.50 + $0.40) off.

Customer #5 buys 2 Cokes, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.
Based on validated criteria R

This method specifies that the coupon is given each time the criterion is reached.

The Based on validated criteria method has to be specified for each criterion that it applies to.

Examples:
Electronic coupon #1: 2 Sprites                           = $0.30 off
Electronic coupon #2: 1 Coke + 1 bag of chips     = $0.50 off
Electronic coupon #3: 1 Coke + 1 chocolate bar   = $0.40 off

The Based on validated criteria method is applied to each criterion.

Customer #1 buys 2 Sprites. They will get $0.30 off.

Customer #2 buys 4 Sprites. They will receive $0.60 (2 x $0.30) off.

Customer #3 buys 1 Coke, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.

Customer #4 buys 2 Cokes, 2 bags of chips and 2 chocolate bars. They will get $1.80 (2 x $0.50 + 2 x $0.40) off.

Customer #5 buys 2 Cokes, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.
Based on quantity sold Q

This method specifies that the coupon is given for each item sold that matches the coupon criterion.

As the Based on quantity sold method gives fractional quantities, it is useful for discounts per pound when a minimum poundage is reached.

This method has to be specified for each criterion that it applies to.

Examples:
Electronic coupon #1: 2 Sprites                           = $0.30 off
Electronic coupon #2: 1 Coke + 1 bag of chips     = $0.50 off
Electronic coupon #3: 1 Coke + 1 chocolate bar   = $0.40 off

The Based on quantity sold method is applied to each criterion.

Customer #1 buys 2 Sprites. They will get $0.60 (2 x $0.30) off.

Customer #2 buys 4 Sprites. They will get $1.20 (4 x $0.30) off.

Customer #3 buys 1 Coke, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.

Customer #4 buys 2 Cokes, 2 bags of chips and 2 chocolate bars. They will get $1.80 (2 x $0.50 + 2 x $0.40) off.

Customer #5 buys 2 Cokes, 1 bag of chips and 1 chocolate bar. They will get $1.80 (2 x $0.50 + 2 x $0.40) off.

Fractional quantities example:
Electronic coupon #4: 2 pounds of meat = $0.10 off per pound.

Customer #6 buys 2.65 pounds of meat. They will get $0.27 (2.65 * $0.10) off.

NOTE: For the Code 2 items (pre-scaled or random weight items), the Scalable field of the POS table must be checked ON on the item definitions. Otherwise, no discount will be given on these items since the system is not able to know how many pounds (or kilograms) are sold.

WARNING: Avoid using the Q (Based on quantity sold) method with a minimum amount of $0.01 as condition. In that case, the system can issue a fractional quantity of the coupon, not giving the expected result. A minimum quantity should be used instead.
Use the smallest S

The Use the smallest amount method specifies that when an item of that criterion is sold more than once in a transaction, the coupon is given according to the smallest item, based on the weight.

The Use the smallest amount method is useful when giving a discount per pound/kilogram on the smallest item.
Use the biggest B

The Use the biggest amount method specifies that when an item of that criterion is sold more than once in a transaction, the electronic coupon is given according to the biggest item, based on the weight.

The Use the biggest amount method is useful when giving a discount per pound on the biggest item.
Use the cheapest C The Use the cheapest method specifies that when an item of that criterion is sold more than once in a transaction, the electronic coupon is given according to the least expensive item, based on the amount.
Accumulate for bundle or percent A

This method specifies from which item (or entry) the electronic coupon discount expressed as a percentage is calculated.

The Accumulate for bundle or percent method can also be used on bundle coupons, i.e. selling a group of items at a special price. In this case, the discount is considered to be the price at which the bundle is sold and not that of the electronic coupon.

When the items that are part of the bundle have different prices, the system will use the average price of the items sold to calculate the discount given by the bundle e-coupon. To use the exact price of all items sold instead, the A method should not be used (only the CB method).

NOTES:
The Make sure the discount affects the item(s) in these criteria method can be used with Accumulate for bundle or percent.
The Accumulate for bundle or percent method mixed with the Based on quantity sold (Q) method means that the percentage is given according to the sales. This mix is used with the Promo ID sales, the Sub-department, the Department sales and the Total sales types to give a percentage on an accumulated amount.
Exclude this from accumulation X

This method specifies that the items that are part of the criterion are excluded from the other criteria.

It is used on the Item sales, Sub-department sales and Department sales types to exclude these items from other criteria defined with the Promo ID sales, the Sub-department sales, the Department sales and the Total sales types (exclude the smallest groupings from the biggest groupings).
Items of a Promo ID sales can be excluded from the Total sales type only.

NOTE: The exclusion is based on the same totalizer (totalizer 3 on both criteria, or totalizer 7 on both criteria, etc.).

The Make sure the discount affects the item(s) in these criteria field

The Make sure the discount affects the item(s) in these criteria method establishes the item (or entry) to which the coupon discount applies. Only one electronic coupons discount may be applied to an item or entry. Be sure to not create collisions.

If no match is forced, the electronic coupon discount is applied by the system to any available item in the transaction according to the smallest impact on the item sales.

If the electronic coupon discount cannot be applied on an item or entry, it will not be given.

It is also possible to match the discount to more than one criterion (up to a maximum of 3). In this case, the discount given is split proportionally between the specified items. This works for dollar discounts and bundle discounts; it does not work for percentage discounts and items for free. See the Electronic coupons - Examples topic for an example.

The item(s) in this condition can only participate in one offer field

The item(s) in this condition can only participate in one offer method specifies that the items referred to in that condition may be associated to only one electronic coupon in a transaction.
In this case, the system looks for the best coupon in the software, based on its classification.

This method applies with the Item sales, Promo ID sales, Sub-department, Department sales and Total sales types.

The item(s) in this condition can only participate in one offer method has to be specified for each criterion that it applies to.

Examples:

  • Electronic coupon #1: 2 Sprites = $0.30 off
  • Electronic coupon #2: 1 Coke + 1 bag of chips = $0.50 off
  • Electronic coupon #3: 1 Coke + 1 chocolate bar  = $0.40 off

The item(s) in this condition can only participate in one offer method applies to each criterion.

  • Customer #1 buys 2 Sprites. They will receive $0.30 off.
  • Customer #2 buys 4 Sprites. They will receive $0.30 off.
  • Customer #3 buys 1 Coke, 1 bag of chips and 1 chocolate bar. They will get $0.50 off.
  • Customer #4 buys 2 Cokes, 2 bags of chips and 2 chocolate bars. They will get $0.90 ($0.50 + $0.40) off.
  • Customer #5 buys 2 Cokes, 1 bag of chips and 1 chocolate bar. They will get $0.90 ($0.50 + $0.40) off.

Registration type field

The Registration type field contains alpha parameters used when the item is sold. Three alpha parameters are available in the coupon wizard:

Alpha parameter Value Description
Modify item registration WIT=REG This alpha parameter is used for changing all coupons behavior.
On the POS screen, the coupon will apply on the same line as the item.
Copy item registration WIT=NEWREG This alpha parameter changes the behavior of specific coupons instead of all of them.
On the POS screen, the coupon will apply at the end of the transaction.

On the POS screen, the coupon will apply at the end of the transaction.
Its description is the same with the coupon description.

The Multiply field

The Multiply field allows multiplying the points awarded into the transaction using an electronic coupon.

It works in combination with the Bonus field, which contains the multiplier factor, and with the Point program field that specifies the point program used for multiplying.

In the image above, the Multiply field being selected means that the number of points awarded in Point program M will be tripled when the coupon is applied.

By default, the multiplication of points by an electronic coupon has priority over the multiplication of points set into the system.
This behavior can be changed. Refer to the reseller for more details.

Creating a coupon

Below are the steps for creating an electronic coupon offering a percentage discount on a Promotion ID sale (20% discount on cookies).

1. In the Items menu, click Master.
2. In the Start window, click New next to the Coupon list entry.
3. In the main section of the coupon wizard, add the following coupon information:
Value field: select Percent.
Percent field: enter 20.
Coupon type field: select Electronic.
Description field: add a meaningful description.
Sub-department field: select a sub-department.
4. In the Criteria tab, click Add criteria.
Type field: select Promotion ID sales.
Program field: enter Cookies.
Minimum measure field: select Quantity.
Minimum value field: enter 1.
Multiplication and accumulation field: select Based on validated criteria.
Select Make sure the discount affects in item(s) in this condition checkbox.
Add a Manual description (optional).
Click OK to confirm the changes and to close the Add criteria window.
5. In the Details tab:
Funding field: select Store or Vendor.
Event type field: select Sale.
Enter a Start date.
Enter an End date.
6. Add any necessary description in the Descriptions tab.
7. Click Save in the toolbar to finalize the coupon creation.
The system will automatically assign the record a coupon code.
The Criteria tab will display the following information (Text view and Data view).
