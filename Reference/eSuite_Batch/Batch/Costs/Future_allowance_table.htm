eSuite - Future Allowance Table

eSuite > Items > Batches > Cost information

Information

The Future cost allowance table is used to enter the future allowances given on the cost of the selected items. It contains the future allowance information.

Displaying the Future cost allowance table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Cost information: the Cost batch wizard opens in a new tab.
  3. In the Batch type field, select Allowance.
  4. In the Batch field, select an allowance batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Cost allowance batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Cost batch, then Allowance: the Future cost allowances table opens over the work area.

Content

Here is the description of the Future cost allowance table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future allowance record is added, linking the future allowance record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future allowance record is added, the target value of the current cost record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Format The Format field contains the future buying format description.
When a future allowance record is added, the format of the current cost record is automatically displayed in the Format field.
Vendor The Vendor field contains the vendor ID and name. It links the item to a vendor.
When a future allowance record is added, the vendor ID and the vendor name of the current cost record are automatically displayed in the Vendor field.
Start date  The Start date field is a date field that contains the future allowance change date, i.e. the date on which the future allowance becomes effective. The starting date is inclusive.
When a future allowance record is added, the start date of the Batch header table is automatically displayed in the Start date field.
The starting date will become the allowance starting date in the corresponding cost record.
End date The End date field is a date field that contains the future allowance ending date, i.e. the date on which the future allowance is no longer effective. The ending date is inclusive.
When a future allowance record is added, the ending date of the Batch header table is automatically displayed in the End date field. The ending date will become the allowance ending date in the corresponding cost record.
Qualifier The Qualifier field contains the future qualifying conditions of the allowance. It is used for information purposes only.
When a future allowance record is added, the allowance qualifier of the current cost record is automatically displayed in the Qualifier field.
Off invoice The Off Invoice field contains the identification character of the future off invoice status.
This status specifies when the allowance value is given.
For details, see the Off invoice field section below.
Allow $ The Allowance $ (Allow $) field contains the future reduction amount per case for the allowance, applicable during the effective allowance period.
When a future allowance record is added, the allowance reduction amount of the current cost record is automatically displayed in the Allowance $ field.
Allow % The Allowance % (Allow %) field contains the future reduction per case expressed as a percentage for the allowance, applicable during the effective allowance period.
When a future allowance record is added, the allowance reduction percentage of the current cost record is automatically displayed in the Allowance % field.
Retail needed The Retail needed field contains the future sales condition to obtain the allocation. This sales condition may be expressed as a price at which the item must be sold, or a percentage of discount given on the selling price of the item.
The Retail needed field is currently used for information purposes only.
Reference The Reference field contains the future vendor's reference number used to receive the allocation reduction.
Minimum The Minimum (Min) field contains the future minimum quantity required, in cases, to obtain the allowance reduction.
When a future allowance record is added, the allowance minimum quantity of the current cost record is automatically displayed in the Minimum field.
Maximum The Maximum (Max) field contains the future maximum quantity, in cases, which limits the number of cases bought on the allocation.
When a future allowance record is added, the allowance maximum quantity of the current cost record is automatically displayed in the Maximum field.

The Off invoice field

The Off invoice field contains the identification character of the future off invoice status. This status specifies when the allowance value is given.

When a future allowance record is added, the allowance off invoice value of the current cost record is automatically displayed in the Off invoice field.

The following values are available:

Value Description
Off invoice The allowance value is subtracted from the invoice total.
This affects the Next cost and Cost values in the Cost table.
End of promotion The allowance value is given at the end of the promotion (as a check) according to the number of items bought.
This does not affect the Next cost and Cost values in the Cost table.
On statement The allowance value appears on the vendor's statement. It is given according to the number of items bought.
This does not affect the Next cost and Cost values in the Cost table.
Movement required The allowance value is given (as a check) according to the number of items sold.
This does not affect the Next cost and Cost values in the Cost table.
