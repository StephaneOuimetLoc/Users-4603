eSuite - Category Table

eSuite > Dimensions > Categories > List

Information

Categories in eSuite are used for classifying items that have similar characteristics but are assigned to different departments, sub-departments, vendors or to other eSuite divisions.

Juices, for example, can belong to the same sub-department, which makes it difficult to extract them from other items by using the type as a criterion. However, by creating a category called 'Vegetable juices' and linking the vegetable juices to that category, it becomes easy to select the vegetable juices only, regardless of the vendor, department or sub-department they belong to.

Categories can be created and managed in the Category table.

Display the Category table

In the Launcher, click the arrow in the Dimensions entry to display its menu and then:

  1. Click on Categories: the Category window opens in the Category work area.
  2. Click on List: the Categories tab opens in the same window, displaying the Category List.
  3. To modify an existing category, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit - Category F3 command in the bottom menu: the Category table opens, overlapping the work area.
  4. To create a new category, click on Add in the bottom menu of the grid: the Category table opens, overlapping the work area.

Content

Here is a description of the Category table fields and their functionalities:

FIELD DESCRIPTION
Number The Number field contains the category number.
It is a required field used to uniquely identify records in the Category list.
Descriptor The Descriptor field contains the description of the category.
It is not a required field, but a description should be entered as it helps identify the category in the categories reports.
Group 1 The Group 1 field contains the number of the first group of the category.
For details, see the Group 1 field section below.
Group 2 The Group 2 field contains the number of the first group of the category.
For details, see the Group 2 field section below.
Group 3 The Group 3 field contains the number of the first group of the category.
For details, see the Group 3 section below.
Sub link The Sub-department link (Sub link) field links the category to a sub-department.
For details, see the Sub-department link field section below.
Sub link txbl The Sub-department link taxable (Sub link txb) field links the category to a taxable sub-department.
For details, see the Sub-department link taxable field section below.

For details, see the Filter field section below.

The Group 1 field

The Group 1 field assists in further classifying the category items according to some particular meaning (see the example below). It works on the same principle as a sub-category.

Example

The following items are offered at the grocery store across the street:

Name of the item Category number
Strawberry juice 123121
Strawberry pie 104121
Peaches 105127
Canned peaches 122127

As an example, the Group 1 field could contain the first two (2) digits of the category number, with its own implicit meaning established by the store decision-makers:

Group 1 number Group 1 meaning
10 Perishable food
12 Non perishable food

The Sub-department link field

The Sub-department link (Sub link) field links the category to a sub-department.

Items are linked to sub-departments and categories. Categories, too, can be linked to sub-departments. Note that categories can be linked to different sub-departments than the items linked to them.

To link the category to a sub-department, either enter the sub-department number in the Sub link field or double-click the chevron in the Sub link field to select the sub-department from the Sub-department table.

NOTES:
A category may be linked to a non-taxable sub-department and a taxable sub-department at the same time.
The Sub-department link taxable field is useful when the item information comes from the Head Office (import), to preserve the existing structure of sub-departments of the stores (when the sub-department structure of the Head Office is different from that in the stores).

The Sub-department link taxable field

The Sub-department link taxable (Sub link txb) field links the category to a taxable sub-department.

Items are linked to sub-departments and categories. Categories can also be linked to taxable sub-departments. Note that categories can be linked to different sub-departments than the items linked to them.

To link the category to a taxable sub-department, either enter the sub-department number in the Sub link txb field or double-click the chevron in the Sub link txb field to select the sub-department from the Sub-department table. Note that the records must already exist in the Sub-department table for the Sub link txb window to display any choices.

NOTES:
A category may be linked to a non-taxable sub-department and a taxable sub-department at the same time.
The Sub-department link taxable field is useful when the item information comes from the Head Office (import), to preserve the existing structure of sub-departments of the stores (when the sub-department structure of the Head Office is different from that in the stores).

The Filter field

The Filter field contains the identification letters of the modules (Care, Buy, System, Manage) where the category must appear.

The category will only appear in the specified modules, or in none of the modules if the Do Not Show value is entered.

To enter the modules where the category must appear, enter the letter of the respective module or double-click on the chevron in the Filter field to select the module in the list of choices:

Module Code
Selling S
Inventory I
Costing C
Forecast F
Budget B
Hour T
Supply purchase E
Supply inventory P
Packing purchase Y
Packing inventory U
Receiving R
Trend Book K
DoNotShow N
E-Commerce W
