Random Weight Items

Random Weight item formats can be weighed or by count. The barcode printed on the label is a specific format: Leading with a 002 or 02, followed by 5 or 6 digits for the PLU of the item, then 4 to 5 more digits containing the actual sale price.
The barcode contains the sale price, not the sell price and a check digit to ensure barcode validity. 

In some places, such as Latin America, the barcode may contain the weight of the item as opposed to the sale price. Configuration varies depending on the physical scale settings. Please refer to the most common examples in the table below.



002 Code 02 Code
002PPPPPCXXXX
002 code + PPPPP for PLU + C for CheckDigit + XXXX for price/weight. 		02DPPPPPCXXXX
02 code + D for Department + PPPPP for PLU + C for CheckDigit + XXXX for price/weight.

02 code + D for Department + PPPPP for PLU + XXXXX for price/weight.

Weighed Items

Weighed Random Weight items, are items that are weighed on production scales in departments such as: Deli; Meat; Fruit and Vegetable. Once an item is weighed, the scale will print a label with the price.

When the item is scanned at the POS, the barcode will be decoded and the PLU will be used to find the item. The item will be added to the sale, using the sell price on the item, example $19.99/kg. The POS will then take the sale value from the barcode and divide it by the sell price to get a quantity, for example $5.00/$19.99 = 0.25; e.g.: Mozzarella is priced at $19.99/kg and the customer gets a piece weighing 0.25 kg. The item is worth $5.00. The scale will produce a label with a barcode on it which is placed on the mozzarella. When this is scanned at the front-end POS, the mozzarella will be added to the sale at $5.00.

For steps on how to program a weighed item, see Weighed items topic in Deploying Items section below.

By Count Items

By Count items, are items that are normally pre-packaged at the same price, with the same or slight variance in weight. The scale will print a label for the price; e.g.: Chicken wings are priced at 0.35¢/each and the customer gets 10 wings. The box is worth $3.50. The scale will produce a label with a barcode on it which is placed on the box containing the chicken wings. When this is scanned at the front-end POS, the box of chicken wings will be added to the sale at $3.50.

For steps on how to program a by count item, see By Count items topic in the Deploying Items section below.

NOTE: 'By Count' items that are weighed are not supported by many scales, do not set if not supported by the scale.

Exception flags

Exception flags are used to add settings that are unsupported by eSuite. They consist of manually programmed item values in order to achieve scale specific behavior(s) and passed in the HOSTCHNG file. They can be added to the URL (F2660) field in the POS table in eSuite.

For context, new features are regularly added to the market for production scales (existing or non-existing). To support these features, Invatron adds fields to their driver(s). By using exception flags, it is possible to use these new fields without having to wait for the next version/release of eSuite.

NOTES
• Exception flags may also be used to support existing Invatron parameters, which were not added to eSuite.
• If using the URL field, do not duplicate a key that already exists in eSuite.

Print Packaged Date/Time

In order to enable Print Packaged Date and Print Packaged Time for production scale labels, a value needs to be entered into the URL (F2660) field of the POS table.

Value Detail
XCHSCALE.PPD=Y,XCHSCALE.PPT=N Print date only
XCHSCALE.PPD=Y,XCHSCALE.PPT=Y Print date and time
XCHSCALE.PPD=N,XCHSCALE.PPT=N DO NOT print date and time
Expiration by hour

In order to enable expiry by hour instead of days, a value needs to be entered into the URL (F2660) field of the POS table.

Value Detail
SLT=0 Print expiry in days (default)
SLT=1 Print expiry in hours

Deploying Items

In order to properly deploy an item to a scale, the random weight format must be used (Code 002 or 02). Type the PLU, then press the front slash ( / ) key, the PLU will automatically expand into a random weight UPC format.

Weighed Item

  1. Create a random weight item with a 5 digit or 6 digit PLU.
    To do so, type the PLU desired and select '/' before hitting enter to confirm item creation. The code will expand into a Random Weight format.
    e.g.: Enter PLU '12345' then /. PLU will automatically expand to become '0021234500000'.
  2. You need to set a 'Sub-Department' and sell 'Price' of the item.
    e.g.: Sell price would be $19.99/kg based on example above.
  3. You need to flag the item as 'Scalable' in the POS table.

By Count Item

  1. Create a random weight item with a 5 digit or 6 digit PLU.
    To do so, type the PLU desired and select '/' before hitting enter to confirm item creation. The code will expand into a Random Weight format.
    e.g.: Enter PLU '12345' then /. PLU will automatically expand to become '0021234500000'.
  2. You need to set a 'Sub-Department' and sell 'Price' of the item.

Specific Behavior Flags

The 'Others' field flags can be used when wanting specific behavior for an item. For a summary on these flags please refer to the Other flags (POS table), Other flags (Sub-department table) and Other flags (Department table). For more detail, please refer to the descriptions in the table below.

The 'Divisor' field flags can be used when wanting a weight divisor for an item. For a summary on these flags please refer to the Divisor field - POS table document. For more detail, please refer to the descriptions in the table below.

Not Flagged Scalable with Others

  1. Scale Fix Weight - A scale fix weight item is an item that has no variance in weight.
    e.g.:     Box of frozen salmon filet, individual portion, 6oz.
  2. Scale Fix Weight by Count - A scale fix weight by count item is an item that has no variance in weight and is calculated by unit. The weight is tracked for inventory purposes.
    e.g.:     Store made cookies. Each cookie weighs 3 ounces but is sold by unit, 1/$1.00.
  3. Scale Fix Price - A scale fix price item is an item that has no variance in price. Weight may differ slightly, price remains the same.
    e.g.:     A pack of 5 boneless chicken breast at $10.00/pack.
  4. Scale Open Price - A scale open price item is an item that does not have a set price. The operator will be prompted to enter a price value once having placed the item on the scale. The system will calculate the weight versus price entered to provide sale amount.
    e.g:     Applies to any regular random weight item, but allows item price to be modified at the scale.
  5. Scale Turkeyware - A scale turkeyware item is an item that is weighable and by count. The turkey is being sold by unit regardless of its weight. The weight is tracked for inventory purposes, as normally throughout the year the item would be sold dollar per pound/kilogram.
    e.g:     Thanksgiving Holiday - Turkey Special, 3/$30.00.
    NOTE: 'By Count' items that are weighed are not supported by many scales. This should not be set if not supported by the scale.
  6. Scale Force Tare - A scale force tare item is an item that does not have an automatic tare set. The operator will be prompted to enter a tare value once having placed the container on the scale. Then, the tare will be zeroed out to proceed with weighing the item(s). The customer is charged for the weight of the item(s), not the container.
    e.g: Popcorn chicken $1.00/gram - Small, Medium, Large formats (Small box = 0.5 grams, Medium box = 0.8 grams, Large box = 1 gram).

Flagged Scalable with Others

  1. Scale Fix Weight - A scale fix weight item is an item that has no variance in weight.
    e.g.: Box of frozen salmon filet, individual portion, 6oz. Weight is tracked for inventory.
  2. Scale Fix Price - A scale fix price item is an item that has no variance in price. Weight may differ slightly, price remains the same.
    e.g.: A pack of 5 boneless chicken breast at $10.00/pack. Weight is tracked for inventory.
  3. Scale Force Tare - A scale force tare item is an item that does not have an automatic tare set. The operator will be prompted to enter a tare value once having placed the container on the scale. Then, the tare will be zeroed out to proceed with weighing the item(s). The customer is charged for the weight of the item(s), not the container. Weight is tracked for inventory.
    e.g: Popcorn chicken $1.00/gram - Small, Medium, Large formats (Small box = 0.5 grams, Medium box = 0.8 grams, Large box = 1 gram).

Flagged Scalable with Divisor

  • Item Quarter Pound - A quarter pound item is an item that is sold by a quarter of a pound. In order to achieve this, a value of 0.25 needs to be entered in the divisor field.
    e.g.: Ground beef patties, sold at a 1/4 lb each.
  • Item Half Pound - A half pound item is an item that is sold by a half of a pound. In order to achieve this, a value of 0.5 needs to be entered in the divisor field.
    e.g.: Ground turkey patties, sold at a 1/2 lb each.

Flagged Scalable with Divisor and Others

  • Scale Open Price - Quarter Pound - A sale open price quarter pound item is an item that is sold by a quarter of a pound at the amount entered by the operator. In order to achieve this, a value of 0.25 needs to be entered in the divisor field with 'SO' in the 'Others' field.
    e.g.: Pre-packed slices of cheese, sold at a 1/4 lb each. The manager decides to reduce the price because they are getting close to the expiry date.
  • Scale Open Price - Half Pound - A sale open price half pound item is an item that is sold by a half of a pound at the amount entered by the operator. In order to achieve this, a value of 0.5 needs to be entered in the divisor field with 'SO' in the 'Others' field.
    e.g.: Pre-packed meat, sold at a 1/2 lb each. The manager decides to reduce the price because the cut isn't satisfactory.
  • Scale Open Price - Kilogram/Pound - A sale open price half pound item is an item that is sold by a half of a pound at the amount entered by the operator. In order to achieve this, a value of 0.5 needs to be entered in the divisor field with 'SO' in the 'Others' field.
    e.g.: Pre-packed strawberries, sold at a 1 kg/lb each. The manager decides to reduce the price because they are no longer fresh.
