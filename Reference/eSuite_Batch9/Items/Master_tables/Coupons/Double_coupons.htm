eSuite - Preferred Shopper Double Coupons

eSuite > Items

Information

Please revise and update once all features are available in eSuite. (Raluca, 2019-09-06)

Another kind of promotion allowed by the Shopper level table is the add-on value on the vendor coupons, i.e. an additional amount given over the vendor coupon value.

Add-on values are applied on all types of vendor coupons, including the electronic ones.
By default, the system considers the add-on values as store coupons.

The present document focuses on the following:

  • Add-on values
  • Restrictions
  • Some vendor coupons do not multiply
  • Notes

Add-on values

There are two distinct fields used for add-on values:

  • Multiplying the value of the vendor coupons
  • Increasing the value of the vendor coupons by a fixed amount

Multiplying the vendor coupons values

The values of the vendor coupons may be doubled, tripled or increased by half, one and a half, etc. using the Coupon multiply field of the Shopper level table.

Increasing the vendor coupons values

The values of the vendor coupons may be increased by a fixed amount using the Add value field of the Shopper level table.
The amount must be a numeric value and must contain a decimal point as the dollars/cents separator. There is no fixed limit maximum amount. 

Restrict the add-on values

The add-on values given can be restricted by a maximum coupon value or a maximum discount amount on all vendor coupons.

Restrict the value that can be doubled

The add-on amounts given can be limited or restricted to a maximum value of the vendor coupons that can be doubled using the Maximum value field of the Shopper level table.
The amount must be a numeric value and must contain a decimal point as the dollars/cents separator. There is no fixed limit maximum amount.

The Maximum value field works with the Coupon multiply field or the Add value field. See the examples for details.

Restrict the discount amounts

The add-on amounts given can be limited, or restricted, to a maximum discount amount for all vendor coupons using the Max total field of the Shopper level table.
The amount must be a numeric value and must contain a decimal point as the dollars/cents separator. There is no fixed limit maximum amount.

The Maximum total field works with the Coupon multiply field or the Add value field. See the examples for details.

Some vendor coupons do not multiply

On some vendor coupons, it is specified that they cannot be 'doubled'. This means that the store cannot multiply the value of these vendor coupons or give an add-on amount as an additional discount. To avoid multiplying vendor coupons, make sure the Multipliable field of the POS table for the related items is not checked.

NOTE: The manufacturers should provide the stores with the list of the items for which the doubling on vendor coupons is not allowed.
