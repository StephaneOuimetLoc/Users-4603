eSuite - Future General Cost Table

eSuite > Batches

Information

The Future general cost table is used to change the future general cost information of the selected items.

Displaying the Future general cost table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Cost information: the Cost batch wizard opens in a new tab.
  3. In the Batch type field, select Cost general.
  4. In the Batch field, select a cost general batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Cost general batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Cost batch, then General: the Future general cost table opens over the work area.

Layout

The Future general cost table is divided in two parts: the main section and the Future general cost table grid.

Main section

* Click on a field to get information on it.

Future general cost table grid

* Click on a field to get information on it.

Content

Here is the description of the Future general cost table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future general cost record is added, linking the future general cost record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future general cost record is added, the target value of the current cost record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Format The Format field contains the future buying format description.
When a future general cost record is added, the format of the current cost record is automatically displayed in the Format field.
Vendor The Vendor field contains the vendor ID and name. It links the item to a vendor.
When a future general cost record is added, the vendor ID and the vendor name of the current cost record are automatically displayed in the Vendor field.
Date available The Date available (Date avail.) field contains the future date at which the item will be available.
When a future general cost record is added, the available date value of the current cost record is automatically displayed in the Date available field.
Vendor code The Vendor code field contains the future code used by the vendor to identify the item. This code is usually used to order the item from the vendor.
When a future general cost record is added, the vendor code value of the current cost record is automatically displayed in the Vendor code field.
Comment The Comment field contains a future comment on the cost of the item.
When a future general cost record is added, the comment of the current cost record is automatically displayed in the Comment field.
Authorized The Authorized (Auth) field specifies whether the current future cost record is the future official authorized cost (checked or null), or not (unchecked).
When a future general cost record is added, the authorization value of the current cost record is automatically displayed in the Authorization field.
Indicator The Indicator field contains the future identification characters of the item status.
When a future general cost record is added, the item status value of the current cost record is automatically displayed in the Indicator field.
Inventory The Inventory (Invent) field contains the future identification letter of the method which defines the item cost used for the inventory value of the item.
When a future general cost record is added, the cost method value of the current cost record is automatically displayed in the Inventory field.
Weight The Weight field contains the future weight of a case, including tare. It is usually used in warehouses for receiving information, or to calculate the shipping fees charged by weight.
When a future general cost record is added, the weight value of the current cost record is automatically displayed in the Weight field.
Cubic The Cubic field that contains the future volume of a case. It is usually used in warehouses for receiving information, or to calculate the shipping fees charged by volume.
When a future general cost record is added, the cubic value of the current cost record is automatically displayed in the Cubic field.
Cases The Cases (Cs) field contains the future number of cases per layer on a pallet. It is usually used in warehouses for receiving information.
When a future general cost record is added, the cases value of the current cost record is automatically displayed in the Cases field.
Layer The Layer (Layr) field contains the future number of layers on a pallet. It is usually used in warehouses for receiving information.
When a future general cost record is added, the layer value of the current cost record is automatically displayed in the Cases field.
Split code The Split code field consists of the future UPC code of the item contained in the current item. It is used for kits that contain only 1 item (1 UPC code). It works in combination with the Split quantity field. When a future general cost record is added, the split code value of the current cost record is automatically displayed in the Split code field.
Split quantity The Split quantity (Split qty) field contains the future number of the item contained in the current item. It works in combination with the Split code field.
When a future general cost record is added, the split quantity value of the current cost record is automatically displayed in the Split quantity field.
Sequence The Sequence field contains the future number assigned by the retailer used to sort items within a vendor.
When a future general cost record is added, the sequence value of the current cost record is automatically displayed in the Sequence field.
UOM The Unit of measure (UOM) field contains the future identification characters of the buying format available for the current item.
When a future general cost record is added, the unit of measure value of the current cost record is automatically displayed in the UOM field. 
Control The Control field contains the future identification letters of the control information on this item, used to add limits to the item.
When a future general cost record is added, the control value of the current cost record is automatically displayed in the Control field.
Delivery days The Delivery days (D. Days) field contains the future number of days needed to receive the item from the vendor after it is ordered.
When a future general cost record is added, the delivery days value of the current cost record is automatically displayed in the Delivery days field.
