Registry information

@dbHot(REGGEN,<RegistryKey>)

This macro returns a counter represented by a key in the registry.

Parameter Description Required/Optionnal
RegistryKey The name of the key.

Location for windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter		 Required
Return value
Return the value of the registry.
Format: Length 8 padded with zeros on the left.
Example
@dbHot(REGGEN,MyCounterA,1)      will return 00000001

@dbHot(REGGEN,<RegistryKey>,<Increment>,<MaxValue>)

This macro builds a counter with a key in the registry.
If <MaxValue> is passed to the macro, it represents the maximum value before returning to 0.

Parameter Description Required/Optionnal
RegistryKey The name of the key created.

Location for windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter		 Required
Increment Increment counter by this value Required
MaxValue The maximum counter value before going back to 0 Optionnal
Return value
Return the value of the registry before incrementation.
Format: Length 8 padded with zeros on the left.
Examples
@dbHot(REGGEN,MyCounterA,1)      will return 00000001
@dbHot(REGGEN,MyCounterA,1)      will return 00000002
@dbHot(REGGEN,MyCounterA,1,4)    will return 00000003
@dbHot(REGGEN,MyCounterA,1,4)    will return 00000004
@dbHot(REGGEN,MyCounterA,1,4)    will return 00000000
@dbHot(REGGEN,MyCounterA,1,4)    will return 00000001

@dbHot(REGGEN,MyCounterB,1,4)    will return 00000001
@dbHot(REGGEN,MyCounterB,5,4)    will return 00000000

@dbHot(REGVALUE,<RegistryKey>,<DataType>)

This macro returns a value of a registry key based on the data type.

See macro @dbHot(REG,...) for more flexibility.

Parameter Description Required/Optionnal
RegistryKey Name of the registry to find. Location is optional.

If no location is specified, the key is searched under :
   For windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
   Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter		 Required
DataType CURR:    Currency value
DWORD: Integer value
STRING: String value		 Optionnal
Default : STRING
Return value
The value of the registry
Examples
@dbHot(REGVALUE,MyRegistryKey)
@dbHot(REGVALUE,@PROFILE/MyRegistryKey)
@dbHot(REGVALUE,MyCounter,DWORD)
@dbHot(REGVALUE,\\HKEY_LOCAL_MACHINE\HARDWARE\DESCRIPTION\System\Version,STRING)

@dbHot(REG,<RegistryKey>,<TextDisplayFormat>,<DefaultValue>)

This macro returns a formatted value of a String registry key.

Parameter Description Required/Optionnal
RegistryKey Name of the registry to find. Location is optional.

If no location is specified, the key is searched under :
   For windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
   Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter

Please note: The name of the key can not be SET or DELETE.		 Required
TextDisplayFormat Format for displaying a text value. Optionnal
DefaultValue The default value will be used if the registry key does not exist. The format is also applied to the default value. Optionnal
Return value
The formatted value of the registry
Examples
@dbHot(REG,MyStringKey)
@dbHot(REG,@PROFILE/MyStringKey)
@dbHot(REG,MyStringKey,RAW,My default value)
@dbHot(REG,MyStringKey,HTML)

Location for windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter
 Required

@dbHot(VARSET,<RegistryKey>,<Value>)

This macro saves a String value in a registry key.

Parameter Description Required/Optionnal
RegistryKey Name of registry key to save.

Location for windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter
 Required
Value The value to save in registry key Required
Return value
None
Examples
@dbHot(VARSET,PARAM1,Hello) will store Hello in PARAM1 String registry key
@dbHot(VARSET,@PROFILE\PARAM2,Hello) will store Hello in PARAM2 String registry (\\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Store Management\Counters\Default\ where @PROFILE contain Default)

@dbHot(VARSET,<PoolParameterName>*)

This macro saves the value of one or more pool parameters in String registry keys of the same name.

Please note : Character * is mandatory to execute this macro.
                     This macro do the same thing as @dbHOT(REG,SET,<PoolParameterName>*)

Parameter Description Required/Optionnal
PoolParameterName Name of the pool parameter to save in registry.

Location for windows 7 or higher using 64 bit: \\HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter
Otherwise: \\HKEY LOCAL MACHINE\SOFTWARE\Store Management\Counter

Use * at the end of the pool parameter name allows you to define all parameters whose names begin with these characters.

Please note: Using the folder delimiter (\ or /) in pool parameter name allows the creation of subkeys in Windows Registry.
Example: A pool parameter with name=SECTION\MYKEY and a call to this macro : @dbHOT(REG,SET,SECTION\MYKEY) will create the key MYKEY in the subkey SECTION of // HKEY LOCAL MACHINE\SOFTWARE\WOW6432Node\Store Management\Counter		 Required
Return value
None
Examples
@WizSet(Param1=Value1)
@WizSet(Param2=Value2)
@dbHot(VARSET,Param1*)  will store Value1 in Param1 Registry (\\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Store Management\Counters)
@dbHot(VARSET,Param*)    will store Value1 in Param1 Registry and Value2 in Param2 Registry (\\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Store Management\Counters)
@dbHot(VARSET,Param3*)  will store nothing

Where @Profile contain Default
@WizSet(@Profile\Param1=Value1)
@WizSet(@Profile\Param2=Value2)
@dbHot(VARSET,@PROFILE\*)   will store Value1 in Param1 Registry and Value2 in Param2 Registry (\\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Store Management\Counters\Default\
