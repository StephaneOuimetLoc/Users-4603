Alternate Codes

 The Alternate code table contains other codes that can be used to buy, sell and maintain items. Each UPC can have an unlimited number of alternate codes. Each item can be sold using its UPC or its alternate codes.


Displaying the Alternate code table


In the Launcher, click the arrow in the Items entry to display its menu and then:


  1. Click on Master: the Start window opens in the Items work area.
  2. Click on Item list: the Wizard tab opens in the same window, displaying the Items - General List.
  3. Enter the search criteria and click Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select a record and click on Edit on the bottom of the item grid: the option menu is displayed.
  5. Select Special items and then Alternate codes: the Alternate code edit table opens overlapping the work area.


 

Layout

The Alternate code table is divided in two parts: the main section and the Alternate code table grid.

Main section

Alternate code table grid


  
* Click on a field to get information on it.


The Alternate code table grid offers quick information on the item alternate codes. The following columns are displayed: Target and Alternate code.


Columns are movable, resizable and may be selected/deselected for a customized experience. They allow sorting the records in an ascending or a descending order.
Records are displayed in ascending order by target, with the PAL target record selected by default. Selecting a record allows displaying and editing its information in the main section of the Alternate code table.

The Alternate code field

The Alternate code field contains an alternate code, i.e. a code different than the UPC.

The item can be sold using the alternate code. Each item can have an unlimited number of alternate codes.


Notes on Alternate code use


An alternate code should not match an existing UPC or PLU. If it does, the alternate code will be ignored by the POS.


If the alternate code is a regular scanned UPC, it should NOT be entered in a normalized format, i.e. no extra zeroes in front of the UPC. However, if the alternate code needs to be a random weight code (2) linked to a scalable item, such as PLU mapping in eWIC, it must be entered in its normalized form of 13 digits (e.g. 0022100200000, not 022100200000 or 22100200000).


The alternate code entered manually, such as PLUs mapped to WIC items, should be entered as is (e.g. 12, not 0000000000012).


Alternate codes and XchScale


The alternate code may also be used to send data to XchScale as a random weight item. No modification needs to be made to the main code (F01) for the item. This field does not require normalization to 13 digits to work with XchScale. You may simply begin the code with a two (2) followed by the PLU (5 or 6 digits) and five (5) zeros (0) for the price (e.g. 21234500000). The preceeding zeros (0) will be automatically appended by the system if not provided when sent to XchScale.

The Force package price # field


The Force package price # field contains the price to use for the alternate code. It is used to force the system to sell the item with a specific quantity when the alternate code is used.


The Force package price # field is designed to allow selling a case of an item just by scanning the alternate code.


The possible values are:


Use unit price

This forces the system to use the unit price with the quantity specified. For example, if the Price field is set to $0.99/1, 1 item is sold when the alternate code is scanned once. The price for this item is $0.99.

Use Pkg price / Price 2

This forces the system to use the package price with the quantity specified. For example, if the Package price field is set to $9.99/12, 12 items are sold when the alternate code is scanned once. The price for these 12 items is $9.99.

Note:
Price 2 is used with the Volume pricing.

Use Limited qty / Price 3

This forces the system to use the limited quantity price with the quantity specified. For example, if the Limited quantity field is set to $44.95/60, 60 items are sold when the alternate code is scanned once. The price for these 60 items is $44.95.

Note:
Price 3 is used with the Volume pricing.


It is possible to display at the POS and print on the receipt a description with these features. See the reseller for details.
