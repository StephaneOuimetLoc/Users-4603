eSuite - Inventory Trigger Table

eSuite > Items > Master > Item list

Information

The Inventory table is used to manage the item shortages and overages. It helps identifying when the items need to be ordered, with a suggested quantity to order.

Displaying the Inventory trigger table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens in the Items work area.
  2. Click on Item list: the Wizard tab opens in the same window, displaying the Items - General List.
  3. Enter the search criteria and click Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select a record and click on Edit on the bottom of the item grid: the option menu is displayed.
  5. Select Special items and then Inventory trigger: the Inventory edit table opens overlapping the work area.

Layout

The Inventory trigger table is divided in two parts: the main section and the Inventory trigger table grid.

Main section

* Click on a field to get information on it.

Inventory trigger table grid

* Click on a field to get information on it.

The Inventory trigger table grid offers quick information on the item shortages and overages. The following columns are displayed: Target and Trigger unit.

Columns are movable, resizable and may be selected/deselected for a customized experience. They allow sorting the records in an ascending or a descending order.
Records are displayed in ascending order by target. Selecting a record allows displaying and editing its information in the main section of the Inventory trigger table.

Content

Here is a description of the item Inventory trigger table fields and their functionalities:

FIELD DESCRIPTION
Main section
Target The Target field contains the target identifier, i.e. the store group.
It is a required field. Its default value is PAL. 
It is grayed out unless the system is set up for multi targets (see the dealer for more details).
Minimum unit The Minimum unit field contains the minimum quantity, in units, that you want to have in your inventory for the item.
Trigger unit The Trigger unit field contains the suggested quantity of items in units to order when the inventory is under the minimum quantity.
Maximum unit The Maximum unit field contains the maximum quantity, in units, that you want to have in your inventory for the item.
Turnover The Turnover field contains the number of times the turnover is done during a week for the item, i.e. how many times the shelf needs to be refilled during a week for the item.
It is used for information purposes only.
Shrink factor The Shrink factor field contains the percentage that is allowed to lose on a regular basis for this item. It is mostly used with perishable products.
It is used for information purposes only.
Backstore capacity The Backstore capacity field contains the number of units that can be stored in the backstore for this item.
It is used for information purposes only.
