eSuite Care - Point Redemption Examples

The points accumulated by the customers may be used to pay a part or the totality of their purchases. The discount given using points is called a point discount.

The system does not need a customer to be entered in the transaction to give and redeem points. The points given in a transaction must be used (redeemed) in that transaction since they cannot be added to a customer's point balance. The unused points are lost. See the Preferred shopper - Point redemption topic for details on the point redemption programming.

List of examples

  1. A fixed number of redeemed points is defined for each item (both rebate and item for free).
  2. Points are automatically redeemed on all items at the 10-point redemption rate.
  3. 50 points are redeemed when the store coupon tender is used.
  4. Points are redeemed on all items at the 10 point redemption rate when the customers have a minimum of 5000 points in their points balances.
  5. A fixed number of redeemed points is defined for each item (both rebate and item for free). These points are automatically redeemed from September 12 to September 18, 2019.
  6. A fixed number of redeemed points is defined for each item (item for free only). These points are reduced by half for the customers that are part of the shopper level 2.
  7. A fixed number of redeemed points is defined for each item (rebate only). These points are reduced by half on Wednesdays from 8:00am to 12:00pm.
  8. 50 points are automatically redeemed for each item sold for a point discount of $0.25 per item.
  9. Points are automatically redeemed on all items at the 10-point redemption rate for maximum point discounts of 50% of the item sales.
  10. 50 points are redeemed to obtain $0.02 off on a liter of gas.

1. A fixed number of redeemed points is defined for each item (both rebate and item for free)

This example uses the manual point redemption on preset items.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Allow redeem on item point method in the Redeem method field.

  4. Save the changes.

In the Redeem tab of the Price table, set up the information for each item:

  1. In the list of items (Items > Master > Items - General list), select the proper item.
  2. Type the number of points needed to obtain the rebate in the Discount points field.
  3. Type the amount of the rebate given in the Discount amount field.
  4. Type the number of points needed to obtain the item for free in the Free points field.

  5. Save the changes.

The Point redemption function must be executed manually at the POS for the customers to obtain a point discount on the items. Since both a rebate and a free value are defined for the items, the system prompts the cashiers to know which one to use.

The Point redemption function must be executed manually for each item on which a point discount could be given. The customer must have enough points; otherwise, the point discount is not given.

2. Points are automatically redeemed on all items at the 10-point redemption rate

This example uses the automatic point redemption on entry amounts.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Automatic redeem point method in the Redeem method field.
  4. Type 10 in the Red/penny field.

  5. Save the changes.

Set up the automatic redemption method used in theSet up the automatic redemption method used in the System configuration file:

  • Under the Customer link, enter Y;P50 in the Automatic Redeem Method parameter, and press the Enter key.
  • See your reseller for details on the Automatic redeem method parameter.

The point redemption function is automatically executed for each item. The system automatically gives the maximum point discount according to the number of points the customers have. Following this example, suppose the point redemption function is automatically executed on an item sold at $2.00. If the customer has 3000 points, they obtain the item for free (2000 points redeemed). If the customer has 750 points, they obtain a point discount of $0.75 (750 points redeemed), etc.

The customer must have enough points; otherwise, the point discount is not given.

3. Fifty (50) points are redeemed when the store coupon tender is used

This example uses the point redemption on functions with a fixed number of redeemed points defined for the store coupon tender.

Set up the store coupon tender (function 107) in the Function table:

  1. Type REDEEM=50 in the Alpha parameter field.
    See your dealer for more details on the REDEEM alpha parameter.

  2. Save the changes.

When the store coupon tender is used, 50 points are automatically redeemed from the customer's point balance.

The customer must have enough points; otherwise, the store coupon tender cannot be used.

4. Points are redeemed on all items at the 10 point redemption rate when the customers have a minimum of 5000 points in their point balances

This example uses the manual point redemption on entry amounts with a minimum points restriction.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Automatic redeem point method in the Redeem method field.
  4. Type 10 in the Reduction/penny field.

  5. Type 5000 in the Minimum points field.

  6. Save the changes.

The Point redemption function must be executed manually for the customers to obtain a point discount on the items. The system prompts the cashiers for the number of points to redeem. If the point balance of the customer is less than 5000 points, the point discount is not given.

The Point redemption function must be executed manually for each item on which a point discount could be given.

5. A fixed number of redeemed points is defined for each item (both rebate and item for free). These points are automatically redeemed from December 12 to December 18, 2019

 This example uses the automatic point redemption on preset items limited by dates.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will apply.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select the Automatic redeem point method in the Redeem method field.
  4. Save the changes.

In the Redeem tab of the Price table, set up the information for each item:

  1. In the list of items (Items > Master > Items - General list), select the proper item.
  2. Type the number of points needed to obtain the rebate in the Discount points field.
  3. Type the amount of the rebate given in the Discount amount field.
  4. Type the number of points needed to obtain the item for free in the Free points field.
  5. Select 12/12/2019 in the Start field.
  6. Select 12/18/2019 in the End field.

  7. Save the changes.

Set up the automatic redemption method used in the System configuration file:

  • Under the Customer link, enter I in the Automatic Redeem Method parameter, and press the Enter key.
  • See your reseller for more details on the Automatic Redeem Method parameter.

The point redemption function is automatically executed for each item having a point discount defined in the Price table. The system automatically gives the point discount as soon as the item is sold, if the customer has enough points and if the actual date is between December 12 and December 18, 2019 (the dates are inclusive).

Since both a rebate and a free value are defined for the items, the system automatically gives the best point discount (item for free or redemption rebate) according to the number of points the customer has. Following this example, if the customer has 3000 points, they automatically obtain the item for free (2000 points redeemed). If the customer has 1500 points, they automatically obtain a point discount of $0.75 (500 points redeemed). If the customer has 300 points, they do not obtain any point discount.

7. A fixed number of redeemed points is defined for each item (rebate only). These points are reduced by half on Wednesdays from 8:00 am to 12:00 pm.

This example uses the manual point redemption on preset items divided during a specific period.

This example is particular since it uses the upgrade-to-level concept. The targeted shopper level contains the period for which the points are reduced by half. For this period, the customers upgrade to another shopper level that reduces by half the number of points to redeem to obtain the same point discount.

In this example, the targeted shopper level upgrades to the shopper level 4 to reduce the points by half.

See the Preferred shopper - Shopper level upgrade topic for more details on the upgrade-to-level concept.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will be reduced by half.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. In the Upgrades tab, type 4 in the Upgrade to level field.

  4. Type 0800 in the Wednesday (Wed) Start field.
  5. Type 1200 in the Wednesday (Wed) Stop field.
  6. Save the changes.

Set up the point redemption information in the shopper level 4:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select shopper level 4 and open the corresponding Shopper level table.
  2. Select the Allow redeem on item point method in the Redeem method field.

  3. Type 2 in the Multiplier (redeem) field.

  4. Save the changes.

In the Redeem tab of the Price table, set up the information for each item:

  1. In the list of items (Items > Master > Items - General list), select the proper item.
  2. Type the number of points needed to obtain the item for free in the Discount points field.
  3. Type the amount of the rebate given in the Discount amount field.

  4. Save the changes.

The Point redemption function must be executed manually at the POS for the customers to obtain a point discount on the items. Since only the rebate values are defined for the items, the system automatically gives the rebate amount as point discount. The number of points redeemed from the points balance of the customer is reduced by half on each Wednesday from 8:00am to 12:00pm, if the customer is part of the targeted shopper level.

The Point redemption function must be executed manually for each item on which a point discount could be given.
The customer must have enough points; otherwise, the point discount is not given.

9. Points are automatically redeemed on all items at the 10-point redemption rate for maximum point discounts of 50% of the item sales

This example uses the automatic point redemption on entry amounts limited to a maximum percentage of the item sales.

Set up the Shopper level table information:

  1. In the list of shopper levels (Items > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will be reduced by half.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select Automatic redeem in the Redeem method field.

  4. Type 10 in the Red/penny field.

  5. Save the changes.

Set up the automatic redemption information used in the System configuration file:

  • Under the Customer link, enter F50:0.25 in the Automatic Redeem Method parameter, and press the Enter key.
    See your reseller for more details on the Automatic Redeem Method parameter.

The Point redemption function is automatically executed for each item. The system automatically gives the maximum point discount according to the number of points the customers have up to 50% of the item sales. Following this example, suppose the Point redemption function is automatically executed on an item sold at $2.00. If the customer has 3000 points, he obtains a point discount of $1.00 (50% of the item sale, 1000 points redeemed); if the customer has 750 points, he obtains a point discount of $0.75 (less than 50% of the item sale, 750 points redeemed); etc.

The customer must have enough points; otherwise, the point discount is not given.

10. Fifty (50) points are redeemed to obtain $0.02 off on a liter of gas

This example uses the manual point redemption on preset items.

In the example below, 50 points are redeemed for each liter of gas bought to obtain $0.02 off the liter.

Set up the Shopper level table information:

  1. In the list of shopper levels (Customers > Auxiliary tables > Shopper levels), select the shopper level for which the redeemed points will be reduced by half.
  2. Click Edit in the bottom menu to open the Shopper level table.
  3. Select Allow redeem on item in the Redeem method field.

  4. Save the changes.

In the Redeem tab of the Price table, set up the information for each item:

  1. In the list of items (Items > Master > Items - General list), select an item.
  2. Type 50 in the Discount points field.
  3. Type 0.02 in the Discount amount field.

  4. Save the changes.

The Point redemption function must be executed manually at the POS for the customers to obtain the point discount on the gas items. Since only a rebate is defined for the gas items, the system automatically gives $0.02 off the liter of gas bought as point discount. The number of points redeemed depends on the number of liters bought (50 x number of liters bought) and the number of points the customer has.

The customer must have enough points; otherwise, the point discount is not given. However, since the total point discount is given according to the number of liters bought, the point discount given on the gas items may be applied only to a part of the gas amount. For example, to obtain the maximum point discount when 20 liters of gas is bought ($0.02 x 20 liters = $0.40 off), the customer needs 1000 points (50 points x 20 liters = 1000 points). If a customer has 500 points, they will only have $0.20 off (500 points Ã· 50 points/liter = 10 liters; $0.02 x 10 liters = $0.20 off ).
