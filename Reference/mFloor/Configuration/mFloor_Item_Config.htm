mFloor - Item Configuration

mFloor Configuration

Information

The Config section within the Products module is used to configure the behavior when scanning an item.

 

Changing Item Config settings

Modifying the configuration settings for items is the same as that for any other mFloor module, the individual settings are altered and saved to a profile, in this way individual users may have their own settings and preferences.

To access and change the Item Config settings follow the procedure below:

  1. From the Products module click or tap Config.
  2. Enter the access code (the default value is 999) and click or tap OK.
  3. At the Config Item screen there are three options available to the user:
    • Remove not for sale - Remove the not for sale flag (F86) from the item when scanned.
    • Prompt to create - If a scanned item is not in the database, prompt user to create item.
    • Mark as verified - This feature is used to set the Price audit field (F41) in the Price table to generate reports on activity in the Electronic Journal of SMS Pro. To work effectively, this setting must be used along with initializing items on shelf and printing labels.

  4. When all settings have been changed click or tap save and the Save profile dialog box appears. To discard changes and return to Products menu, click or tap Cancel .
  5. Either enter the profile name associated to the user/operator or enter "1" and click or tap OK to save to the Default for ALL users. More information on profiles can be found in the Profile - Operator table field topic.
      
  6. On successful saving of the profile the Profile saved dialog box displays the saved profile name for confirmation. In the example below settings were saved to the "DEFAULT" profile, used by all users.
