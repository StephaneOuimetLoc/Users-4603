eSuite - Dimensions Vendors Table

eSuite > Dimensions > Vendors > List

Information

The Vendor table contains the supplier/vendor information.

It is security referenced, requiring a password. Different security levels allow different accesses. By default, clerks can only view the list, managers can add new vendors, owners and programmers have complete access.

Displaying the Vendor table

In the Launcher, click the arrow in the Dimensions entry to display its menu and then:

  1. Click on Vendor: the Vendor window opens in the Vendor work area.
  2. Click on List: the Vendors tab opens in the same window, displaying the Vendor List.
  3. To modify an existing vendor record, select it in the list and double-click on it, press the F3 key on the keyboard or click the Edit - Vendor F3 command in the bottom menu: the Vendor table opens, overlapping the work area.
  4. To create a new family record, click on Add in the bottom menu of the grid: the Vendor table opens, overlapping the work area.

The Target field

The Target field contains the target identifier, i.e. the store group. 

It is a required field, meaning that the new vendor records must have a valid entry in the Target field for the record to exist. It is used as a key field, along with the ID field, to uniquely identify records in the Vendor table. 

The default value of the Target field is PAL. The eSuite software is designed to support multi-stores or regional pricing. The Target field is grayed out (not editable) unless the Multi target option in the System.ini file is selected (see the reseller for details).              

To modify the target identifier (in a multi-stores setting), type the target identifier in the Target field or click in the field to select a target identifier from the Target list.

The ID field

The ID field contains the vendor's identifier. It is a required field, i.e. new vendor records must have a valid entry in the ID field for the record to exist. It is used as a key field to uniquely identify records in the Vendor List.

The ID field does not have to be numeric. It can contain characters, numbers and symbols, except for @ / \ & < > ' " and 'space'. An error message will be displayed if any of these symbols are entered in the filed.

Both the vendor number and name are displayed when a vendor is selected for an item in the Cost table.

The Account number field

The Account # field contains the store's account number at the vendor's. The account number appears on the printed purchase orders.

Special entries are used in the Account # field:

Special entries Description
STORE The STORE entry specifies that the vendor is an eSuite warehouse to which the buying documents will automatically be exported when they are closed.
In this case, the vendor ID must correspond to the store number of the warehouse, i.e. 998.

The Categories field

The Categories field contains the types of items that the vendor offers.

Special entries are used in the Categories field:

Special entries Description
SHIPVIA The SHIPVIA entry specifies that the vendor is a carrier. When the "ship via" entry is selected (in purchase orders, for instance), only the carriers are listed.
ANY The ANY entry specifies that any item from that vendor can be received, even if no item is associated to that vendor in the Cost table. This value is useful for two reasons. First, if some items are bought from different vendors, you might not want to program all the different vendors in the Cost table for each of those items.
Second, you would build a big purchase order which contains any item from any vendor, and then split it later.

The Unit of measure (UOM) field

The Unit of measure (UOM) field contains the identification characters of the buying format available for the current vendor.

The Unit of measure field has effects on the purchase orders. The buying formats available in the purchase orders depend on the values of the Unit of measure field. If the Unit of measure field does not contain any value, the case format is the only buying format available.

More than one buying format can be specified.

NOTE: The unit of measure may also be specified at the item level (the Unit of measure field in the Cost table). In this case, the value defined in the Cost table has precedence over the value defined in the Vendor table.
