eSuite - How to Display a Reason Prompt

eSuite > Items > Item auxiliaries

Information

Needs revising and updating once feature available in eSuite.
 Please replace the SMS images with the equivalent in eSuite. Raluca, 2019-09-03

Once the reason codes have been defined in the Reason code table, they can be attached to the use of some functions (usually on negative ones). This means that each time the cashier will use one of these functions, they will be prompted to enter a reason code.

As a hint, the reason codes related to the function used are displayed on the cashier screen with their descriptions. The cashier will only have to enter the proper reason code.

Here are the functions for which the use of the reason codes is possible:

  • Points adjustment
  • Global discount
  • Proportional discount
  • Item discount
  • Manual vendor and store coupons
  • Voids (last and previous)
  • Cancel order
  • Correction
  • Refund key
  • No sale
  • Price entry
  • Price modify
  • Tax exempt
  • Tax shift

These functions are not set up by default to prompt for a reason code. See your reseller for the programming.

Example

The Refund key function is set up to ask for a reason when it is used. At the POS, the list of reason codes related to the Refund key function are displayed to the cashier.

The cashier has to enter the code of the desired reason, press the Enter key, then continue the transaction as usual (in this example, by scanning the item to refund).

NOTE: The selected reason code is displayed in the Electronic Journal.
