The Label on price event discrepancy tool is used to:

  • Initialize labels for one or more items;
  • Print labels for items with label discrepancies.

NOTE: When using the Label on price event discrepancy tool, the Item Price window must be closed. Otherwise, the information being displayed in the tool may be inaccurate if an item has multiple price levels.

Using the Label price event discrepancy tool

Before using the Label price event discrepancy tool , the [Label]LABEL_REGGEN_METH key in Application.Ini file needs to be set:

  • LABEL_REGGEN_METH=0 for Detect retail price change
  • LABEL_REGGEN_METH=1 for Include Instore price as retail price
  • LABEL_REGGEN_METH=2 for Include Instore price and TPR as retail price
  • LABEL_REGGEN_METH=3 for See all events including the return to regular price

Displaying the Label price event discrepancy tool

In the Launcher, click the arrow in the Tools entry to display its menu and then:

  1. Click on Item validity checks: the Start window opens in the Execute work area.
  2. Select Label price event discrepancy. - item grid opens?
  3. Select the item to analyze.

The Label on price event discrepancy tool displays the differences between the last printed label an item (Printed column) and the information the label should display (Active column).
Only items with discrepancies will show in the browser.

There are several options when working with label discrepancies:

OPTION DESCRIPTION
Reset active item Resets the currently selected item Print label on next batch flag.
This simulates printing a label and can be used to initialize an item's label status.
Reset all items in browser Resets the Print label on next batch flag for all items in the browser.
This simulates printing a label and can be used to initialize the label status for all the items in the browser.
Refresh browser Refreshes the browser to display only items with differences between the current label to be printed and the last printed label.
This can be used when having performed Reset active item.
Print all items in browser Selects all the items in the browser.
Using the same methodology as Label on price events, it displays the Label by next event screen that allows to generate and print labels.
For more information on how to generate and print labels, see Generating and printing labels.
NOTES:
  • Changes must be deployed before using the Label on price event discrepancy tool.
  • Multi-target is not supported when using Print all items from browser. Items will only be displayed on the Label by next event screen with one target and one price level.
