Radiant Interface - Report Users Guide

Radiant Unmanaged

Information

This document provides a brief description on how data is imported between the LOC SMS back office server and the Radiant back office server.

Reporting

Data is polled approximately every two (2) minutes via the Exchange Provider (XAL). When found, the journal (PJR) files are imported then processed to populate the proper report tables (RPT_).

There are 2 types of journal (PJR) files that are generated and that can/will be imported from the Radiant back office server to the SMS back office server. First, End of Day PJR File Generation, which happens once the retailer has completed the daily close in the Radiant system. Second, PJR files can be manually generated by IT support. If SMS reports are blank, please contact the reseller in order to verify why.

NOTE: Prepay will affect the information totalizer 3505.
