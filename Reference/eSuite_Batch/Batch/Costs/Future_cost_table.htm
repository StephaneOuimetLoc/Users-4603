eSuite - Future Cost Table

eSuite > Items > Batches

Information

The Future cost table is used to change the future cost of the selected items. It contains the future cost basic information.

Displaying the Future cost table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Cost information: the Cost batch wizard opens in a new tab.
  3. In the Batch type field, select Future cost.
  4. In the Batch field, select a future cost batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Cost batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Cost batch, then Future cost: the Future cost table opens over the work area.

Layout

Content

Here is the description of the Future cost table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future cost record is added, linking the future cost record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future cost record is added, the target value of the current cost record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Format The Format field contains the future buying format description.
When a future cost record is added, the format of the current cost record is automatically displayed in the Format field.
Vendor The Vendor field contains the vendor ID and name. It links the item to a vendor.
When a future cost record is added, the vendor ID and the vendor name of the current cost record are automatically displayed in the Vendor field.
Start date The Start date field is a date field that contains the future cost change date, i.e. the date on which the future cost becomes effective. The starting date is inclusive.
When a future cost record is added, the start date of the Batch header table is automatically displayed in the Start date field.
Case size The Case size field contains the future number of units per case for the item.
When a future cost record is added, the case size of the current cost record is automatically displayed in the Case size field.
Base cost The Base cost field contains the future regular item cost per case.
When a future cost record is added, the base cost of the current cost record is automatically displayed in the Base cost field.
Fee % The Fee % field contains the future fees value for one case, expressed as a percentage.
For details, see the Fee % field in the Cost table.
Fee $ The Fee $ field contains the future fees amount for one case, expressed in dollars.
For details, see the Fee $ field in the Cost table.
Split The Split field contains the future fees amount by unit charged when the current item is bought in split case quantities.
See the Split field in the Cost table for more details.
