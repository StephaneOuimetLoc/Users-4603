eSuite - How to Create Items for Multitenant

eSuite > Items > New

Table of Contents

Creating a new item for a specific store
Creating a new item using target-of-target
Copying items from one store to another

Information

In Multitenant, every store is pointing to a single database at the host eliminating the store local database. Therefore, creating an item will generate the records in the master tables specifically for that store (target). 

Creating a new item for a specific store

1. From the launcher, select Items then New.

 

2. Enter the UPC in the Item Number field and the target for the store you're creating the item for.

*** Note if this is being created by an operator linked to a store then that store will be set by default in the Target field.
For example, if the operator used to login is linked to store 001 then only target 001 will be available.

 

3. You will then get prompted to create a new item. Select Yes.

 

4. Fill in the information for the master tables: MAIN, POS, PRICE, COST and Save.

*** Refer to the document "eSuite - How to Create Items" for more details on the layout for each field.




Creating a new item using target-of-target

Target-of-target is used to create an item for all the targets. This can be useful for a user at the head office that wants to create a new item for multiple stores without having to repeat the steps for each store.
In order to achieve this, the operator must NOT be linked to a store.

Follow the same steps from 1 to 4 and make sure to put a check for the option "Use target as target-of-target for new item".
Since the operator isn't linked to a store, the default target will be PAL.
 Enter the information for each table and Save.

 

You can verify if all targets were created in the master tables by searching for the item added. Select PAL for the target and include POS, Price and Cost then Execute:

 

You will see all the targets that have been created in the master tables for this item.

 

Copying items from one store to another

Copying items from an existing store could be very useful when adding a new store to the chain.

Login with an operator that is NOT linked to a store.

1. From the launcher, select Items then Items.

 

2. In the Target field enter the store that you want to copy the items from (in this example we're copying from store 001). Then search for a range of items by adding more criteria for item code start and end then Execute.

 

3. Select the item(s) that you want to copy and select Action>Copy to store.


4. In the copy items window, enter the destination target (in this example we're copying to store 002) then select to wich master tables you want to copy the item(s) and Confirm.
