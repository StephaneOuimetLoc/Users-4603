eSuite Buy - Transaction Line Table

eSuite Buy

Information

The Transaction line table allows editing the lines in the buying documents. It opens with the Edit selected line button located in the toolbar of the Lines section.

NOTE: In DSD documents, this button opens the payment tables (cash, check, drop).

Layout

The Transaction line table is divided in two parts: the main section and the tab section.

Main section

*Click on a field to get more information.

Tab section

*Click on a field to get more information. *Click on a field to get more information. *Click on a field to get more information.

Content

Here is the description of the Transaction line table fields and their functionalities:
FIELD DESCRIPTION
Main section
Line The Line field contains the order number of the current line in the set of lines.
Code The Code field contains the UPC of the current item.
Descriptor The Descriptor field contains the description of the current item.
Vendor code The Vendor code field contains the code used by the vendor to identify the current item.
Format The Format field contains the buying format description of the current item.
Authorized The Authorized field if the current cost record is the official authorized cost or not.
This field is useful in a multi-vendors case when an item has more than one cost record.
Order quantity  
Case The Case field contains the number of cases ordered for the item.
Case checkbox When selected, it makes the Case field non-editable.
Unit The Unit field contains the number of units ordered for the current item.
Unit checkbox When selected, the Unit checkbox field makes the Unit field non-editable.
Weight The Weight field contains the quantity in weight ordered for the item.
Weight checkbox When selected, the Weight checkbox field makes the Weight field non-editable.
Order cost  
Base cost The Base cost field contains the regular item cost per case for the item.
Net cost The Net cost field contains the net cost per case for the item.
Total cost $ The Total cost $ field contains the total cost for the item, according to the number of cases/units/weight ordered.
Unit  
Case size The Case size field contains the number of units per case for the item.
Total units The Total units field contains the total number of units ordered for the item.
Unit cost The Unit cost field contains the unit cost of the item.
Retail  
Price The Price field contains the retail price of the item.
Total The Total field contains the total retail amount of the item according to the number of items ordered.
Margin The Margin field contains the profit margin of the item.
   
Scalable The Scalable field specifies if the item is sold by weight and therefore scaled at the POS (checked) or not (unchecked).
Refund The Refund field specifies whether the current item is for refund (checked) or not (unchecked).
Free The Free field specifies whether the current item is for free (checked) or not (unchecked).
Delivery days The Delivery days field contains the number of days needed to receive the current item from the vendor after the order.
Order tab
Current The Current field contains the quantity in units that is available in inventory for the item.
On order The On order field contains the total number of units on order for the item. It includes the quantity on the purchase orders (open and closed) and on the open receiving for the current item.
The value is calculated by the system using the Calculate live counters job in the Scheduler.
On process The On process field contains the total number of units on process for the item. It includes the quantity on the sale orders (open and closed), on the shipping manifests (open and closed) and on the open invoices for the current item. The value is calculated by the system using the Calculate live counters job in the Scheduler.
Price The Price field opens the Price table for the current item.
Split item The Split item field displays the UPC of the item contained in the current item.
Used for kits that contain one item only (1 UPC code). It works in combination with the Split quantity field.
Split quantity The Split quantity field displays the number of split items that are contained in the current item.
It works in combination with the Split item field.
On process The On process field contains the total current quantity on process for the item contained in the current item. It includes the the number of units on the sale orders (open and closed), on the shipping manifests (open and closed) and on the open invoices for the item contained in the current item. The value is calculated by the system using the Calculate live counters job in the Scheduler.
Status The Status field contains the identification characters of the status of the current item.
The following values are possible: 0 (Active), BO (Allow BO), 1 (Discontinued), 2 (Seasonal), 3 (To be discontinued), 4 (Special order).
Comments The Comments field contains a comment on the purchase order or bill.
Note The Note field contains a note on the purchase order/bill.
Fees tab
Environmental  
Bottle deposit The Bottle deposit field contains the deposit amount charged by case.
Bottle deposit total The Bottle deposit total field contains the total deposit amount charged for the item.
Environmental fee The Environmental fee field contains the environmental fee amount charged by case.
Environmental fee total The Environmental fee total field contains the total amount of environmental fees charged for the item.
Landed fees  
Cost fee $ The Cost fee $ field contains the amount of the cost fees for one case, expressed in dollars.
Cost fee $ total The Cost fee $ total field contains the total amount in dollars of cost fees charged for the item.
Cost fee % The Cost fee % field contains the value of the cost fees for one case, expressed in percentage.
Cost fee % total The Cost fee % total field contains the total amount in percents of cost fees charged for the item.
Total fees The Total fees field total cost fees for the item, expressed in dollars.
Other fees  
Split unit fees The Split unit fees field contains the fee amount by unit charged when the current item is bought in split case quantities.
Fee 1 The Fee 1 field contains an additional fee amount charged for the item.
Fee 2 The Fee 2 field contains a second additional fees amount charged for the item.
Transportation - cubic  
Size The Size field contains the metric volume of a case.
Total The Total field contains the total metric volume for the current item.
Rate The Rate field contains the rate of the shipping fees charged by volume for the delivery of the merchandise for this vendor.
Fees The Fees field contains the total amount of the metric volume fees charged for the item.
Transportation - weight  
Size The Size field contains the weight of a case, including the tare.
Total The Total field contains the total weight for the item.
Rate The Rate field contains the rate of the shipping fees charged by weight for the delivery of the merchandise for this vendor.
Fees The Fees field contains the total amount of the weight fees charged for the item.
Deals tab
Allowance  
Start date The Start date field contains the date on which the allowance becomes effective.
End date The End date field contains the date on which the allowance is no longer effective.
Amount The Amount field contains the reduction amount per case for the allowance, applicable during the effective allowance period.
Percent The Percent field contains the reduction per case expressed as a percentage for the allowance, applicable during the effective allowance period.
Total The Total field contains the total reduction amount for the allowance.
OI (Off invoice) The OI field contains the identifier for the off invoice status of the allowance. It specifies when the allowance value is given.
The following values are possible:
  • 1 (Off invoice), subtracted from the invoice total.
  • E (End of promotion), given as a check at the end of the promotion according to the number of items bought.
  • S (On statement), given according to the number of items bought; appears on the vendor's statement.
  • M (Movement required), given as a check according to the number of items sold.
Only the Off invoice (1) value affects the amount of the total cost.
Minimum The Minimum field contains the minimum quantity required, in cases, to get the allowance reduction.
Maximum The Maximum field contains the maximum quantity, in cases, for which the allowance reduction can be applied on.
