Information

The Refund functions allows you to refund items which needs the intervention of the manager. There are Two types of Refunds :

The Refund Mode function 660 and the Refund Key function 650.

Refund Mode

The Refund mode function is used to refund many items. This Function must be performed in a separate transaction that is called a refunded transaction. Since it is a mode, the Refund Mode key displays refunded items in the Detail window with positive amounts. The Refund Mode function stays ON until the end of the transaction.

To refund items using the Refund Mode, follow these steps:

  1. Open an Epos session.
  2. Enter the manager login.
  3. Select the Refund Mode button.
  4. The "Refund mode 1=Yes/0=No?" message is displayed in the Entry window.
  5. In the Entry window, enter 1 to turn on the refund mode or 0 to cancel.
  6. Press the Enter key or click on the E button to validate your entry.
  7. The "Refund mode on" message is displayed in the Entry window and the epos status window. The Refund mode stays ON until the end of the transaction. The transaction type will become "REFUND"
  8. Enter the items you wish to refund.
  9. Press Subtotal and your epos should display the amount due to the customer under the message "REFUND DUE $00.00"
  10. Select a tender for the refund.

Refund Key

The Refund Key function is used to refund a single item at a time. The refunded items are identified in the transaction detail window with a negative amount. The Refund Key can be performed in a sale. You can mix sales and refunds in the same transaction.

To refund an item using the Refund key, follow these steps:

  1. Open an epos session.
  2. Enter the manager login.
  3. Call the Refund Key button.
  4. Enter the item you wish to refund followed by the Enter key or click on the E button to validate your entry. The item will display with a negative amount.
  6. Press Subtotal and your epos should display " Balance +/- $00.00" as a positive or negative value depending on the transaction.
  7. Complete the transaction by selecting a method of payment.
