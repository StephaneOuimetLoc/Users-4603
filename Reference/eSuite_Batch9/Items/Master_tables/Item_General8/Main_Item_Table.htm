eSuite - Main Item Table

The eSuite software uses relational tables to store item information. The Main item table contains the information that uniquely identifies items, UPC codes and general item information.

Displaying the Main item table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Master: the Start window opens in the Items work area.
  2. Click on Item list: the Wizard tab opens, displaying the Items - General List window.
  3. Enter the search criteria and click Execute: results will display in a record grid, in the Items tab of the same window.
  4. Select a record.
  5. Click Edit in the bottom menu to display the options.
  6. Select Master tables, then Main master: the Main item table opens, overlapping the work area.

Content

Here is the description of the Main item table fields and their functionalities:

FIELD DESCRIPTION
Main part
UPC The UPC field contains the item code.
It is a required field used to uniquely identify records in the Items - General List.
The UPC field is not editable by default: as there's only one single and unique UPC associated per item, a second UPC for the same item will never be accepted by the database.
Description The Description field contains the main item description. A short description, used at the points of sale, is entered in the POS information tabl..
Brand The Brand field contains the name of the item manufacturer.
Size The Size field contains the item size (e.g., 125 ml).
See the Size field section.
Alternate brand The Alternate brand field contains an alternate name for the item manufacturer.
It is generally used for a second language manufacturer name.
Alternate description The Alternate description field contains an alternate main description of the item.
It is generally used for describing the item in a second language.
Alternate size The Alternate size field contains an alternate item size.
It is generally used for indicating the item size in a second language.
Category The Category field allows displaying the list of categories and linking the item to the selected category.
It contains the category number and its description.
For details on categories in eSuite, see the Category table.
Family The Family field allows displaying the list of family codes and linking the item to the selected family code. 
It contains the family code and its description.
For details on family codes in eSuite, see the Family code table.
Report The Report field allows displaying the list of report codes and linking the item to a report code. 
It contains the report code and its description.
For details on report codes in eSuite, see the Report code table.
NACS The NACS field links the item to a NACS code.
NACS (National Association of Convenience Stores) codes are used by gas stations to identify what is being charged to a fleet credit card.
Account The Account field contains the general ledger account code associated to the item. 
It is used when the item must be associated to a different general ledger account than the usual one.
It is used for information purposes only.
Section The Section field contains the item section.
Sections are another way of grouping items.
Alias code The Alias code field contains an item UPC code used to group items. 
This way of grouping items may be used to facilitate orders.
Sell pack The Sell pack field contains the number of units included in the item.
Volume/weight The Volume/Weight field contains the unit volume (or unit weight) of the item.
Measure The Measure field allows displaying the list of measuring units available for the item and contains the selected value.
Shelf tab
Graphic file The Graphic file field contains the file name of the graphic that will display on the label.
The eSuite software only allows modifying or updating this field.
Container The Container field contains the item packaging format.
See the Container field section.
Width The Width field contains the item width in metric or imperial units, according to the system configuration.
Height The Height field contains the item height in metric or imperial units, according to the system configuration.
Depth The Depth field contains the item depth in metric or imperial units, according to the system configuration.
Long description The Long description field contains a more detailed description of the item.
Pharmacy tab
Medical code The Medical code field contains the drug code of the item.
The code is given by the store.
It is used for pharmaceutical items.
NDC/DIN The NDC/DIN field contains the standardized drug code of the item.
It is used for pharmaceutical items.
See the NDC/DIN field section.
Manufacturer code The Manufacturer code field contains the manufacturer code (for example, the 5 manufacturer digits in the UPC code) used to group items.
Competitive code The Competitive code field contains the UPC of an item that is considered as a competitive item to the current item (an item that can replace the current item).
It is used for pharmaceutical items.
Handling type The Handling type field contains the way the item must be handled and/or preserved.
Classification The Classification field contains keywords that are used to find items in the system.
It allows entering more than one keyword, separated by comma.
Owner The Owner field contains the store number of the store or host that is in the possession of the record.
The other stores cannot modify the record unless they're in its possession.
It is used for information purposes only.
NuVal The NuVal field contains information on the specific properties of the item (gluten free, organic, low sodium, etc.).
Marketing tab
Target customer The Target customer field contains the target age group of the customers that use the item.
It is used for information purposes only.
Target store The Target store field contains the type of store in which the item is sold.
The following options are available: Small stores, Medium stores and Large stores.
It is used for information purposes only.
Marketing justification for this item The Marketing justification for this item field contains an internal note on the item.
Miscellaneous tab
Inter-department code The Inter-department code field contains the code used for the inter-department transfers.
See the Inter-department code field section.
Weight The Weight field contains the net weight of the item. 
It is usually used in warehouses for shipping information.
Cubic The Cubic field contains the metric volume of the item in meters. 
It is generally used in warehouses for shipping information.
Ship box The Ship box field contains the number of boxes needed to ship the item.
It is generally used in warehouses for shipping information.
Operator owner The Operator owner field contains the number of the operator who is responsible with the purchases of the item.
The value can be entered or selected from a drop-down list displaying the operators in the Operator table.
Level The Level field contains the number of the minimum operator level needed to manage the current main item record.
See the Level field section.

The NDC/DIN field

The NDC/DIN field contains the standardized drug code of the item. It is given to pharmaceutical items.

The Drug Identification Number (DIN), used in Canada, is assigned by Health Canada to each approved drug.
The National Drug Code (NDC), used in the USA, is assigned jointly by the FDA (Food and Drug Administration) and companies; its first part is assigned by the FDA, and its last part, by the company that manufactures the pharmaceutical item.

The Inter-department code field

The Inter-dept code field contains the code used for the inter-department transfers, i.e. to transpose the inventory when the original item code is sold.

The inter-department transfer is used to transfer items from one department of the store to another one. It is important to manage correctly the inventory by department when items in a department are used in another department. This is often the case for the Deli department that uses meat, fruits, vegetables, etc. to produce recipes. See the Inter-department sales topic for all details.
