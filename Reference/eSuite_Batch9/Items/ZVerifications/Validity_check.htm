eSuite - Item Validity Check

eSuite > Items > Validity Check

Information

The Validity check option is used to identify the items in the database for which certain information is missing. The report returns the list of all items in violation of validity checks.

To access the validity check tool:

  1. In the launcher, click on Items.
  2. Click on Master: the Start window opens, displaying the item-related options.
  3. Click on Validity check.

The Validity check wizard

The Validity check wizard offers criteria selection that includes department, category, vendor, items, sellability validators, quality validators. The More options button adds further criteria on main item, POS, price and cost information.

Wizard queries

Here is the description of what the different queries in the Validity check wizard will list in the Validity check report.

QUERY DESCRIPTION
Sellability validators
Missing in Main item Lists items with a POS or price record, but no record in the Main item table.
Missing POS record Lists items with a main item record, but no record in the POS or Price tables.
Missing Price record Lists items with a main item record, but no record in the POS or Price tables.
Bad or missing sub-department/department Lists items that are not linked to a sub-department/department.
It also displays items linked to an invalid sub-department/department (one that does not exist in the database).
Bad tare link Lists items linked to an invalid tare value.
Zero price (and F1120 not EZ) Lists items with no active price or an active price that is equal to zero ($0.00).
Conflict between main and alternate code Lists items whose main code is used as an alternate code for a different item.
Bad link in the Kit table Used for kits that link two (2) items of the same type, such as a carton of cigarettes. In this case, the Bad link in the Kit table query verifies if the number of cigarette packs in the kit matches the size of the case that was sold.
Bad link to scale text Lists items that are linked to invalid scale text information.
Show like code with different price Lists items whose price is different from that of their like codes.
Quality validators
Missing cost record Lists items with no cost record.
Missing location record Lists items with no location record.
Bad or missing category Lists items that are not linked to a category.
It also displays items linked to an invalid category (one that does not exist in the database).
Bad or missing family Lists items that are not linked to a family.
It also displays items linked to an invalid family (one that does not exist in the database).
Bad or missing classification Lists items that are not linked to a classification.
It also displays items linked to an invalid classification (one that does not exist in the database).
Bad report code Lists items linked to an invalid report code (one that does not exist in the database).
Bad or missing vendor ID Lists items that are not linked to a vendor.
It also displays items linked to an invalid vendor (one that does not exist in the database).
Blank vendor code Lists items with no vendor code in the cost record.
Authorized vendor flag verification Lists items with no authorized vendor code in the cost table.
Zero cost Lists items with the cost equal to zero ($0.00).
Kit or split item with inventory Used for kits and split items that link two (2) items of the same type and have inventory.
For a carton of cigarettes, the Kit or split item with inventory query verifies if the number of cigarette packs in the kit matches the size of the case specified in the inventory.
Validate for latest changes only When checked, the report will only list the most recent changes to items.

The report returns the list of all items in violation of validity checks.
