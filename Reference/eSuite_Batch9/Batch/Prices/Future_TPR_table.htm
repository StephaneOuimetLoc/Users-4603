eSuite - Future TPR Table

eSuite > Items > Batches > Price Information

Information

The Future TPR table is used to change the future TPR prices of the selected items. It contains the future TPR price information.

Display the Future TPR table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Price information: the Price batch wizard opens in a new tab.
  3. In the Batch type field, select TPR.
  4. In the Batch field, select a price TPR batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Price TPR batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Price batch, then TPR: the Future TPR price table opens over the work area.

Layout

The Future TPR table is divided in two parts: the main section and the Future TPR table grid.

Content

Here is the description of the Future TPR table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier that uniquely identifies the batch.
Not editable. Its value comes from the Batch ID field of the Batch header table and is automatically displayed when a future TPR record is added, linking the future TPR record to the current batch.
Target The Target field contains the target identifier, i.e. the store group.
When a future TPR record is added, the target value of the current price record is automatically displayed in the Target field.
Not editable unless the Multi target option in the System.ini file is selected.
Level The Level (Lev) field contains the price level number.
See the Level field section below.
Start date The Start date field is a date field that contains the starting date of the future TPR event, i.e. the date at which the future TPR event becomes effective. The starting date is inclusive.
When a future TPR record is added, the start date of the Batch header table is automatically displayed in the Start date field. The starting date will become the TPR starting date in the corresponding price record.
End date The End date field contains the ending date of the future TPR event, i.e. the date at which the future TPR event is no longer effective. The ending date is inclusive.
When a future TPR record is added, the ending date of the Batch header table is automatically displayed in the End date field. The ending date will become the TPR ending date in the corresponding price record.
Price The Price field contains the future TPR price of the item.
When a future TPR record is added, the current TPR price of the item is automatically displayed in the Price field.
Quantity (price) The Quantity (price) field contains the future TPR quantity.
When a future TPR record is added, the TPR price quantity is automatically displayed in the Quantity (price) field. 
TGM (price) The TGM (price) field contains the future TPR profit margin on the item, in percentage.
It is used to generate the item TPR price.
Package price The Package price (Pkg price) field contains the future TPR package price of the item, i.e. a special price if a certain quantity of the item is bought by a customer within the same transaction.
When a future TPR record is added, the current TPR package price of the item is automatically displayed in the Package price field.
Quantity (package price) The Quantity (package price) field contains the future TPR package price quantity.
When a future TPR record is added, the current TPR package price quantity is automatically displayed in the Quantity (package price) field. 
TGM (package price) The TGM (package price) field contains the future TPR package price profit margin on the item, in percentage.
It is used to generate the item TPR package price.
Limited The Limited field contains the future TPR limited quantity price for the item.
When a future TPR record is added, the current TPR limited quantity price is automatically displayed in the Limited field.
Quantity (limited) The Quantity (limited) field limits the quantity of an item a customer may purchase at a special TPR price.
When a future TPR record is added, the current TPR limited quantity of the item is automatically displayed in the Limited quantity field.
TGM (limited) The TGM field related to the Limited quantity field contains the future TPR limited quantity profit margin on the item, in percentage.
It is used to generate the item TPR limited quantity price.
Reference number The Reference number field contains the future reference number related to the TPR price.
Kick back host The Kick back host field contains the future amount given back to the store by the host for each item sold at the future TPR price.
The amount is considered as a compensation from the host for the reduced price.
Used for information purposes only.
Discount $ The Discount $ (Dsc $) field contains the future amount of discount applied on the item when it is sold at the TPR price.
When a future TPR record is added, the current TPR amount of discount applied on the item is automatically displayed in the Discount $ field.
The Apply item disc $ field must be checked in the corresponding shopper levels for the Discount $ to work.
Discount % The Discount % (Dsc %) field contains the future percentage of discount applied on the item when it is sold at the TPR price.
When a future TPR record is added, the current TPR percentage of discount applied on the item is automatically displayed in the Discount % field.
The Apply item disc % field must be checked in the corresponding shopper levels for the Discount % to work.
Points The Points field contains the future number of points given to the customer when he buys the item at the TPR price.
When a future TPR record is added, the current TPR number of points given at the TPR price for the item is automatically displayed in the Points field.
Vendor The Vendor field contains the future amount given back to the store by the vendor for each item sold at the future TPR price.
The amount is considered as a compensation from the vendor for the reduced price.
Used for information purposes only.
Mix The Mix field links the item to a mix & match, which is used to mix items to match them to a TPR price.
When a future TPR record is added, the current TPR mix & match value of the item is automatically displayed in the Mix field.
Method The Method field contains the identification letters of the future TPR price method used, corresponding to a special TPR price behavior.
When a future TPR record is added, the current TPR price method of the item is automatically displayed in the Method field.

The Level field

The Level field contains the price level number.

When a future TPR record is added, the level value of the current price record is automatically displayed in the Level field.

NOTE:
Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.
