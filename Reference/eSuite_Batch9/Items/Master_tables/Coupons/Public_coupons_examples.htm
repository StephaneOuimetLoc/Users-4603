eSuite - Public Coupons Examples

eSuite > Items > Master > Coupon list

Information

Public coupons are store and vendor coupons given by the system when the specified criteria are reached.

They may only be used with the eCommerce option.

Here are some examples of how to create public coupons.

List of examples

  • Buy 5 pounds (or kilograms) of ground beef for a total of $50.00 in the transaction, get $5.00 off on the ground beef item
  • Buy packs of gum (any flavor) & for a total of $10.00 in the transaction, get $0.50 off on every pack of gum bought (the packs of gum are excluded from the required $10.00)
  • Buy 2 bags of Oreo cookies, get $1.00 off if the customer has accumulated at least 3000 points in the main point program
  • Buy a bottle of Coke and a candy bar for $2.00
  • Buy 2 bags of Oreo cookies, get 1 box of Premium Plus crackers for free

Buy 5 pounds

(or kilograms) of ground beef for a total of $50.00 in the transaction, get $5.00 off on the ground beef item

If the ground beef is sold using a code 2 item (random weight item), the Scalable flag must be checked in the ground beef item definition for the system to know how many pounds (or kilograms) the amount sold is associated to.

The Price field in the ground beef item definition must contain its price per pound (or kilogram).

1. In the main section of the coupon wizard, add the following coupon information:
  Value field: select Cents off.
  Discount field: enter 5.
  Coupon type field: select Public. 
  Description field: add a meaningful description. 
  Sub-department field: select the meat sub-department. 
 
 
2. In the Criteria tab, click Add criteria to add the first criteria.
  Type field: select Item sales. 
  Equal to field: enter the ground beef UPC. 
  Minimum measure field: select Weight. 
  Minimum value field: enter 5. 
  Select Make sure the discount affects the item(s) in this criteria.
This method forces the system to apply the public coupon discount on the ground beef item.
  Add a Manual description (optional).
  Click OK to confirm the changes and to close the Add criteria window. 
 
 
 3. In the Criteria tab, click Add criteria  to add the second criteria.
  Type field: select Sub-department. 
  Equal to field: enter the meat sub-department number.
Minimum measure field: select Dollar.
  Minimum value field: enter 50.
  Add a Manual description (optional). 
  Click OK to confirm the changes and to close the Add criteria window.
 
 
3. In the Details tab:
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
 
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
 
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The customer gets $5.00 off when they buy 5 pounds (or kilograms) of ground beef in the meat sub-department for a total of $50.00 in the transaction.
  • Even if the customer buys 10 pounds (or kilograms) of ground beef in the meat sub-department for $100.00 or more, they will only get $5.00 off. To repeat the electronic coupon, add the Based on validated criteria method (R) to each criterion that must be reached again.

Buy packs of gum

(any flavor) for a total of $10.00 in the transaction, get $0.50 off on every pack of gum bought (the packs of gum are excluded from the required $10.00)

In this example, all gum flavors must be included in the same promotion; the same promotion identification number must be entered in the Promotion ID field of the POS table for each gum flavor.

 1. In the main section of the coupon wizard, add the following coupon information:
  Value field: select Cents off. 
  Discount field: enter 0.5. 
  Coupon type field: select Electronic. 
  Description field: add a meaningful description. 
  Sub-department field: select a sub-department. 
   
 
2. In the Criteria tab, click Add criteria  to add the first criteria.
  Type field: select Promo ID sales. 
  Equal to field: enter the promotion identification for the packs of gum.
  Minimum measure field: select Quantity. 
  Minimum value field: enter 1. 
  Multiplication and accumulation field: select Based on validated criteria.
This method repeats the electronic coupon for each pack of gum (R). 
  Multiplication and accumulation field: select Exclude this from accumulation.
This method excludes the packs of gum from the required $10.00 (X).
  Select Make sure the discount affects the item(s) in this criteria.
This method forces the system to apply the electronic coupon discount on the packs of gum (M).
  Add a Manual description (optional).
  Click OK to confirm the changes and to close the Add criteria window. 
   
 
 3. In the Criteria tab, click Add criteria  to add the second criteria. 
  Type field: select Total sales. 
  Totalizer field: enter 3.  
Minimum measure field: select Dollar.  
  Minimum value field: enter 10.  
  Add a Manual description (optional). 
  Click OK to confirm the changes and to close the Add criteria window.  
   
 
 3. In the Details tab: 
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
   
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
   
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The customer gets $0.50 off when they buy 1 pack of gum (any flavor) and an additional total of $10.00 in the transaction.
  • If the customer buys 2 packs of gum (any flavor) and an additional total of $10.00, they will get $1.00 off (2 x $0.50).
  • If the customer buys 3 packs of gum (any flavor) and an additional total of $10.00, they will get $1.50 off (3 x $0.50), and so on.

Buy 2 bags of Oreo cookies

Get $1.00 off if the customer has accumulated at least 3000 points in the main point program

In this example, the customer gets $1.00 off when they buy 2 bags of Oreo cookies in the transaction if they have accumulated at least 3000 points. The customer code must be entered in the transaction to get the points balance.

 1. In the main section of the coupon wizard, add the following coupon information:
  Value field: select Cents off. 
  Discount field: enter 1. 
  Coupon type field: select Public. 
  Description field: add a meaningful description. 
  Sub-department field: select a sub-department. 
   
 
2. In the Criteria tab, click Add criteria  to add the first criteria.
  Type field: select Item sales. 
  Equal to field: enter the Oreo cookies UPC.
  Minimum measure field: select Quantity. 
  Minimum value field: enter 2. 
  Select Make sure the discount affects the item(s) in this criteria.
This method forces the system to apply the electronic coupon discount on the Oreo cookies item (M).
  Add a Manual description (optional).
  Click OK to confirm the changes and to close the Add criteria window. 
   
 
 3. In the Criteria tab, click Add criteria  to add the second criteria. 
  Type field: select Minimum point balance. 
  Program field: enter 1.
The main point program is identified by the character 1. 
Minimum balance field: enter 3000.  
  Add a Manual description (optional). 
  Click OK to confirm the changes and to close the Add criteria window.  
   
 
 3. In the Details tab: 
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
   
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
   
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • Even if the customer buys 4 bags of Oreo cookies in the transaction, they will only get $1.00 off if they have accumulated at least 3000 points.
  • For the customer to get $2.00 off when they buy 4 bags of Oreo cookies, $3.00 off when they buy 6 bags of Oreo cookies, etc., the Based on validated criteria (R) method must be added to the Oreo cookies criterion.
  • The Minimum point balance (PTS) method is considered as a threshold, meaning that the discount cannot be applied twice in the same transaction, even if the customer has 6000 points.

Buy a bottle of Coke and a candy bar for $2.00

 1. In the main section of the coupon wizard, add the following coupon information:
  Value field: select Bundle price. 
  Discount field: enter 2. 
  Coupon type field: select Public. 
  Description field: add a meaningful description. 
  Sub-department field: select a sub-department. 
   
 
2. In the Criteria tab, click Add criteria  to add the first criteria.
  Type field: select Item sales. 
  Equal to field: enter the Coke UPC.
  Minimum measure field: select Quantity. 
  Minimum value field: enter 1. 
  Multiplication and accumulation field: select Accumulate for bundle or percent.
This specifies that the bottle of Coke is part of the bundle (A).
  Select Make sure the discount affects the item(s) in this criteria.
This method forces the system to apply the electronic coupon discount on the bottle of Coke.
  Add a Manual description (optional).
  Click OK to confirm the changes and to close the Add criteria window. 
   
 
 3. In the Criteria tab, click Add criteria  to add the second criteria. 
  Type field: select Promo ID sales. 
  Equal to field: enter 1. 
Minimum measure field: enter Quantity.  
  Minimum value field: enter 1. 
  Multiplication and accumulation field: select Accumulate for bundle or percent.
This specifies that the candy bar is part of the bundle (A). 
  Add a Manual description (optional). 
  Click OK to confirm the changes and to close the Add criteria window.  
   
 
 3. In the Details tab: 
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date. 
  Enter an End date. 
   
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
   
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • The customer pays $2.00 for a bottle of Coke and a candy bar.
  • The electronic coupon discount, which is the difference between the total value of the 2 items and $2.00, is applied on the bottle of Coke.
  • For the customer to pay $4.00 for 2 bottles of Coke and 2 candy bars, $6.00 for 3 bottles of Coke and 3 candy bars, etc., the Based on validated criteria (R) must be selected in the Multiplication and accumulation field of the Add criteria window for both criteria.

NOTES:
For the discount to be added as a new entry in the transaction instead of being applied on the bottle of Coke, select Copy item registration or Create a coupon registration in the Registration type field of the Details tab.
Ideally, a price has to be specified in the Price field for each electronic coupon even if its value is a bundle price, in order to quantify the value of the electronic coupon. This quantification defines the priority of the electronic coupon and is useful when a limited quantity of electronic coupons can be given. In this case, the best electronic coupon is given according to its classification (value set in the Price field multiplied by the number of times it can be given).

Buy 2 bags of Oreo cookies

Get 1 box of Premium Plus crackers for free

In this example, the customer buys 2 bags of Oreo cookies and get 1 box of Premium Plus crackers for free.

For the box of Premium Plus crackers to be given for free, it has to be scanned in the transaction; however, the public coupon will not add it to the transaction.

 1. In the main section of the coupon wizard, add the following coupon information:
  Value field: select Buy X get Y free. 
  Buy X field: enter 2. 
  Get Y field: enter 1.
  Coupon type field: select Public.
  Description field: add a meaningful description.
  Sub-department field: select a sub-department. 
   
 
2. In the Criteria tab, click Add criteria  to add the first criteria.
  Type field: select Item sales. 
  Equal to field: enter the Oreo cookies UPC.
  Minimum measure field: select Quantity. 
  Minimum value field: enter 2. 
  Add a Manual description (optional).
  Click OK to confirm the changes and to close the Add criteria window. 
   
 
 3. In the Criteria tab, click Add criteria  to add the second criteria. 
  Type field: select Item ID sales. 
  Equal to field: enter the Premium Plus crackers UPC. 
Minimum measure field: enter Quantity.  
  Minimum value field: enter 1. 
  Multiplication and accumulation field: select Accumulate for bundle or percent.
This specifies that the Premium crackers are part of the bundle (A). 
  Add a Manual description (optional). 
  Click OK to confirm the changes and to close the Add criteria window.  
   
 
 3. In the Details tab: 
  Funding field: select Store or Vendor.
  Event type field: select Sale. 
  Enter a Start date.
  Enter an End date.
   
 
4. Add any necessary description in the Descriptions tab.
 
5. Click Save in the toolbar to finalize the coupon creation.
  The system will automatically assign the record a coupon code. 
   
  The Criteria tab will display the following information (Text view and Data view).
      

In this example:

  • If the customer buys 1 bag of Oreo cookies and 1 box of Premium Plus crackers, they will not get any discount.
  • If they buy 2 bags of Oreo cookies and 1 box of Premium Plus crackers, they will get the box of Premium Plus crackers for free.
  • If they buy 4 bags of Oreo cookies and 2 boxes of Premium Plus crackers in the same transaction, they will get 1 box of Premium Plus crackers for free.
  • If they buy 6 bags of Oreo cookies and 3 boxes of Premium Plus crackers in the same transaction, they will get 1 box of Premium Plus crackers for free, etc.
  • For the customer to get 2 boxes of Premium Plus crackers for free when he buys 4 bags of Oreo cookies in the same transaction, add Based on validated criteria (R) in the Multiplication and accumulation field of the Add criteria window for both criteria. In this case, a box of Premium Plus crackers will be given for free for every 2 bags of Oreo cookies bought by the customer in the same transaction.
