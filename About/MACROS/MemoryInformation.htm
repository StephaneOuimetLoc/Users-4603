@dbHot(NET_OFFICE,<StoreNumber>,<TerminalNumber>)

This macro returns the office path of a particular node by looking for the store and terminal number inside the exchange provider to get the path this node. The sending target path is used to get the office directory by removing the end of the driver path.
Example of sending driver: \\POS1\C\STOREMAN\OFFICE\XF001001\ and the result of the macro would be \\POS1\C\STOREMAN\OFFICE\.

Parameter Description Required/Optionnal
StoreNumber Searched store number Required
TerminalNumber Searched terminal number Required
Return value
Office path
Example
@dbHot(NET_OFFICE,001,901)

@dbHot(APPLICATION)

This macro returns what is the application process that is running this macro.

Return value
SMSHTTPS if it's the HTTPS server process
WINSERVER if it's the Win server process
WINPROCESSOR if it's the Processor process
WIN or TRS if it's the 350 applications.
Example
@dbHot(APPLICATION)

@dbHot(TABLEBYTERMINAL)

This macro returns the string _@STORE@TER if TableByTerminal=1 in System.ini
@STORE is replaced by the current store number.
@TER is replaced by the current terminal/station number.

Reminder : TableByTerminal specifies if the system avoids to use the same transaction tables on all POS terminals.

Return value
_@STORE@TER if TableByTerminal=1 in System.ini
Example
@dbHot(TABLEBYTERMINAL) return _001901

@dbHot(CURRENT,TEMP) 

This macro returns a unique name. Often used to generate a unique file name.

The name is composed by :
     Current store number.
     Current terminal/station number.
     A underscore character.
     Current session number (length = 3, left padding with 0).
     TMPDB counter defined in Windows Registry. Equivalent to @dbhot(regGen,TMPDB,1,30). (length = 2, left padding with 0).

Return value
Returns a unique name.
Examples
@dbHot(CURRENT,TEMP)            will return 001901_00112
TMP_@dbHot(CURRENT,TEMP)     will return TMP_001901_00113
