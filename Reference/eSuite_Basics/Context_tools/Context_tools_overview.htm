eSuite - Context Tools Overview

eSuite > Items > Batches

Information

The Context Tools window are visible on thessss right-hand side of all the eSuite modules. It displays information on the element selected from the active window on the desktop. This is why it is contextual, remember the right click in Windows? If you wanted to see the available options the right click was there. There is no right click on web pages instead there are contexutal tools that show which elements or settings you can view or modify based on the selected record. .

The layout of the Context Tools window will vary depending on the type of installation (single target or multi-target) and on the entry selected in the Launcher.

The image below displays the eSuite Care module open to the Cost of Goods report. In this case, item ...0000123 Energy Drink is selected in the item list. The Context Tools utility displays information that may not be shwon in the work area and also provides additional tools linked to that item.

Common fields

The list below shows the icon image, the field name and the functionality of the general buttons in the Context Tools window:

ICON FIELD DESCRIPTION
Collapse panel Hides the content of the panel.
Expand panel Shows the content of the panel.
Unlocked Indicates the value of the selected field is unlocked.
Click will toggle to a locked state.
Locked Indicates the value of the selected field is locked.
Click will toggle to a unlocked state.
Search Opens the search window for the selected field.
Linked query Opens the query menu linked to the selected field.
Edit Opens the table for the selected field.

Panels

The following panels are available in Context Tools:

  • Customer
  • Offer (only available in the Care module)
  • Transaction (description to come)
  • Activity log
  • Item
  • Auxiliary
  • Global change
  • Batch
  • Target of targets (multi-target installation)

Customer panel

The Customer panel displays information on the record selected in the customer list:

The Customer edit windows button provides access to the customer auxiliary tables:

Offer panel

The Offer panel displays information on the record selected in the customer list:

The Assign offer to customers button provides access to the Link offer table.

Activity log panel

The Activity log panel displays changes made in records and allows further action. Currently under development.

Item panel

The Item panel displays information on the item selected in the item list:

The Like code section displays two pull-down menus:

  • Apply: indicates the action to take relative to applying changes.
  • Action: indicates the action to take relative to the choice made in the Apply field.

NOTE: The Apply and Action fields work together with the corresponding fields in Target of target and the Create as needed field in Global change.

The Multi-window edit button provides access to the item-related tables and to the label generation tools.

Auxiliary panel

The Auxiliary panel displays information on the Dimensions entries in the Launcher (Promo ID, Sub-department, Category, Family, Report, Vendor ID):

It provides access to the tables and the query menus linked to the selected field.

Global change panel

The Global change panel displays the master tables (Main item, POS, Price and Cost) where changes have been made.

The Apply global change button provides access to the window where item changes may be reviewed and applied globally.

NOTE: The Apply global change window allows creating records when needed. It works together with the Apply and Action fields in Item and Target of target panels.

Batch panel

The Batch panel displays information on the batches selected in the batch list:

Target of target panel

The Target of target panel allows applying changes on the same item for multiple targets.

The Target of target panel displays two pull-down menus:

  • Apply: indicates the action to take relative to applying changes.
  • Action: indicates the action to take relative to the choice made in the Apply field.

NOTE: The Apply and Action fields work together with the corresponding fields in Item (Like code section) and the Create as needed field in Global change.

Example: running a linked query report on a category

The example given below provides the steps to run a Multi Totals linked query report on the item category.

  1. From the Items entry in the Launcher, click Master.
  2. Click Cost of Goods report from the Items start window.
  3. In the Items - cost of goods window, select a value in the Period field, choose the Start and Stop dates and add any other necessary criteria.
  4. Click Execute or press Enter: the window expands and the Items tab appears with the Items - Cost of goods report in the window.
  5. Select an item: Context Tools displays the item code along with additional fields from the Main item, POS, Price and Cost tables.
  6. To open the linked queries for the item category, click the Linked query button next to Category.
  7. The Category report menu linked to the item in the Context Tools window opens.
    Report menus linked to a particular item are identified by the word 'linked' in parentheses (LINKED), as highlighted below:
  8. Click Multi Totals to open the Items - multi totals wizard:
  9. Select a value in the Period field, choose the Start and Stop dates, the target (if multi-targets) and any other necessary criteria.
  10. Click Execute or press Enter.
  11. The resulting Items - Multi totals report displays the details of the totalizers for the date range selected.

The same procedure may be used to run linked queries on any other of the fields displayed in the Context Tools window. The fields shown in Context Tools change depending on the report or list from which the element is selected.
