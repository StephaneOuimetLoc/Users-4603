eSuite - Future Sale Price Table

eSuite > Items > Batches  > Price information

Information

The Future sales price table is used to change the future sales prices of the selected items. It contains the future sales price information.

Displaying the Future sales price table

In the Launcher, click the arrow in the Items entry to display its menu and then:

  1. Click on Batches: the Start window opens in the Batches work area.
  2. Click on Price information: the Price batch wizard opens in a new tab.
  3. In the Batch type field, select Sale.
  4. In the Batch field, select a price sale batch.
  5. Enter all other necessary criteria.
  6. Press Execute: results will display in the Price sale batch grid.
  7. Select a record and click Edit in the bottom menu of the grid.
  8. Select Price batch, then Sale: the Future sales price table opens over the work area.

Layout

The Future sales price table is divided in two parts: the main section and the Future sales price table grid.

The toolbar includes an additional button: View regular price . When activated, it opens the Price table for the selected item.

Content

Here is the description of the Future sales price table fields and their functionalities:

FIELD DESCRIPTION
Main section
Batch The Batch field contains the batch identifier.
It is not editable. Its value comes from the Batch ID of the Batch header table and is automatically displayed when a future sale record is added, linking the future sale record to the current batch.
Target The Target field contains the target identifier (the store group).
When a future sale record is added, the target value of the current price record is automatically displayed in the Target field.
It is grayed out (not editable) unless the Multi target option in the System.ini file is selected.
Level The Level field contains the price level number.
See the Level field section below.
Start date The Start date field contains the starting date of the future sale event, i.e. the date on which the future sale event becomes effective. The starting date is inclusive.
When a future sale record is added, the start date of the Batch header table is automatically displayed in the Start date field. The starting date will become the sale starting date in the corresponding price record.
Start time The Time (Start time) field contains the time at which the future sale event becomes effective during the effective day.
When a future price record is added, the effective time of the Batch header table is automatically displayed in the Time (Start date) field. The starting time will become the sale starting time in the corresponding price record.
If there is no specified time, the system uses 00:00 as the effective time.
End date The End date field contains the ending date of the future sale events, i.e. the date on which the future sale event is no longer effective. The ending date is inclusive.
When a future sale record is added, the ending date of the Batch header table is automatically displayed in the End date field. The ending date will become the sale ending date of the corresponding price record.
End time The Time (End date) field contains the time at which the future sale event terminates during the ending day.
When a future sale record is added, the ending time of the Batch header table is automatically displayed in the Time (End date) field. The ending time will become the sale ending time in the corresponding price record.
If there is no specified time, the system uses 23:59 as the effective time.
Price The Price field contains the future sale price of the item.
When a future sale record is added, the current sale price of the item is automatically displayed in the Price field.
Quantity (Price) The Quantity (Price) field contains the sale quantity for the price.
It is automatically displayed when the future sale record is added.
TGM (Price) The TGM Price field contains the future sale profit margin on the item, in percentage.
It is used to generate the item sale price.
Package price The Package price field contains the future sale package price of the item, i.e. a special price if a certain quantity of the item is bought by a customer within the same transaction.
When a future sale record is added, the current sale package price of the item is automatically displayed in the Package price field.
The Apply package price field must be checked in the corresponding shopper levels for the Package price to work.
See the Package field for more information on the package price behavior.
Quantity (Package price) The Quantity (Package price) field contains the sale quantity for the package price.
TGM (Package price) The TGM (Package price) field contains the future sale package price profit margin on the item, in percentage. It is used to generate the item sale package price.
Limited The Limited field contains the retail limited quantity price for the item.
Quantity (Limited) The Quantity (Limited) field contains the retail limited quantity for the item.
TGM (Limited) The TGM (Limited) field contains the future sale limited quantity profit margin on the item, in percentage.
It is used to generate the item sale limited quantity price.
Reference number The Reference number field contains the future reference number related to the sale price.
Kick back host The Kick back host field contains the future amount given back to the store by the host for each item sold at the future sale price.
The amount is considered as a compensation from the host for the reduced price.
The Kick back host amount is used for information purposes only.
Discount $ The Discount $ (Dsc $) field contains the future amount of discount applied on the item when it is sold at the sale price.
When a future sale record is added, the current sale amount of discount applied on the item is automatically displayed in the Discount $ field.
The Apply item disc $ field must be checked in the corresponding shopper levels for the Discount $ to work.
Discount % The Discount % (Dsc %) field contains the future percentage of discount applied on the item when it is sold at the sale price.
When a future sale record is added, the current sale percentage of discount applied on the item is automatically displayed in the Discount % field.
The Apply item disc % field must be checked in the corresponding shopper levels for the Discount % to work.
Points The Points field contains the future number of points given to the customer when he buys the item at the sale price.
When a future sale record is added, the current number of points given at the sale price for the item is automatically displayed in the Points field.
The Preset items point method must be selected in the Point method field of the corresponding shopper levels for the points to be given according to the item definition.
Vendor The Vendor field contains the future amount given back to the store by the vendor for each item sold at the future sale price.
The amount is considered as a compensation from the vendor for the reduced price. The Vendor amount is used for information purposes only.
Mix The Mix field links the item to a mix & match, which is used to mix items to match them to a sale price.
When a future sale record is added, the current sale mix & match value of the item is automatically displayed in the Mix field.
Method The Method field contains the identification letters of the future sale price method used, corresponding to a special sale price behavior.
When a future sale record is added, the current sale price method of the item is automatically displayed in the Method field.
Page The Page field contains the page number where the item will be found in the flyer.
It is used for information purposes only.
Ad The Ad field contains the identification letter of the advertising type used for the item.
For details, see the Ad field in the Price table.
Sale quantity limit The Sale quantity limit field limits the quantity of an item that a customer may purchase at a special price.

The Level field

The Level field contains the price level number.

When a future sales record is added, the level value of the current price record is automatically displayed in the Level field.

NOTE:
Starting v350x, Application.ini[TARGET_TRS] is used to determine which target to use for each item table.
Application.ini[TARGET_TRS]PRICE_TAB='PAL' will force the Host to use the PAL record.
Application.ini[TARGET_TRS]PRICE_TAB='@WIZGET(TARGET)' will force the Host to use the Global Store Number record.
Application.ini[TARGET_TRS]PRICE_TAB=ISNULL((SELECT TOP 1 SUB_PRI.F1000 FROM PRICE_TAB SUB_PRI JOIN LNK_TAB SUB_LNK ON SUB_PRI.F1000=SUB_LNK.F1000 JOIN LNK_TAB SUB_LNK2 ON SUB_LNK.F1056=SUB_LNK2.F1056 AND SUB_LNK.F1057=SUB_LNK2.F1057 AND SUB_LNK2.F1000='@WIZGET(TARGET)' JOIN STO_TAB SUB_STO ON SUB_LNK.F1000=SUB_STO.F1000 WHERE SUB_PRI.F01=PRI.F01 AND SUB_PRI.F126=PRI.F126 ORDER BY SUB_STO.F1937 DESC),'PAL'). That will use the proper target set for the Global StoreID selected.
