eSuite - Alternate Codes Table

eSuite > Customers > Master > General List

Information

The Customer alternate code table contains one or several additional codes (driver's license, credit card number, etc.) that can be used to identify customers.

Alternate codes are directly related to the customer's ID and so they can neither be added nor edited until the customer account has been created.

Customers can be entered in a transaction by using their customer ID or alternate codes.

Alternate codes

The Alternate codes list displays the customer alternate codes that can be used at the lane to identify customers. These codes may be a driver's license number, credit card number, etc. Each customer may have more than one alternate code.

By default, records are displayed in ascending order by customer ID.

In the bottom menu:

  • Edit provides access to the Alternate code table.

Below is a brief description of the information displayed by default in the Alternate codes list.

COLUMN DESCRIPTION
Customer ID The customer account number.
Alternate customer number The alternate number that may be used at the POS to identify the customer.
Type The identification letters that define the type of alternate code among the link, credit card, check account, shopper card, driver's licence, social security, telephone, WIC card and gift card numbers.
Main alternate code Displays the value 1 when the selected alternate code is the main alternate code for the customer's account. 
Target The target to which the alternate code applies for the customer.

Editing Alternate codes

In the Launcher, click the arrow in the Customers entry to display its menu and then:

  1. Click on Master: the Start window opens in the Customer work area.
  2. Select General list.
  3. Enter the search criteria in the wizard and execute.
  4. Select a record in Customers - General list.
  5. Click Edit in the bottom menu and select Alternate codes: the Alternate codes table opens, overlapping the work area.

The Alternate codes tables is also available from the Auxiliary tables under Customers. See the Customer auxiliaries topic for details.

Columns are movable, resizable and may be selected/deselected for a customized experience. They allow sorting the records in an ascending or a descending order.

The Target field

The Target field contains the target identifier, i.e. the store group. It is a required field, meaning that new account link records must have a valid entry in the Target field for the record to exist.

It is used as a key field, along with the Alternate number and Type fields, to uniquely identify records in the Alternate code table. Its default value is PAL.

The Target field is grayed out and not editable unless the system is set up for multi targets.

The Type field

The Type field contains the identification letters that define the type of alternate code. It is used as a key field, along with the Target and Alternate number fields, to uniquely identify records in the Alternate code table.

The following types are available:

FIELD VALUE DESCRIPTION
LINK The Auto link type defines the alternate number as the customer's link number.
It only works with an in-house points program; the Shopper card (PS) type should be used when the system is connected with a points program located on another host.
CC The Credit card type defines the alternate number as the customer's credit card number.
CK The Check account type defines the alternate number as the customer's check account number.
PS The Shopper card type defines the alternate number as the customer's shopper card number.
It only works when the system is connected with a points program located on another host; the Auto link type (LINK) should be used in the case of an in-house points program.
DL The Driver's license type defines the alternate number as the customer's driver's license number.
SS The Social security type defines the alternate number as the customer's social security number.
TL The Telephone type defines the alternate number as the customer's telephone number.
WC The WIC card type defines the alternate number as the customer's WIC card number.
GIFT The Gift card type defines the alternate number as the customer's gift card number.
